Interface definition "Medium"

Purpose: byte-level persistent storage
Features:
  • read/write byte arrays
  • get flags (Is it read-only? Is it removable?)
  • ejection with notification (listeners can veto)
  • remount (for jukeboxes/robots)
Maintainer: StefanReich
Status: stable, no modifications planned
JavaDoc: org.jos.fs.Medium
Source code: org.jos.fs.Medium
Current users: The FAT file system (under development)
Current implementors: A virtual disk driver
Possible implementors: Native device drivers (floppy, hard disk etc.)
Version history:
  • 11-MAR-98: stable version (part of FileSystem Release 3)
  • 13-FEB-98: initial release
