    1700 - 17:20, 18:00 - : General Discussion:

    • SourceServer glitches discussed
    • JeffWood will be WebSteward
    • adric will be PRSteward
    • meet again in a week to discuss progress. 1400 GMT next sunday.
    • we need to have some people with names and e-mail that new JOS members can contact (TheirryMa)
    • The state of CVS is discussed. According to RobertFit only kernel hackers have write access now, but it is available to other projects.

      export CVSROOT=:pserver:anonymous@jos.org:/sites/jos/cvs/cvsroot

    17:20 - : PRSteward responsibilities:

    • main updates suggested to be done by an UpdateSteward (JamundFer)
    • Each project lead should then report to this UPdateSteward who would update the static pages
    • A ststic page for first impressions and news is suggested (JamundFer)
    • This would be for the PRSteward (adric)
    • I figure the PRSteward would be the point of contact for outside orgs. This puts them responsible for all press relations, publicity, and maybe even advertising (when the time comes for it). They would need to work clsoely with other Stewards to keep all the data and presentation straight. (adric)
    • but shouldnt the PR page look nice.. for first impressions.. istead of a page that just has information like on the WIKI (JamundFer)
    • People browsing are: PotentialEndUsers, Suits, PressPeople, etc. Perhaps a static set of pages for each one of thiese would be approrpriate? (Clark)
    • it's hard to get someone to take final responsibiltiy, since anyone can (Clark)
    • The possiblity of page ownership on Wiki is discussed, as are alternate methods for changing pages (ftp, ssh ..)
    • It is agreed that even w/o software, ownership enforcement hasn't been a problem. It is noted that this could change in the future. (JeffWood)
    • It is agreed that page ownwership should be asserted in the existing wiki with a line like : "This page is maintained by: $name" "It is requested that you editthispagefreely/only add do not change/do not edit or change" (JamundFer)
    • It is further agreed that that should be added to the Wiki template, and it should be optional.
