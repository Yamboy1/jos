I started programming in Java about one and a half year ago. I tried to stay in touch with the jdks, IDEs, Swing and new features introduced by Sun. I reimplemented a C++ toolbox for GUI object synchronisation (If anybody is interested to get an impression: http://collide.informatik.uni-duisburg.de/Software/Docs/JavaMatchMaker/) in Java, supported several university lessons with some algorithms and applets and currently I am working on two major things: Integrating intelligent agents (and the appropriate architecture of a collegue) with Java applications and doing some work on software support tools within an educational context (no CASE tool). Java is currently my "native language" and I am glad to have less to do with ugly C / C++ code or Windows specific stuff (which I did for years)! My specific interests in FB are not yet clear. Although I am ready to do some work on special modules (as far as I have the time, but this is true for the most of us, I assume), e.g. debugging, semantic in-place support, etc. I am not so much interested in the GUI, but I could imagine to do some low-level "admin-work", e.g. "Who is Who in FB", URL lists or so.
