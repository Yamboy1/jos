Greetings.

I had an idea as to how to implement commands in the command line
interface of JOS. Why not have a centric registry with all the available
commands, and then use this registry to map the command function to a
specific class that implements org.jos.command.Command or something?

Example:

CD=org.jos.command.commands.CD
MKDIR=org.jos.command.commands.MKDIR

etc.?

The command interface could be:

public void processCommand(CommandLineSession session, String command,
String[] parameters);

and CommandLineSession can give access to the IN and OUT of the session
(screen, port, server, etc.) and other things, like current directory,
command history list, etc.

This way, users can implement their own commands, possibly download
remote registries of commands on the fly, or write their own commands
without the need to reimplement the entire command line sub system of
JOS. Even native commands can then be added by distributors, for
example, a server machine may want to have commands for direct access to
the fan and cooling system, or the UPS; the admin won't have to open a
special program to send the commands.

What do you think?
