> Ok I think we had the "lets not make it UNIX, we want other people to be able
> to use it too" argument, but if you want to get on the UI mailng list we
> could always design a BASH PUI (plug UI) after we
> get JADE designed and beat.

In what why does *supporting* bash commands make it harder to use? Is it more
difficult to type "cd <directory>" in Unix than in DOS? 

I don't think I'm the only one who wants to see a *powerful* CLI (the DOS
command line does not qualify). But just because the features are there, it
doesn't mean you have to use them. I've followed the discussion on the UI
mailinglist, but from them I certainly haven't got the impression that a
non-powerful leight-weight not-really-usable CLI is what people want.

We *do* want to support scripts, right? Then why limit the command line by not
allowing the same syntax there as in script files? One can easily add dir, del,
ren etc commands to that shell if that's easier to use (not my opinion).
