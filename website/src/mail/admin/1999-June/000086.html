Also, speaking of adding/maintaining CVS accounts,
how hard would it be to automate the process?  I think
we're "scaring" away developers because there are so
many things you need to do to get started.  First 
you need a WikiName and password, then you need to
add yourself to the MemberList, and join the right
mailing lists, and then get a separate CVS account.
I was thinking about trying to create a central
"account" management page/script that will allow
a person to manage all account information in a
single place.  Ideally choose/edit one WikiName,
account info, and password, set some check marks, and everything is
configured.  CVS commit access would have 
to be "approved" by module "owners" for pre-existing
CVS modules, but creating new modules (following
certain rules) should be easy/automatic.  Do you
think this is possible?

All the source is in the /source dir of the jos.org
account.  The dir structure is arranged per package
name I believe.  There is a raw (as submitted java
source), java source (pretty printed), and html version 
of each file.  If we could move them over to CVS,
it may be best to do so (just to remove the confusion
of having two source repositories, with no correlation
between what's in which).  I propose simply putting
them in the CVS (the pretty printed version of the
source) with only the CVS admin having commit access
to them.  Then as the authors start contributing,
they can be granted commit access.  Since SourceServer
has been broken since, I believe March 1999, and I
have only had one person ask why its broken, I don't
think the source in the SourceServer is currently
under active development (or if it is, its being
developed outside of JOS).
