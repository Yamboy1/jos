I think it is important to keep cvs accounts separate from wiki accounts
to keep security tight. New CVS accounts are created with the same wiki
name but a different password. I have asked people to encrypted their
desired passwords before sending them to me rather than just sending
plain text.

I also want to keep the number of CVS accounts to a minimum. It is
typical for there to be only a few people with CVS accounts (the inner
circle), while other contributers (the outer-circle) just send in
patches (eg. bug fixes) to the main developers. In our case, each
project leader definately has a CVS account. Other developers for that
project can be given an account if they are trusted.

There may be other ways - you're welcome to think of any. Perhaps we
could have two CVS servers - one for main developers, and another for
untrusted developers. It would *somehow* be automatically updated to
reflect changes in the main repository. I don't see this happening any
time soon though :-)
