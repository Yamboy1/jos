Hi Ryan
The resultant boost in speed is actually of secondary consideration.
The main reason is that you want to get away from calling an interpreter 
for every instruction.
In GW-BASIC or PERL you have a script that calls an interpreter and loads 
that into RAM.
Nearly 5 MB of RAM is tied up until that is completed. If something loops 
without closing,  just a measly one hundred of them will bring a 512 MB RAM 
dual processor machine to a halt. THAT's the main reason for porting to C 
and compiling to self-sufficient code.

The speed gains are a welcome fringe benefit :-)
Helmut
