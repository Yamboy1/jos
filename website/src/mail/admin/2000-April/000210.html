Hi Ryan
As far as I know, there is a robots.txt file higher up in JOS. That was put 
there years ago.
Crawlers spider that machine continuously, but I doubt that they ever go 
that deep into JOS.
And wherever they go, they didn't stop going there when the cvsweb.cgi was 
chmoded to not executable.

Just reading text and data files should not be a problem either.

Compare for example all the big sites that have a program to check through 
hundreds of thousands of postcard data files every midnight to see it they 
are due for delivery on that day. Even though the stats and the back-ups 
are run at the same time, it's no problem for a machine like that, because 
all those programs use fast C code that executes and gets out of the way, 
unlike PERL or BASIC.
