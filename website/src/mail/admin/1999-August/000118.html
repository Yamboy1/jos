Could permissions on files be the global wiki problem?

From my own website, I get this message when my httpd program does not have
permissions according to Linux. Since the httpd program cannot read a file,
I cannot read a file either.

What was the solution? I had to change the permissions on files in the
htdocs directory. If wiki is CGI, the httpd program must have execute
permissions on the CGI program. Without execute permission, all requests to
the CGI program return that "forbidden" message.

I imagine that the CGI program could have been uploaded/replaced on Sunday
night or Monday morning. Did someone forget to set the permissions of the
CGI program?

Unfortunately, there is no way to *prevent* someone from interactively
setting the wrong permissions. You can encourage people to set the right
permissions.
