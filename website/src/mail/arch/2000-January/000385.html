I just wrote a program to grab .class files from rt.jar. The byte: scheme
converts any InputStream to a byte[]. The zip: scheme treats the embedded
URI as a ZIP file. Here's the basic idea:

URI uri = new URI( "in:byte:zip:[file:rt.jar]/java/lang/Class.class" );
byte[] buf = (byte[]) uri.getObject();
if ( buf == null ) {
  return;
}
:

How many lines of code would this take without the Smart API?
