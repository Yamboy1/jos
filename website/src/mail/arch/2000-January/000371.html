Distribution 6 of Smart API - Release 2a is coming soon. It is part of JOS
Distribution 1e.

Get ready to take the Universal Browser for another test drive. With the OS
program controller, it puts "Universal" back in the Universal Browser. It
is nothing short of spectacular!

The Program Browser can run all these kinds of programs:

Java applications (run:main:)
JOS Programs (run:program:)
José Programs (run:jose:)
native OS programs (run:os:)

Here is how you get native OS programs in a mostly-bytecode shell.

1. The Universal Browser uses the Program Browser.
2. The Program Browser uses the Smart API.
3. The Smart API uses the all-new OSProgramController class.
4. The OSProgramController class converts run:os:somecommand to
run:program:org.jos.gchii.osprogram1a.OSProgram?cmd=somecommand
5. The OSProgram class uses java.lang.Process to start a native program and
redirect its output to a console.

-----

Run Java tools:

java tool - run:os:java someclass
javah tool - run:os:javah -jni someclass
jar tool - run:os:jar cvf somejar somefile
javadoc tool - run:os:javadoc -classpath somedirectory somepackage

-----

Run eric tool - run:os:eric
Run miniserver tool - run:os:minisrv

-----

About Windows(tm)

The run:os: subscheme is a prefix to a Windows(tm) command line. The
standard output is redirected to the browser's console.

Windows Explorer - run:os:explorer
Notepad - run:os:notepad
Registry Editor - run:os:regedit
Telnet - run:os:telnet
HTML Browser - run:os:start index.html
batch files - run:os:start somefile.bat

-----

About Linux(r)

The run:os: subscheme is a prefix to a Linux(r) command line. The standard
output is redirected to the browser's console.

ls - run:os:ls
tar - run:os:tar -xvzf somearchive.tgz
cat - run:os:cat somefile
find - run:os:find -name somefile

-----

The run: scheme is not just interactive. Embed it in your mostly-bytecode
applications to run other programs.

 - Redirect output to a file.
 - Run other programs within a script (batch mode).
 - Run other programs, including CGI scripts, from within a Java servlet.

-----

About desktop

Create a custom desktop with buttons for your favorite commands. The
Desktop Browser supports all kinds of schemes, including the new run:os:
scheme. Create a desktop file with links to both bytecode and native OS
programs.
