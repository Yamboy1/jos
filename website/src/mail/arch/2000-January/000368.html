Within Smart API, the run:main: sub-scheme corresponds to the main()
method. Of course, a main() method is a public static void main( String[] )
method.

The MainProgramController implements a run:main: sub-scheme. In a clever
twist, this controller uses run:program: to invoke MainProgram. In turn,
MainProgram uses practical reflection to invoke any class which implements
a main() method.

Unfortunately, this was not working correctly in Distribution 5 of Smart
API - Release 2a. It is fixed for the next distribution.

-----

For example, the CommandLineBrowser class implements a main() method. To
start the CommandLineBrowser, use the following URI:

run:main:org.jos.gchii.commandlinebrowser1a.CommandLineBrowser

The java tool equivalent is:

java org.jos.gchii.commandlinebrowser1a.CommandLineBrowser

-----

To pass parameters (arguments) to main(), a URI uses a syntax like this:

run:main:org.jos.gchii.browser1b.UniversalBrowser?http://www.jos.org/

The java tool equivalent is:

java org.jos.gchii.browser1b.UniversalBrowser http://www.jos.org/

-----

The experimental command-line browser uses AWT components for its virtual
console, which the virtual console is under construction. I would like to
use a command-line browser on Linux and across a TELNET connection. I would
like to use a command-line browser like I use an MS-DOS Box on Windows 95.
