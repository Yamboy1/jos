Introduction

The build module is part of CJOSCVSOnSourceForge. It contains a directory for each target archive (jar).
  • khimera1a directory. This directory is for building the khimera1a.jar archive.
  • khimera1b directory. This directory is for the khimera1b.jar archive.
  • khimera1c directory. This directory is for the khimera1c.jar archive.
  • khimera1d directory. This directory is for building the khimera1d.jar archive.
  • khimera1d directory. This directory is for the khimera1d.jar archive.
  • jose1d directory. This directory is for the jose1d.jar archive.
  • smartapi1a directory. This directory is used to build the smartapi1a.jar archive, which contains the latest packages closely related to Smart API 1a.
  • smartapi1b directory. This is for the smartapi.jar archive, which contains the latest packages closely related to Smart API 1b.
  • smartapi2a directory. This is for the smartapi2a.jar archive, which contains the latest packages closely related to Smart API 2a.
  • smartapiutil1 directory. This is for the smartapiutil1.jar archive, which contains packages related to SmartAPI and UniversalBrowser.

Process

Copy source code from an original directory to one or more build directories and compile it there. The original Java code is found in a java_x_ module, such as java0 or java1. Source code is copied with the Package Archive utility, which is part of BulkMake. This process enables us to maintain a single copy of a source code file and compile it (carefully) for more than one product.
