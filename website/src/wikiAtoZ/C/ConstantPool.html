BytecodePages

Article contributed by GilbertHerschberger (10 May 2000).

Introduction

A constant value is a constant because it never changes. Constant values are used in all kinds of programs and all kinds of programming languages.

Since a constant is read-only and may be used many times in a program, it makes sense to optimize constants and eliminate any duplicates. By pooling, or putting all constants together in the same place, the size of a program can be greatly reduced. It eliminates wasted space. In a low level language, a programmer might maintain a constant pool by hand. In a high level language, programming tools are often used to maintain a constant pool.

Using an ID, a program can copy a constant value from the constant pool. When a new value is added to a constant pool, it is given a unique ID. An ID is an index into the constant pool.

Location

Constants can be pooled at a variety of places. A constant pool can exist at the class, module or application level. A library is equivalent to an application. A static linker is often used to eliminate duplicate constants at a module or application level.

About Java

In Java, a class is the smallest independent unit of programming. Each class contains its own constant pool. This makes a class self-contained. The javac tool, for example, merges the constant pool for a class automatically.

When a constant pool is implemented at the class level, duplication may exist at runtime. Depending on your virtual machine, duplicate constant values may or may not be eliminated during dynamic linking.

In Java, a package is nearly equivalent to a module. It is possible to further reduce the size of a package by creating a package-level constant pool. A static linker, like the jar tool, could merge the constant pool of all classes in a package.

Types

A constant pool must be able to contain values of different types. Often the prefix to a constant value contains a type ID and possibly a payload length.

The constant pool is greatly simplified in Java because of its limited number of primative types. And, it is impossible to create a constant object.

Code

Throughout a program, machine code (or bytecode) can reference a constant value through its ID rather than its value. Through the ID, the type and value can be copied out of the constant pool. As an object, a constant pool must have some mechanism to return the type and value when given an ID.

Most often, a constant is copied onto the thread's stack. Because these values are read-only, the constant pool is thread safe.

Optimization

For a platform-independent constant pool, like the one used in Java, an important optimization should be performed. The constant pool should be translated to a platform-dependent form.

  void *getIndependent( int index );
  void *getDependent( int index );

The getIndependent() method always returns the value actually saved in a class file, without regard for platform (processor). Likewise, the getDependent() method always returns the platform-specific value.

A signed Java integer of 32-bits can be stored in a platform-independent form inside a constant pool. The most significant bits are saved in the first octet. The least significant bits are saved in the last octet. The getIndependent() method returns a pointer to the integer this way.

The getDependent() method must be recompiled for each platform. On a big-endian processor, the getDependent() method returns the same value as getIndependent(). It may be written like this:

  void *getDependent( int index ) {
	 return getIndependent( index );
  }

On a little-endian processor, the getDependent() method returns a different value. The order of the bytes must be precisely reversed. It may be written something like this:

  void *getDependent( int index ) {
	 if ( isFirst ) {
		isFirst = FALSE;
		convertConstantPool();
	 }

	 if ( index < 1 ) {
		return NULL;
	 }

	 if ( index > size ) {
		return NULL;
	 }

	 return list[ index ];
  }

When writing convertConstantPool(), remember that it can safely use getIndependent().

The list field is an array of void pointers. It is declared something like this:

  int size;
  void **list;

When allocated, the size of this array is always equal to the number of entries in the constant pool. The size of a constant pool can be determined with the getConstantPoolCount() method.

Summary

A constant pool is created by a compiler and linker. When duplicates are eliminated, it can reduce the static size of a program.
