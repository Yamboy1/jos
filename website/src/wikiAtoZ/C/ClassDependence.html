Introduction

A class may depend on other classes at runtime. A class depends on other classes for its superclass, its interfaces and class references in its constant pool. There are two kinds of class dependence, direct and indirect.

Pure Reflection for C might be the first to implement class dependence functions.

Direct class dependence

This kind of dependence can be extracted from a single class. No other class files are required. At this level, there is no concept of a CLASSPATH or CODEBASE. A program can use PureReflection to display a list of all classes an arbitary class depends on.

Indirect class dependency

This kind of dependence goes the extra step. It is recursive. Not only does it extract information from a single class, it extract information from the classes a class depends on. A collection of classes is required. Something like CLASSPATH and/or CODEBASE may be useful to define a collection of classes.

Dynamic class dependency

Indirect class dependency does not begin to address the needs of dynamic class dependency. Dynamic dependency is created when an arbitrary class is loaded dynamically at runtime, possibly by name.

Missing classes

Indirect class dependency can be used to determine if a class depends on a non-existant class. A class is missing. Missing classes may help determine an appropriate composition for CLASSPATH and CODEBASE passed to a virtual machine.

Dependency changes over time. Class files, which may have existed at compile time, are moved or deleted.

The correct composition of CLASSPATH and/or CODEBASE may be unknown. Experimenting with CLASSPATH is possible through a class dependence utility.

Specification

I would like a program capable of generating an HTML page. For each package, it would generate one page. For each class, it might display a row in an HTML table.

This program would accept a given CODEBASE (codebase=somecodebase) and generate the corresponding pages using pure reflection. An optional package name (package=somepackage)would reduce the scope of the program. For each class within scope, it would display the name of a class and the name of all its dependent classes. It would use one font for direct dependence and another for indirect dependence. In addition, it might use emphasis for dependence on interfaces.

Any class that cannot be loaded using the given CODEBASE must be clearly indicated and an additional page should contain a list of all missing classes.

Dependent classes might be listed in alphabetical order, rather than in the order they are found during reflection.
