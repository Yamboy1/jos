Introduction

This article explains how to compile the Java source code for JJOS using the Java Development Kit (JDK) for Linux.

Procedure

This procedure illustrates how to compile Java source code without using BulkMake.
  • Prerequisite Install and configure the Java Development Kit (JDK), version 1.1.6 or higher.
  1. cd common/bytecode Switch to the base directory for common, architecture independent, Java source code. This corresponds to the default package in Java. This directory is the base of the tree which corresponds to the JJOS-related packages.
  2. make Run the make script found in this directory. It invokes the javac tool, as needed, for each Java source code file. It invokes the zip program to create the architecture independent jjos.zip archive.
  3. make install This target copies the jjos.zip archive, if necessary, to the arch/host/bytecode or arch/i386/bytecode directories. Each version of JJOS may approach this differently.

About the jjos.zip archive.

The jjos.zip archive is not created from scratch. Rather, it starts with a copy of classes.zip and adds or overwrites content. For more information, see also common/bytecode/Makefile.

Dependencies

  • Java Development Kit (JDK), version 1.1.6 or higher. Each version of JJOS may require a specific version.
  • /java/lib/classes.zip The make script for the Java source code looks for the classes.zip file in the /java/lib directory. This directory can be a symbolic link to your JDK, or you can copy this file from a JDK on another machine. It is also possible to use the jjos.zip archive from one of the binary editions of JJOS as your copy of classes.zip.
