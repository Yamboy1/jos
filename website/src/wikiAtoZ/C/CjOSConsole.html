Article contributed by GilbertHerschberger (9 March 2001).

Introduction

Like the CjOSKernel, there will only be one console in machine code. Therefore, any method in a virtual machine can get the console using the following C function: 
extern "C" mc_Console *mc_getConsole();

For a given build, there can be one and only one console. It is not necessary, therefore, to manage multiple consoles at runtime.

There may be many implementations of a console from one platform to the next. While the interface to the mc_getConsole() function is platform independent, its implementation is platform dependent. Each kernel implementation must provide a mc_getConsole() function and extend the mc_Console interface (pure virtual class).

Example

Here is an example of how a typical method of a virtual machine might send a message to the console. This example sends "Hello, World!" to a console. 
#include <mc_console.h>

void helloWorldExample() {
  const char *MIDDLE = "orl";

  mc_Console *console = mc_getConsole();
  console->print( "Hello," );
  console->print( " " );
  console->kprintf( "W%sd", MIDDLE );
  console->println( "!" );
}

Prototype

A console is completely free and independent of a kernel. A console might exist before a kernel object is instanciated. The kernel object represents the functions of a kernel. The actual kernel object may be written in assembler, C, C++ or any other programming language capable of emitting machine code. 
class mc_Console {
  public:
	 virtual void print( const char *v ) const = 0;
	 virtual void println( const char *v ) const = 0;
	 virtual void kprintf( const char *fmt, ... ) const = 0;

  protected:

  private:

};

Not required

The kernel is not required to maintain a pointer to the console. The console is not required to maintain a pointer to the kernel. Through the mcgetKernel() and mcgetConsole() functions, the console and kernel are always available to other subsystems.

Implementation

Here is one possible implementation of the mc_getConsole() function. It instanciates a console object the first time it is invoked. Otherwise, it returns the same console object each time it is invoked. 
// <EM>Custom</EM>Console.cpp
#include <mc_console.h>

static mc_Console *console;

class <EM>Custom</EM>Console : public mc_Console {
  public:
	 <EM>Custom</EM>Console();
	 virtual ~<EM>Custom</EM>Console();

	 virtual void print( const char *v ) const;
	 virtual void println( const char *v ) const;
	 virtual void kprintf( const char *fmt, ... ) const;

  protected:

  private:

};

<EM>Custom</EM>Console::<EM>Custom</EM>Console() {
}

void <EM>Custom</EM>Console::print( const char *v ) const {
:
}

void <EM>Custom</EM>Console::println( const char *v ) const {
  print( v );
  print( "n" );
}

void <EM>Custom</EM>Console::kprintf( const char *fmt, ... ) const {
:
}

mc_Console *mc_getConsole() {
  if ( console == NULL ) {
    console = new <EM>Custom</EM>Console();
  }

  return console;
}

About JJOS

The mc_getConsole() function has been implemented for both the host and i386 builds for JJOS 1i. With the implementation of a standard C library, it becomes clear that most C/C++ code never comes in direct contact with the console. Rather, most code invokes an I/O function in the standard C library, such as putch(), puts(), printf(), etc., and that I/O function uses the built-in console.
