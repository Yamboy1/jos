Article contributed by GilbertHerschberger (11 December 2000).

Introduction

The CJOS kernel is part of the CJOS Project. Early on in its design, it supports a KernelModule.

Module

A kernel module is a plug-in. Each kernel module must implement a well-known interface. In C++, the interface is a pure virtual class. 
class km_Module {
  public:
    virtual void init() = 0;
    virtual void start() = 0;
    virtual void stop() = 0;
    virtual void destroy() = 0;

    virtual bool isInit() const = 0;
    virtual bool isStart() const = 0;

  protected:

  private:

};

Life-cycle

A CJOS kernel interacts with a module through the km_Module interface. When the kernel is initialized, it invokes the init() method of each installed module. When the kernel is started, it invokes the start() method of each installed kernel modules. Modules installed after the kernel is started are started as they are installed. When the kernel is stopped, it invokes the stop() method of each kernel module. When the kernel is destroyed, it invokes the destroy() method of each kernel module.

Kernel

A kernel itself is a kernel module. A kernel must implement km_Module interface. 
class km_Kernel : public km_Module {
  public:
    virtual void addModule( km_Module *v ) = 0;
    virtual km_Module *getModule( const char *v ) = 0;
    virtual void removeModule( const char *v ) = 0;

  protected:

  private:
};

extern "C" {
km_Kernel *km_getKernel();
}
Any C function or C++ method can invoke the km_getKernel() function to get the kernel. Since there can only be one kernel inside each machine code process, this method always returns the kernel.

Device

Devices are kernel modules. A device should install its interrupt handler(s) when its start() method is installed. The interrupt handler device might be a kernel module, too.

The kernel scheduler is a kernel module. It is expected to manage machine code processes and threads.

The root file subsystem is a kernel module. It is expected to provide kernel support for stream buffers and enable file subsystem plug-ins for all kinds of stream-based I/O.

The kernel console device is a kernel module. It is expected to implement the kernel's standard input stream and standard output stream.

To get a device, first get the kernel with km_getKernel() and then get the device by name with its getDevice() method.

Virtual machine

Virtual machines are kernel modules. A virtual machine might be the last kernel module started. By the time it is started, a kernel module environment is fully defined. There is a machine code scheduler, file subsystem, kernel console and everything else a virtual machine designer might need to implement a virtual machine.

Statically linked

Kernel modules are expected to be statically linked to a boot image. The init() method of the kernel itself starts adding other kernel modules. The entry point of the CJOS kernel is where the boot loader invokes the getKernel() method. 
static km_Kernel *kernel = NULL;

km_Kernel *getKernel() {
  if ( kernel == NULL ) {
    kernel = new NullKernel();

    kernel = new BasicKernel();
    kernel.init();
    kernel.start();
  }
  return kernel;
}
When the kernel is initialized, it initializes its kernel modules. 
void BasicKernel::init() {
  addModule( new BasicInterruptDevice() );
  addModule( new BasicVirtualMemoryManager() );
  addModule( new RootFileSystem() );
  addModule( new BasicConsoleDevice() );
  addModule( new BasicVirtualMachine();
}
When the kernel is started, it starts its kernel modules. 
void BasicKernel::start() {
  ModuleNode *pcurr = ptop;

  for (;;) {
    if ( pcurr == NULL ) {
      break;
    }
    pcurr->start();
    pcurr = pcurr->getNext();
  }
}
When the kernel is stopped, it stops its kernel modules. 
void BasicKernel::start() {
  ModuleNode *pcurr = ptop;

  for (;;) {
    if ( pcurr == NULL ) {
      break;
    }
    pcurr->stop();
    pcurr = pcurr->getNext();
  }
}
When the kernel is destroyed, it destroys its kernel modules. 
void BasicKernel::destroy() {
  ModuleNode *pcurr = ptop;

  for (;;) {
    if ( pcurr == NULL ) {
      break;
    }
    pcurr->destroy();
    pcurr = pcurr->getNext();
  }
}
