Article contributed by GilbertHerschberger (4 August 2001).

Warning: The classes in jasmin/stub0 are not intented for use at run-time. They are intended only for design-time research and academic study.

Introduction

At least, I have some control over the pre-processor in Jasmin, a Java assembler. I am starting a stub project to assemble the core packages using the Java assembler language.

I am working toward a single-source edition of the core packages. By running core packages through a pre-processor, one source file will be capable of compiling multiple class files.

  • JDK10 - This property will be defined only when compiling for the Java 1 platform, implemented by Sun Microsystems' JDK 1.02.
  • JDK11 - This property will be defined only when compiling for the Java 2 platform, implemented by JDK 1.1.x.
  • JDK12 - This property will be defined only when compiling for the Java 2 platform, implemented by JDK 1.2.x.
  • JDK13 - This property will be defined only when compiling for the Java 2 platform, as implemented by JDK 1.3.x.
  • JDK14 - This property will be defined only when compiling for the Java 2 platform, as implemented by JDK 1.4.x.

Potential uses

Much of this work is for the purpose of platform analysis. Every public class will exist. Every public field will exist, but may not contain exactly the right default value. Every public and protected method will exist, but may not have a Code attribute.

A single-source collection will be able to compile (eventually) the core packages for Java technologies. Researchers can use the assembled class files to analyse the differences between platforms.

This collection will be an excellent resource for programmers using the Java assembler. There will be plenty of examples of the assembler code. Examples will demonstrate the creation of interfaces and classes.

Virtual machine designers can use the single-source collection to highlight the differences between platforms. This is the beginning of an effort to build a version-less virtual machine.

This collection may also serve those who need to decode a class file. A Java assembler enables you to be far more specific about the content of a class file when building both good and bad examples.

Not javac

I have stopped trying to use the javac tool with the core packages. The javac tool is oversimplified when it uses one classpath for its own executable code and the target application. With the Java programming language, I have little control over its pre-processor.
  • I am unable to redefine methods within the java/lang package, such as java/lang/Object.
