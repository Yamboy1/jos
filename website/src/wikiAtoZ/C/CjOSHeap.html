CjOSPages; VMKit

Article contributed by GilbertHerschberger (30 November 2000).

Introduction

A heap is where arrays, strings, classes and other Java objects are stored. A heap can be developed into a VMKit component.

Handle

In a classic design, each object on the heap is given a handle. As part of the CJOS project, such a heap will be developed as an example.

typedef union {
  cuint32 id;
  struct {
	 cuint16 hid;
	 cuint16 oid;
  } a;
} gc_HANDLE;

Heap API

In C, the heap API includes a method to open an object, update an object and close an object.

gc_HANDLE gc_openHeap( cuint32 pid );
gc_closeHeap( gc_HANDLE h );

gc_HANDLE gc_openObject( gc_HANDLE h, cuint16 count );
gc_getInt( gc_HANDLE o, cuint16 index );
gc_setInt( gc_HANDLE o, cuint16 index, cuint32 v );
gc_closeObject( gc_HANDLE o );

Both the heap component and the garbage collection component are replace-able. Eventually, a heap must work in concert with a garbage collection component. The heap and garbage collection component are separate components within a VMKit. While the heap stores objects, a garbage collection component examines all objects on a heap to determine if they should be finalize'd and free'd.

Heap ID

For a design with multiple heaps, the heap ID selects a heap. One bytecode process owns a heap. A bytecode process can open more than one heap. Heaps opened with a process ID equal to one (1) are owned by the system process; zero (0) is not a valid heap ID.

A heap is usually closed by the system process. When a heap is closed, all of its objects are destroyed; none of the objects are garbage collected.

Object ID

For each heap, an object ID begins with 1. Zero (0) is not a valid object ID. When an object is closed, a heap can re-use the object ID. An object is usually closed by a garbage collection component.

Arrays

It might be necessary to expand the heap API to include arrays.

gc_HANDLE gc_createByteArray( gc_HANDLE h, cuint16 count );
gc_HANDLE gc_createCharArray( gc_HANDLE h, cuint16 count );
gc_HANDLE gc_createShortArray( gc_HANDLE h, cuint16 count );
gc_HANDLE gc_createIntArray( gc_HANDLE h, cuint16 count );
gc_HANDLE gc_createLongArray( gc_HANDLE h, cuint16 count );
gc_HANDLE gc_createFloatArray( gc_HANDLE h, cuint16 count );
gc_HANDLE gc_createDoubleArray( gc_HANDLE h, cuint16 count );

Note: This API is expected to grow and mature as heaps are constructed and statically linked to a virtual machine.
