Introduction

The CJOS Project is a sister project of the JOS Project. It is open source and free license software. It is consistently distributed under GPL/LGPL. It may be one of the most comprehensive collections of a Java-based operating system to date. It covers both C, C++ and Java source code.

Multiple platform

Unlike some other Java-based operating systems, CJOS is multi-platform. Keep your current operating system to immediately gain most of the benefits of this software. CJOS runs on Linux(r). CJOS runs on Windows(tm). CJOS runs on OS/400. When a kernel/virtual machine are available for the JOS platform, CJOS is expected to run on JOS.

Multiple architecture

Unlike some other Java-based operating systems, CJOS is multi-architecture. For maximum compatibility and smallest possible size, it uses the Java 0 Platform made popular by Sun Microsystems' JDK 1.02. It runs as an applet in a Java-enabled HTML browser. It runs as a servlet in a Java-enabled HTTP server. It runs as a standard Java application. The eric tool enables you to integrate CJOS into your Microsoft Windows(tm) operating system. With your help, a corresponding eric tool will be available for Linux(r).

New mechanism

The CJOS Project enables us to use a new mechanism to distribute JOS-related products. We are planning to use the SourceForge website to manage its 400,000+ lines of code. The definition of most products and utilities remain the same.

Focus

The current focus of this project is the consistent distribution of open source code under the GPL/LGPL software license. When complete, its source code will be uploaded to the CVS service on SourceForge. It will be a milestone in the development of a Java-based operating system to have C, C++ and Java organized for concurrent development in a Internet-based CVS service.

After that, the continuous improvement of JOS-related products will become the focus.

Ready-to-run

All source code uploaded to the CJOS Project must be ready to run. If it won't compile cleanly or it has not been tested, it won't be uploaded.

At first, C and C++ source code will be developed on Windows(tm) and ported to Linux(r). There will be a clear separation of kernel, standard C library and virtual machine. A standard C library will be developed inside its own module (stdlib).
