Article contributed by GilbertHerschberger (2 June 2003).

Introduction

This article provides more information about mapping an abstract machine to the Java Native Interface (JNI) model.

Note: While JNI supports both C and C++, CjAM only supports C++.

Review

  1. The JNI_GetDefaultJavaVMInitArgs() function fills the initial arguments structure in a VM-specific way.
  2. The JNI_CreateJVM() function creates a virtual machine and then returns JavaVM, a pointer to a JNIInvokeInterface structure.
  3. The The JNI_GetCreatedJavaVMs() function returns a pointer for virtual machines that have already been created.

Domain

A "domain" is similar to a virtual machine, or JNIInvokeInterface structure. The first call to JNI_CreateJVM() should return a primary domain. Subsequent calls should create secondary domains. A primary domain has special priviledges. It can, for example, create and access system objects.

Likely, your primary domain depends on a specific version of a virtual machine. Your primary domain components may require a specific version. In this case, the first call to JNI_CreateJVM() will fail if a wrong version is requested.

Arguments

The JavaVMInitArgs structure is used to pass arguments to a virtual machine.

Optionally, arguments can be read from a digital resource. Create an argument structure and save it as a resource. Invoking the JNI_GetDefaultJavaVMInitArgs() function sets default arguments based on the version property and then, when used, sets properties from a resource.

Version

A version must be part of the JavaVMInitArgs structure. Your code must set the version property. This is how a specific version is selected in a version-less virtual machine. When a virtual machine supports more than one version of the Java platform, a version must be selected before invoking the JNI_CreateJVM() function.

Inside the JNI_CreateJVM() function, different defaults may be provided for each version. A pointer to a specific domain is part of the JNIInvokeInterface.

The JNI_CreateJVM() function is critical. It selects components from the VMKit. It assembles them to create a virtual machine. When more than one component is available, a specific component must be selected here.

Version-less

The VMKit provides reusable components for building your own virtual machine. Most of these components are identical for many versions of the Java platform; some are platform-specific. In order to support multiple platforms, a platform-specific component must be constructed.

In turn, the JNI_CreateJVM() function might depend on a virtual machine factory component. In your custom implementation of this function, you can use an off-the-shelf virtual machine factory. The default virtual machine factory creates a version-specific virtual machine.
