Article contributed by GilbertHerschberger (19 November 2001).

Introduction

This article explains how to compile the Java source code for JJOS using the BulkMake utility. The BulkMake utility compiles all Java source code and archives all class files in a directory tree. There is no need to reconfigure this utility when you add a new source code file.

Procedure

The following procedure illustrates how to compile Java source code for JJOS using the BulkMake utility.
  1. Switch to the module/common/bytecode directory and start the BulkMake utility. It is important for the current directory to be the default package.
  2. Click the Compile button. The Compiler form is displayed.
  3. Click the Run button. BulkMake invokes the PackageCompiler program, which searches for .java files starting with the current directory. In turn, the PackageCompiler program invokes the javac tool as needed for each directory (or package). If a .class file is older than its corresponding .java file, the package is compiled; one of its classes is out-of-date. Otherwise, nothing is compiled.
  4. Click the Archive button. The Archive form is displayed.
  5. Click the Run button. BulkMake invokes the PackageArchive program, which searches for .java and .class files starting with the current directory. In turn, the PackageArchive program invokes the jar tool twice. It archives the Java source code (.java) into jjos-base-src.zip. And then, it archives the class files (.class) into jjos-base.jar.
  6. Close the BulkMake utility.
  7. Copy jjos-base.jar to arch/host/bytecode/lib.
  8. Switch to the arch/host/bytecode directory and start the BulkMake utility.
  9. Click Compile, then Run.
  10. Click Archive, then Run.
  11. Close the BulkMake utility.
  12. Copy jjos-base.jar to arch/i386/bytecode/lib.
  13. Switch to the arch/i386/bytecode directory and start the BulkMake utility.
  14. Click Compile, then Run.
  15. Click Archive, then Run.
  16. Close the BulkMake utility.

Improvements

When invoked in the common/bytecode directory, the make install command should install the jjos-base.jar archive to the arch/host/bytecode/lib and arch/i386/bytecode/lib directories.

Summary

You have accomplished this skill if you have a corresponding .class file for each .java file in the common/bytecode directory tree and a jjos-base.jar. You'll also have a jjos-host.jar file in arch/host/bytecode and a jjos-i386.jar file in arch/i386/bytecode. The jjos-host.jar has the same name regardless of which version of JJOS you're building from source code.
