About this document

This document was contributed by GilbertHerschberger (23 July 2004). It explains the use of the RPM technology within the CjOS Project. It describes the process that we use to build an RPM and install our product on Linux using an RPM.

As we have migrated away from Microsoft Windows toward Linux, we have increased both our facination and our experience with this technology. To be sure, it isn't the most self-explanatory or eye-popping technology available; but it has proven to be very predictable, very reliable for those who can use it.

Summary

Is it possible to distribute the CjOS software using RPM? Sure it is. This article is the beginning of an effort to learn more about RPM and using it to distribute Java-based applications, libraries and utitilies.

Section A - Our Build Process

Introduction

This section contains more information about our build process. It explains the design of our build process as it is reflected in release 1c-5.2.

Architecture

To indicate that our product is not dependent on a specific architecture, we use "noarch". Our product depends upon a Java Virtual Machine. Typically, a Java Virtual Machine is platform dependent so that our application does not depend upon a specific real-machine architecture.

Target

The target of the RPM build process is an RPM-compatible archive. In turn, the RPM-compatible archive may depend upon a ZIP-compatible archive, such as a jar or war file. In turn, the ZIP-compatible archive depends upon the original Java source code.

Phases

We have separated the build process into two phases. Our first Ant script invokes the rpmbuild command, which invokes the second Ant script.
  1. Our first Ant-compatible build script collects the digital resources required by the rpmbuild command. It builds a gz-compatible archive and writes it to the expected rpmbuild/product directory. This step does not invoke the Java compiler or assemble classes into a jar. In turn, it invokes the rpmbuild command with the rpmbuild-compatible specification file called project.spec.
  2. A separate Ant-compatible script is used by rpmbuild to build the target classes, jars and wars. The build target builds classes from the original Java source code. It assembles classes into jars. It assembles HTML, JSP and jars into wars. The install target copies the built files into the target directory structure.

Building on Microsoft Windows

Of course, the rpmbuild command is not readily available on the Microsoft Windows platform. To build the target classes, jars and wars on Windows, the first Ant script directly invokes the second Ant script.
