Introduction

The main() function of decaf host build illustrates the issue of a kernel factory. There can be only one kernel. This raises other questions. How should it be instanciated? Does it need a constructor? When should its constructor be invoked? This article documents some of the thinking that has gone into the CJOS kernel factory.

mc_Kernel class

The mc_Kernel interface is written into a C++ header. This class provides pure virtual methods for kernel-related functions.

Each platform must define a platform-specific implementation of mc_Kernel. The name of the platform-specific implementation is never directly exposed to other subsystems.

Since there can only be one kernel, the mc_getKernel() function does whatever it takes to instanciate a platform-specific implementation and return it.

extern "C" mc_Kernel *mc_getKernel();

Example 1

Here is an example of a platform-specific implementation of mc_getKernel().

// kernel.cpp
#include <cjos.h>
#include <basickernel.h>

static mc_Kernel *kernel = NULL;

mc_Kernel *mc_getKernel() {
  if ( kernel == NULL ) {
	 kernel = new BasicKernel();
  }
  return kernel;
}

A platform-specific kernel is created automatically the first time any machine code invokes the mc_getKernel() method. It is not necessary to create a kernel explicitly in the main() function of a cjos tool. The implementation of mc_Kernel can be completely encapsulated in a subsystem and instanciated as needed.

If required, it is up to each platform-specific kernel to work out how additional parameters are passed to the kernel. The kernel does not invoke other subsystems. Rather, other subsystems use the kernel.

Example 2

Here is an example of platform-independent code which uses a kernel.

void Example::run() {
  mc_Kernel *k = mc_getKernel();
  k->kprintf( "Hello, world!" );
}

Example 3

Here is an example of a Java program with machine code access to a kernel.

public class Example {
  public void run() {
	 Kernel k = new Kernel();
	 k.kprintf( "Hello, world!" );
  }
}

Since there is only one kernel, the Java object only represents the kernel in machine code. The Bytecode Native Interface is used to invoke kernel methods.

public class BasicKernel
	 implements Kernel {
  public BasicKernel() {
  }
  public native int kprintf( String v );
}
