CjOSPages; SmartAPIPages

Article contributed by GilbertHerschberger (27 January 2002).

Introduction

The MiniServer(tm) product is part of the CjOS Project ( CjOSProject ). It provides a visual interface to a minimal Java Servlet Environment. In my own work, I write a lot of Java Servlets and have come to depend on MiniServer(tm).

Here is how the CjOS Project has had an impact on MiniServer(tm). The names of jars distributed for Smart API have change. The following list illustrates new names for these jars:

  • cjos-smartapi1a.jar (was smartapi1a.jar)
  • cjos-smartapi1b.jar (was smartapi1b.jar)
  • cjos-smartapi2a.jar (was smartapi2a.jar)
  • cjos-smartapiutil1.jar (was smartapiutil1.jar)
The next release of MiniServer(tm), release 4, contains only jars with the new names.

Packages have been migrated from the com.realignsoft namespace to the org.jos or gchii namespace. This means that the packages inside jars with the cjos prefix do not use any class in the com.realignsoft namespace. The following packages illustrate new names for packages:

  • org.jos.smartapi1a (was com.realignsoft.smartapi1a)
  • org.jos.smartapi1b (was com.realignsoft.smartapi1b)
