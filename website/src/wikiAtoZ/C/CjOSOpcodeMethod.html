Introduction

The invocation of opcode methods are critical to the design of a virtual machine.

Strongly typed

An opcode method should be strongly typed. In C/C++, it should have a typedef, like this:

typedef void (TOpcodeMethod)( Thread *t );

Opcode methods act upon a thread. A thread object is passed to each opcode method. In this design, opcode methods are not part of a thread class.

extern TOpcodeMethod mc_iadd;
extern TOpcodeMethod mc_isub;
extern TOpcodeMethod mc_imul;
extern TOpcodeMethod mc_idiv;
extern TOpcodeMethod mc_irem;
:

Opcode method array

Opcodes are numbered 0-255 and 256-511. Opcodes are 0-255 are narrow. Opcodes from 256-511 are wide. An array can contain pointers to opcode methods, like this:

  TOpcodeMethod *list = new TOpcodeMethod *[ 512 ];

Each opcode method is assigned to the array.

list[ Opcode.IADD ] = mc_iadd;
list[ Opcode.ISUB ] = mc_isub;
list[ Opcode.IMUL ] = mc_imul;
list[ Opcode.IDIV ] = mc_idiv;
list[ Opcode.IREM ] = mc_irem;
:

When given a thread, a dispatch method can invoke any opcode method, like this:

  void example( Thread *t ) {
	 Opcode o = t->getOpcode();
	 OpcodeMethod *m = list[ o.getID() ];
	 m( t );
  }
