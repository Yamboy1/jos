Introduction

This is the goal of the Cj operating system (CjOS).

To provide a long-lasting, robust, easy-to-deploy platform for pure bytecode applications, applets, programs and servlets.

Machine code vs. bytecode

This platform will use both machine code and bytecode. At first, it will store machine code for its kernel and kernel modules in a single, statically linked executable image. It may dynamically load kernel modules someday. It will always store bytecode in a standard, platform-independent class file format. On this platform, each machine code API will be reflected in bytecode API.

The C and C++ programming languages will be used. Machine code will be used for:

  • one or more kernels, including real memory management, virtual memory management, kernel bytecode cache, machine code processes, machine code threads, and task scheduler
  • one or more virtual machines;
  • low level device drivers;
  • transitional tools, such as java and javac.

The Jasmin, Java and other programming languages will be used. Bytecode will be used for:

  • everything else.

Non-goals

The following items are not the express goals of this operating system:

  • use the Java programming language for system programming;
  • create interrupt handlers in bytecode;
  • exclude a standard C class library;
  • Just-In-Time compiling (JIT)
  • Ahead-Of-Time compiling (AOT)
