Introduction

A CVS service is typically a server-side program, which runs 24 hours a day, seven days a week. It is an important part of the CVS network protocol.

A CVS service stores computer files in a central location. Like an FTP service, anyone can download files and contributors can upload files. Unlike an FTP service, a CVS service can store multiple versions of a file, along with a complete revision history.

To download or upload files, you must install CVSClient software. To download files, everyone shared the anonymous account, where the password is anonymous. To upload files, you need a personal account with your own password.

On sourceforge.net

A multi-project CVS service is running on SourceForge. It is maintained by a full support staff. Part of the service has been set aside for the JOS Project.
