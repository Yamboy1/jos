Introduction

Package names have changed. Our migration project has moved many packages out of the com.realignsoft namespace. It is no longer appropriate to use that namespace for our open source project. While the latest source code is available through CVS, it is not yet reflected correctly in the JOS Technical Edition. CJOS CVS on SourceForge ( CJOSCVSOnSourceForge ) contains Java source code related to the CJOS Project.

The current plan includes rebuilding every WebSource'd HTML page for every product. WebSource will be reconfigured on a product-by-product basis to read from a product's build directory. Each product has a build directory. See also CjOSAndBuildModule.

No problem

Recently, I added the jpkg module to CVS. Once built, it contains a few hundred ERIC-compatible package files.

When I said there was a problem, I did not understand why WinCVS took so long to start. It turns out I only needed to be more patient.

  • Coincidentally, WinCVS has suddently stopped working on my primary Microsoft Windows 95 computer. It displays a splash screen and then, surprisingly, stops responding to OS messages. Its main window is no longer displayed.
