DownloadBootDisk

Article contributed by GilbertHerschberger (6 July 2000).

Introduction

An operating system must be installed before it is used, right? If a machine has no operating system, an operating system must be installed so that platform-specific applications can run on the hardware.

Creating a boot disk

A boot disk for an IBM PC-compatible machine is needed. How is a boot disk distributed?

  • Manufacture

    Traditionally, a physical boot disk is distributed to a customer. A manufacturer takes a 1.44MB diskette, formats it and adds a boot disk image. When you insert the boot disk and restart your PC, your PC reads a new operating system from a boot disk.

  • Boot disk image

    A boot disk image can be delivered to a PC by distributing a boot disk image. You download the boot disk image. You take your own 1.44MB diskette, format it and add the boot disk image. Like a manufactured boot disk, you insert it and restart your PC. Your PC boots with a new operating system.

    This works as long as your have the right equipment to (a) download a boot disk image, (b) format a 1.44MB floppy diskette and (c) add the boot disk image to the floppy. When your old operating system is incompatible with the new, it cannot format the 1.44MB floppy with the right format. For MS-DOS, the rawwrite program makes it possible to write a boot disk with any boot disk image.

    This is practical only if there are a few boot disk images. Which boot disk image should you use? Sometimes the image depends on your optional equipment, such as a network card.

  • Source code

    Sometimes a boot disk image is distributed in source code. You download the source code and compile it into a boot disk image. You take your own 1.44MB diskette, format it and add the boot disk image.

    This works as long as you have the right equipment to (a) download the source code and all of its dependencies, (b) compile the source code into a boot disk image, (c) format a 1.44MB floppy diskette and (d) add the boot disk image to the floppy. When your old operating system does not support the right compiler, it is difficult, if not impossible, to build a boot disk image from source code.

  • Application

    Sometimes a boot disk image is distributed through an application. An application might make it easier (less technical) for a customer to create a boot disk. A browser application, for example, would enable a customer with one machine to download one boot disk image. A single GUI application would enable a customer with many different machines select the right boot disk image for each machine.

About Boot Disk application

An interesting thing happens when customers have already installed a Java Virtual Machine on their machine.

  1. Create a Java application

    A technician can do the highly technical work. A technician downloads the source code and all of its dependencies. A technician compiles the source code in to multiple boot disk images, covering frequently used optional equipment, such as network cards. A technician creates a boot disk application and distributes it across the network. As a Java application, it could reach the largest possible audience.

    Through the combined efforts of many members of the JOS Project, the latest release of Boot Disk is now available for download.

  2. Download a Java application

    Customers download a boot disk application. It is installed on another operating system, possibly on another machine.

  3. Run a Java application

    Using point-and-click, a customer can select the boot disk image that's right for their optional equipment. By pressing a button, the application exports (copies) the selected boot disk image to a well-known file name. A customer formats a 1.44MB floppy diskette and adds the boot disk image to the floppy.

    This works if a customer can (a) download a Java application, (b) install a Java Virtual Machine, (c) format a floppy and (d) add a boot disk image to the floppy.

    Future

    By adding platform-specific plug-ins, a boot disk application can provide one-stop shopping for any supported platform. A specific plug-in is selected based on (a) configuration or (b) system properties.

    • Format a floppy

      A platform-specific plug-in would know how to format a floppy disk. A customer can press a button to format a floppy instead of looking up the exact command line syntax and parameters.

    • Add a boot disk image

      A platform-specific plug-in would know how to add a boot disk image. A customer can press a button to add their selected boot disk image to a floppy.

    • Restart a machine

      A platform-specific plug-in would know how to restart a computer. Likely, a customer wants to test the boot disk immediately after it has been created. Each operating system provides a way to restart a computer.

    One click?

    By combining these features, a boot disk application might provide a one-click method for formatting a floppy, adding a boot disk image and restart a machine.

    A boot disk application should be polite. It might confirm with a dialog box the format of a floppy and/or restart of a computer.

    For Linux

    A plug-in for the Linux operating system is possible.

    • Format a floppy

      The exec() method of java.lang.Runtime can invoke the Linux program that formats a floppy disk.

      run:os:mkfs.ext2 /dev/fd0
run:os:fdformat /dev/fd0

    • Add a boot disk image

      A boot disk image can be written directly to a floppy disk with a FileOutputStream. There is no need to save an intermediate file.

    • Restart a machine

      The exec() method of java.lang.Runtime can invoke the Linux program that restarts a machine.

      run:os:shutdown -r now

    For Windows

    A plug-in for Windows operating systems are possible.

    • Format a floppy

      The exec() method of java.lang.Runtime can invoke a Windows program that formats a floppy disk. The rawwrite program does not require a floppy disk to be formatted.

      run:os:format A:

    • Add a boot disk image

      The rawrite tool can write a boot disk image to a floppy. An intermediate file must be saved to a local file, such as BOOTDISK.DAT. Here is the syntax for this command:

      run:os:rawrite BOOTDISK.DAT A:

    • Restart a machine

      The exec() method of java.lang.Runtime can invoke a Windows program that restarts a machine. What is the name of that program?

      run:os:???

    Summary

    A boot disk application is a step closer to what most customers want. Their highest priority is to use an operating system that someone else has built.
