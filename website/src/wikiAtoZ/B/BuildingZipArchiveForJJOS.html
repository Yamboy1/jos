Introduction

Once all of the .class files have been built, they must be archived into jjos.zip, a ZIP-compatible archive. This can be performed on Linux or Windows.

On Linux

  1. cd common/bytecode
  2. make The make script in this directory copies classes.zip to jjos.zip and then invokes the zip command to add custom class files. This may replace class files which have already been archives. Like classes.zip, jjos.zip is an uncompressed ZIP-compatible archive.

On Windows

Here are your options on Microsoft Windows.
  • Use the jar tool from a Java Development Kit. Remember to use an extra command line option (-0) to specify archive without compression.
  • Use your favorite ZIP-file program, such as WinZip or NetZip, to add class files to jjos.zip. Remember to choose archive without compression.
  • The BulkMake utility does not support uncompressed ZIP-compatible archives at this time.

Combination of Linux and Windows

Personally, I use a combination of the two. I use BulkMake to build all of the class files efficiently. I copy compiled class files to my JJOS development machine and trust the make script to build the jjos.zip archive.
