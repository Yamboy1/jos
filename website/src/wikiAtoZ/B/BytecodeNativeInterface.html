PlatformAPIPages; JavaNativeInterface; JVMSpecificClasses; JVMSpecificObject

Article contributed by GilbertHerschberger (24 November 1999).

Introduction

It has been a delight and a challenge to think about the potential of a mostly-bytecode JVM. It has been a delight because a mostly-bytecode JVM forces us to take another look at the native interface. It has been a challenge because we're still looking for a simple solution to this complex problem.

As a JVM implements the Java Platform, JVM-specific classes are required. Platform-specific patches are inherent in every platform-independent architecture. JVM-specific classes provide support for many JVM-specific mechanisms that exist "below" the Java Platform.

Other JVMs

Other JVMs are classic JVMs. They run on top of a foreign operating system. They build their JVM-specific classes in machine code. Sun Microsystems created a hack for Java 0 to invoke native methods written in machine code. Netscape couldn't use Sun's design in Navigator because Sun's hack is platform-dependent and Navigator is not. Instead Netscape started developing we now know as the Java Native Interface (or JNI). Sun abandoned their hack in favor of Netscape's and made JNI part of the Java 1 Platform. They require all JVMs to implement JNI.

JOS JVM

Like all JVMs, our JVM must implement JVM-specific classes. Unlike other JVMs, our JVM doesn't run on top of a foreign operating system. It doesn't have pre-existing mechanisms for shared libraries. A mostly-bytecode platform means that our "native" code is bytecode.

What does this mean?

It means that a mostly-bytecode JVM cannot support Sun's JavaNativeInterface because bytecode is the "native" code. It means that we need some other native interface to JVM-specific classes.

Bytecode Native Interface

Unlike a machine code native interface, a bytecode native interface uses JVM-specific classes written in bytecode to perform JVM-specific functions. The bytecode native interface for JOS is a work-in-progress. It is being discussed on the JOS-General mailing list at jos-general@lists.sourceforge.net

For more information on the current distribution, see also AboutBCNI.
