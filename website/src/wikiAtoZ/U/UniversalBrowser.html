AboutUniversalBrowser; SmartAPIPages

Article contributed by GilbertHerschberger (26 November 1999)

Introduction

The Universal Browser is distributed as part of the SmartAPI. It demonstrates what a browser can do. It highlights some of the things a Smart API can do.

Can your browser do this?

The Universal Browser is a browser of browsers. It includes the following browsers today:

  • Applet Browser
  • Connection Browser
  • Desktop Browser
  • Plain Text Browser
  • Program Browser
  • Property Browser
  • Token Browser

Only recently, Universal Browser brought all of these other browsers together under a common umbrella. It incorporates the classic shell, with its traditional command line interface. It incorporates the classic browser, with its location field and uniform resource locator (URL). It incorporates the classic desktop, with its icons, shortcuts and preferences.

Compared to other browser/desktop/shell packages, the Universal Browser might feel a little strange. It breaks a standing rule of separate windows for separate tools. It provides one-stop shopping for all your operating needs. It provides a uniform mechanism for managing all of your digital resources.

The Universal Browser was built from the bottom up. Each brower in this collection is developed independently. It is independently designed, tested and refined.

Tradition is a good thing. The Universal Browser is backward compatible. It runs on Java 0, 1 and 2 platforms. (It even runs inside a Java-enabled browser, like Microsoft Internet Explorer or Netscape Navigator). It runs on your existing operating system. It enables you to use what your comfortable with. It enables you to keep on doing what you have grown accustomed to. If you prefer separate windows for separate tools, each of these browsers are stand-alone.
