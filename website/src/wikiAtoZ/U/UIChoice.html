Introduction

Choice is a fundamental part of making a computer personal. When all of the important choices are made by someone else, a personal computer isn't personal. A personal computer means that everything is my choice.
Long road ahead

I think we have a long way to go. Computers aren't personal yet. I am still searching for a real personal computer. I haven't seen a real personal computer yet. Hardware vendors want to eliminate 80% of my choices by providing a preferred user interface for the machine. It is the user interface they choose. It is the interface they prefer, not me.
Buy once, run anywhere

There's nothing really wrong with the software I already have. Why should I buy all new software just because I want the power of some new machine? I want to run my choice of software on my choice of hardware. Yes, I want to mix and match. And I want it all to work in every combination. I want to buy software once that runs on any machine, anywhere.
Is it possible?

I am inspired by Sun Microsystems' idea of Write Once, Run Anywhere. Someday, I will be able to buy software written in Java and use it on all these different platforms. I get what I really want: I want choice. I get to choose what is best for me.

Suppose this is possible. What is the next step? The idea of "write once" could be taken to the next level. It might be possible to create a hardware-independent operating system. Hardware and software issues must be completely separate. The same OS API would be available on every machine, everywhere.

When I buy hardware, I want to buy it based on the merits of hardware and hardware alone. I do not want to buy IBM PC hardware in order to benefit from the latest advances in user interface design. I do not want to buy the Apple iMac to benefit from the Apple OS and software based on it.

I am interested in a personal computer for one reason and one reason only: personal choice. The process of personalization includes making the computer work for me. It is supposed to help me get my work done. I customize it. I tweak it. I configure it. I program it. I mold it into Gilbert's personal computer.

This is my personal computer because, once customized, it doesn't work for anyone but me. It helps me get my work done. Sorry, you can't use my personal computer to get your work done.
Wish List

In an article about UIPurpose, I said that the first defect in our search for the "best" user interface is thinking that there is only one interface that's best for everything. I am aware of the problem and even I have been affected by thinking there must be only one "best" interface.

After thinking about it more this weekend (March 20-21, 1999), I discovered a simple way to think about choices in personal computers. You should always be able to assemble your own collection of customized desktops. The customized desktops should run independent of hardware.

  1. I don't want one desktop. I want one desktop for each kind of thing that I do. Like the shell, I want my desktop to manage the "current" directory and launch all applications from there.
  2. I want many desktops at the same time. I want to experiment with many different desktops to find the one I like best. I want to see different desktops running side by side. I want desktops that have different behavior and different levels of difficulty.
  3. I don't want a dead-end desktop. I want to be able to extend it, enhance it, and customize it with programming, way beyond what the original designer might have had in mind.
  4. I want a stable, rugged, reliable, high quality, compatible desktop framework. I want to be able to send my favorite desktop to a friend, as an e-mail attachment. I want it to work perfectly on every computer.

With extreme reuse and object-oriented thinking, that's not too much to ask. With a little work, the HTML browser could become an alternative desktop.
