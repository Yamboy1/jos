UIChoice; UserInterfaceDefinitions

Article contributed by GilbertHerschberger.

So you want to write your own operating system and throw in the best user interface ever write while your at it. You must ask yourself a serious question:

What is the true purpose of a user interface?

Everyone entering this discussion should understand that the experts have been arguing about what is the "best" user interface for fifty years. They can't agree.

The first defect in the argument for the "best" user interface is the defect in thinking that there must be only one. Whose computer is it anyway? I think I can decide what's best for each of my computers. I have put a character-based interface on two computers and a graphical user interface on one. That's what I need to get my work done. I don't want my operating system to get in the way by supporting one way, not the other.

Different work requires a different user interface. History has shown that a graphic user interface is only good for a subset of things people do. A character based interface is only good for a subset, too. While a command line interface is good for somethings, a desktop interface is good for others.

The command line interface is not dead. Recently, the popular media reintroduced the command line interface as "innovative" and "easy enough for children to use". Sometimes people don't recognise an interface for what it is. An HTML browser brings together a command line and graphical user interface. The location field of Netscape Navigator and Microsoft Internet Explorer is a command line interface. When you type in the location and press the Enter key, an HTML page appears. This is a command line, pure and simple. And, a command line is a very good design decision for this mechanism. A command line should be used when a set of commands is large or open ended. A command line has nothing to do with being an expert or novice, as the HTML browser has shown.

User interface has been combined in all of the following ways. These are found in commercial products sold today (the year is 1999). While the character-based interface is often associated with a command line interface, the graphical user interface is often associated with the desktop just because someone did it that way once or twice. In addition, the mouse is closely associated with the graphical user interface in spite of the fact that a mouse has never been exclusively graphical.

Command Line Desktop

The purpose of a user interface is to interact with processes running on the computer. The goodness or badness of the user interface is tied to interacting appropriately with the processes running on a computer. An open OS for the IBM PC must provide at least a crude character-based interface (either command line or desktop) because the IBM PC ROM boots into a character-based interface. The graphical user interface should be left open to the shell (either command line or desktop) the user selects based on the work s(he) wants to do.

UserInterfaceDefinitions should be expanded or refined to embrace all of the possibilities.
shell
A kind of application program with the primary responsibility to launch other application programs. Any application program can act as a shell. A desktop is a kind of shell. A browser is a kind of shell. A shell is called a shell because it is a hollow wrapper around other programs. In a single-process OS, a shell is suspended while another application is running. In a multiple process OS, a shell is suspended while another appication is running, unless a separate process is explicitely requested. Because a shell is a kind of application program, a shell can launch another instance of itself or any other shell.
shell framework
A framework (or API) for enabling any application program to act like a shell. This framework refines the kernel API to support different user-interfaces.
The shell, as defined in most traditional operating systems, is not a framework at all. It is a tangible piece of software. Each account should be configured with a default shell. This means a program and not a framework. If we have a shell API or framework, we should call it either shell API or shell framework.

Most traditional operating systems are not object-oriented and offer no shell API or framework. They offer a general purpose API someone might use to write their own shell. A Linux shell is just another Linux program. The Mindspring Desktop for Microsoft Windows is just another Windows program.

While we might need a common class library for shells, we also need implementations built upon that class library. In general, all these implementations are called shells.

In the relationship between a shell and its user-interface, a shell has-a user-interface. No shell is-a user-interface. A shell may have a hardware user-interface, a command-line user-interface or a iconic user-interface. An iconic interface uses icons. A console may be character based or graphical. Yes, you can have a mouse-driven iconic and character-based shell. Ensemble's CommandCenter is a good example. There have been many, many others. -- GilbertHerschberger (26 July 1999).
