AdvocacyPages

Article contributed by GilbertHerschberger (23 August 2001).

Introduction

I encourage you to start a sister project whenever an existing sister project doesn't provide what you want. I encourage you to do it sooner rather than later.

Benefits

By starting a sister project, you decide on the fundamental implementation details of your own Java-based operating system. You decide on at least the following four things:
  1. Software license
  2. Programming language
  3. Hardware platform
  4. Development environment and tools
You are the owner of the sister project. You can determine who else (if anyone) can commit changes to the source code.

By maintaining a sister project status with the JOS Project, you also have instant access to the following infrastructure:

  1. www.jos.org website
  2. jos-general mailing list
  3. JOS Wiki knowledge base
  4. Advice on setting up a project and maintaining CVS respository on SourceForge.
  5. Advice on choosing a software license.
A sister project should be started for any of the following reasons:
  • To distribute your code under a new software license.
  • Experiment with a new programming language.
  • Experiment with a new hardware platform.
  • Isolate a critical issue, such as multiple bytecode process, without contaminating an otherwise perfectly good virtual machine.
Since you can participate in multiple sister project simultaneously, you don't have to abandon one sister project to join another. You don't have to abandon the greater JOS Project because you're interested in a specific product.

Feedback

Each sister project should provide feedback to the JOS Project. Share the lessons learned.
  • Participate in the design of a common framework for all Java-based operating systems. A design for your sister project might work for all sister projects.
  • Develop new techniques for Java technologies. Search for a simple way to create multiple independent bytecode processes within a single virtual machine. Share your ideas with us; we will share ideas with you.
  • Come up to speed quickly on the fundamental issues of building a Java-based operating system. How does it compare to mature operating systems, such as Linux or Microsoft Windows? Find out what is critical and what's irrelvant.

Summary

Find out if there is already a sister project which might meet your needs. If not, I encourage you to start one.
