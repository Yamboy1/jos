MaintenancePages

Article contributed by GilbertHerschberger (23 August 2001).

Introduction

Maintaining articles can be more enjoyable by not attempting the impossible. 1500 articles cannot be reworked overnight, and should not be. They should be reworked with individual attention, slowly and carefully. We should adopt the following approach:
  • Maintain important articles first.
While it might be obvious to some, articles maintenance must have a priority. This is always true. It is more so when there are so many articles and so few writers.

An article is important if it has an effect on the day-to-day operation of the project. If we have changed the way we work, we should let others know about the change on both the jos-general mailing list and, here, on JOS Wiki.

A well-written historic article should require no maintenance at all. Its message is timeless. These articles are the least important to maintain.

Imported forum

Over time, there are fewer articles in the Imported forum. Eventually, there will be none. Articles should be maintained as time allows. This is not a high priority for this project, and should not be. As always, it is inappropriate for us to suspend work on a Java-based operating system just to maintain articles.

The fact is the software providing the JOS Wiki pages has changed. We're now using sfWiki. Most fortunately, this software provides backward-compatible rendering within the Imported forum. Old articles are still easy to read even though they do not use new wiki markup.

An article is important if it can be converted from the old wiki markup to the new. As it is maintained, it must be moved from the Imported forum to the Main forum. Changes cannot be saved to the Imported forum!

Summary

For me, maintaining JOS Wiki has been enjoyable and time-consuming. It has been enjoyable because I like to write. I learn more by becoming a teacher than remaining a student. I am forced to think about things more carefully and re-think things that I once assumed.

It has been time-consuming because reworking an article takes time. Some articles need frequent attention. The real world changes quickly, more quickly than any small group of volunteers can handle.
