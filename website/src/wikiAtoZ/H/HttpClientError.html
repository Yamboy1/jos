Introduction

This article demonstrates the difference between C-style enumeration in Java and object-oriented enumeration in Java.

About HttpClientError class

Instead of using a final integer to represent each client error, the following object-oriented enumeration represents each client error as a final object.

<FONT COLOR="#0000FF">public class</FONT> HttpClientError {
  <FONT COLOR="#0000FF">public static final</FONT> HttpClientError
	 BAD_REQUEST = new HttpClientError( <FONT COLOR="#007F00">400</FONT>, <FONT COLOR="#007F00">"Bad Request"</FONT> ),
	 UNAUTHORIZED = new HttpClientError( <FONT COLOR="#007F00">401</FONT>, <FONT COLOR="#007F00">"Unauthorized"</FONT> ),
	 PAYMENT_REQUIRED = new HttpClientError( <FONT COLOR="#007F00">402</FONT>, <FONT COLOR="#007F00">"Payment Required"</FONT> ),
	 FORBIDDEN = new HttpClientError( <FONT COLOR="#007F00">403</FONT>, <FONT COLOR="#007F00">"Forbidden"</FONT> ),
	 NOT_FOUND = new HttpClientError( <FONT COLOR="#007F00">404</FONT>, <FONT COLOR="#007F00">"Not Found"</FONT> ),
	 METHOD_NOT_ALLOWED = new HttpClientError( <FONT COLOR="#007F00">405</FONT>, <FONT COLOR="#007F00">"Method Not Allowed"</FONT> ),
	 NOT_ACCEPTABLE = new HttpClientError( <FONT COLOR="#007F00">406</FONT>, <FONT COLOR="#007F00">"Not Acceptable"</FONT> ),
	 PROXY_AUTHENTICATION_REQUIRED = new HttpClientError( <FONT COLOR="#007F00">407</FONT>,
		  <FONT COLOR="#007F00">"Proxy Authentication Required"</FONT> ),
	 REQUEST_TIMEOUT = new HttpClientError( <FONT COLOR="#007F00">408</FONT>, <FONT COLOR="#007F00">"Request Timeout"</FONT> ),
	 CONFLICT = new HttpClientError( <FONT COLOR="#007F00">409</FONT>, <FONT COLOR="#007F00">"Conflict"</FONT> ),
	 GONE = new HttpClientError( <FONT COLOR="#007F00">410</FONT>, <FONT COLOR="#007F00">"Gone"</FONT> ),
	 LENGTH_REQUIRED = new HttpClientError( <FONT COLOR="#007F00">411</FONT>, <FONT COLOR="#007F00">"Length Required"</FONT> ),
	 PRECONDITION_FAILED = new HttpClientError( <FONT COLOR="#007F00">412</FONT>,
		  <FONT COLOR="#007F00">"Precondition Failed"</FONT> ),
	 REQUEST_ENTITY_TOO_LARGE = new HttpClientError(  <FONT COLOR="#007F00">413</FONT>,
		  <FONT COLOR="#007F00">"Request Entity Too Large"</FONT> ),
	 REQUEST_URI_TOO_LONG = new HttpClientError(  <FONT COLOR="#007F00">414</FONT>,
		  <FONT COLOR="#007F00">"Request URI Too Long"</FONT> ),
	 UNSUPPORTED_MEDIA_TYPE = new HttpClientError( <FONT COLOR="#007F00">415</FONT>,
		  <FONT COLOR="#007F00">"Unsupported Media Type"</FONT> ),
	 REQUESTED_RANGE_NOT_SATISFIABLE = new HttpClientError( <FONT COLOR="#007F00">416</FONT>,
		  <FONT COLOR="#007F00">"Request Range Not Satisfiable"</FONT> ),
	 EXPECTATION_FAILED = new HttpClientError( <FONT COLOR="#007F00">417</FONT>,
		  <FONT COLOR="#007F00">"Expectation Failed"</FONT> ),
	 REAUTHENTICATION_REQUIRED = new HttpClientError( <FONT COLOR="#007F00">418</FONT>,
		  <FONT COLOR="#007F00">"Reauthentication Required"</FONT> ),
	 PROXY_REAUTHENTICATION_REQUIRED = new HttpClientError( <FONT COLOR="#007F00">419</FONT>,
		  <FONT COLOR="#007F00">"Proxy Reauthentication Required"</FONT> );

  private HttpClientError( int n, String v ) {
	 id = n;
	 message = v;
  }
  public int getID() {
	 return id;
  }
  public String getMessage() {
	 return message;
  }
  public String toString() {
	 return "" + getID() + " - " + getMessage();
  }
  private int id;
  private String message;
}

Example

The following example illustrates how to get the ID and message of a client error.

  public void example() {
	 HttpClientError e = HttpClientError.NOT_FOUND;
	 int n = <STRONG>e.getID()</STRONG>;
	 String v = <STRONG>e.getMessage()</STRONG>;
	 System.out.println( "" + n + " - " + v );
  }
