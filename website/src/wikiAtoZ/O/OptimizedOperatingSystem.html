Introduction

After establishing a purpose for our operating system, the next question is how to optimize it. Optimizing an operating system is about balance. It is too easy to go too far.

Our operating system should be distinctive. It shouldn't be a knock-off of some other system. It shouldn't have the same strengths and weaknesses of other systems. It should be optimized for something. What is it?

Born to run bytecode

JOS takes most of the virtual out of the Java virtual machine. Everything that can be taken out will be taken out.

JOS is optimized to run Java bytecode. It runs bytecode and nothing but bytecode. It is a streamlined, minimized, fully optimized Java virtual machine.

JOS takes out the baggage required to run everything else. It isn't going to provide any sort of compatibility box. People can't run Linux programs or Windows 95/98/NT programs on JOS.

Caution

On the other hand, we must exercise caution. JOS should not go beyond bytecode. It shouldn't be optimized for games, or for business applications, or for distributed computing, or for cluster computing. We should do our best to make sure JOS is a good foundation for all these things, and more.

Benefits

By optmizing JOS for bytecode, we are doing something different. What are the benefits? We maximize the benefits provided by Java. Java bytecodes pack a lot of punch in a tiny package. Hardware running Java gives more bang for the buck. Java provides more functionality in the same "space". Applications are smaller and more powerful.

  • I look forward to being able to install and uninstall custom applications without agony. If I have an application server, I might not need to install custom applications at all.
  • I look forward to being able to reuse software components through extreme reuse. The JOS platform encourages this kind of thinking.

Our operating system should be the most optimized-for-Java operating system on the market. It should exceed Linux as the platform of choice. Linux is a tough act to follow. Are we up to the challenge?
