AboutSmartAPIUtilities

Article contributed by GilbertHerschberger (10 October 2000).

Introduction

Like the Universal Browser, an Object Browser would demonstrates the power of the Smart API. Unlike the Universal Browser, an Object Browser would depend upon the instanceof keyword to select an appropriate viewer.

From location to object

A location is typed at the keyboard and selected with the Run button. Any location is passed to the Smart API to obtain an object. To convert a location into an object, the code might look something like this:

	 String s = getLocation();
	 URI uri = new URI( s );
	 Object o = uri.getObject();

Or more compactly, it might look like this:

	 Object o = new URI( getLocation() ).getObject();

If a location does not properly describe a digital resource, the Smart API returns null. It is not possible to browse a null object.

From object to viewer

An object browser is a viewer environment. Once an object browser obtains an object, it must select an appropriate viewer.

  protected Viewer getViewer( String spec, Object o ) {
	 int iMax = list.length;
	 for ( int i = 0; i < iMax; i++ ) {
		if ( list[ i ].acceptsObject( spec, o ) ) {
		  return list[ i ];
		}
	 }
  }
  private Viewer[] list;

From viewer to browser

Once an appropriate viewer has been selected, a viewer is plugged into the Object Browser and an object is selected.

	 Viewer viewer = getViewer( spec, o );
	 viewer.selectObject( spec, o );
	 innerPanel.add( viewer );
	 innerPanel.validate();

Viewer

The acceptsObject() method of each viewer must return true if it accepts a given object. Otherwise, it must return false. A viewer implements a well-known Viewer interface. A viewer does not require a viewer context.

public interface Viewer {
  public abstract boolean acceptsObject( String spec, Object o );
  public abstract void selectObject( String spec, Object o );
}

Here is a snippet from an input stream viewer:

  public boolean acceptsObject( String spec, Object o ) {
	 if ( o == null ) {
		return false;
	 }

	 /* *L!
	 if ( o instanceof InputStream ) {
		return true;
	 }
	 else {
		return false;
	 }
	 */
	 return ( o instanceof InputStream );
  }

Ultimately, an Object Browser must provide appropriate choice. Both the data input stream viewer and the input stream viewer would accept a data input stream. DataInputStream is an extension of InputStream. While an input stream viewer displays a stream in hexidecimal, a data input stream viewer displays a stream as lines ASCII text. An HTML viewer is possible. Even an EBCDIC viewer is possible.

An Object Browser cannot reuse an object once it has been installed in a viewer. It must create a new object when switching from one viewer to another.

Object conversion

Some objects must be converted before they are dispatched to a collection of viewers. A byte array is a kind of object. For example, a byte array might be converted into an ByteArrayInputStream for the purpose of viewing.

About output stream

When a location is an instance of output stream, a dialog box should be displayed to capture input from the keyboard. When the dialog box is closed, the content of the dialog box will be saved to the output stream.
