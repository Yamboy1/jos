Article contributed by GilbertHerschberger.

So you want to build your own operating system. You must ask yourself a serious question:

  • What is the true purpose of an operating system?

If you think the answer is obvious, be careful. Most of the experts cannot agree.

It is our legacy. The software industry has created a long list of definitions for operating system. The phrase comes from a high level concept of being able to do something useful with the otherwise useless hardware. An operating system is a system to 'operate' the computer.

Since the computer is such a general purpose machine, it can be applied to many different tasks, or applications. Did you know, for example, that the first 'operating system' included all of the application software, too? And, application software and an application are not the same thing.

An application is tied up in its purpose. What is the point? We use a computer to accomplish something. At one level, an operating system is everything put together that helps us accomplish it.

When did the application software became more important than the application? When technicians spend most of their time inside application software that doesn't work. The software industry has been confused about the difference between applications and application software for a long time.

Over time, the 'operating system' came to imply all of the hardware and everything that came bundled with the hardware (API and software). It was a simple distinction between the standard bare-bones machine and a fully customized machine. Customization was supposed to be performed through the application programmer interface (API) only.

No, experts do not agree. Sun Microsystems says that the Java Virtual Machine is not an OS. Microsoft says it is. IBM says it's not. James Gosling says it is. I agree with Mr. Gosling. Sometimes I think established companies (even Sun and IBM) like to spread fear, uncertainty and doubt, to reduce the risk of competition.

The JOS project is supposed to take most of the virtual out of Java Virtual Machine, so that the OS and VM are one and the same.

If you want to build an operating system, you are trying to build an architecture and foundation and model, so that other people can build application software. Like hardware, an operating system isn't supposed to do anything useful by itself.

On the other hand, if you want to build application software, you are trying to build on someone else's foundation. It is an honorable thing to build upon the 100% Pure Java foundation so provided by Sun Microsystems.

A good operating system makes it easier for someone else to build application software. When other people build the application software, their customers take the whole thing, all together and apply it to their work.

Here is the theory. If you build a really good operating system, using all of your strength of mind, programmers will find it easier to build upon. Therefore, it will be easier to build application software. With the same effort, such application software will be able to do so much more. Since it can do so much more, it will be in higher demand in a free market competition.

What is the purpose of an OS? I believe it is this: To satisfy the needs of programmers, technicians, administrators, so that they, in turn, can satisfy the needs of end-users.

Architecture is focused on the big picture. It addresses a whole class of problems with a single rule. Architectural questions naturally follow this.

  • What is an operating system designed for?
  • What are the responsibilities of an operating system?
  • Who is going to write which parts of the complete system?
  • Is it going to be compatible with other operating systems?
