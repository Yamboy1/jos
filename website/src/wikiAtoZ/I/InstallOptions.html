InstallPages

Article contributed by GilbertHerschberger (6 March 2000).

Introduction

Unlike other operating systems, JOS does not have a monolithic installation model. This operating system is flexible. It can installed in a number of ways.

When I think of different ways to install JOS, I think of the present and future. In the future, there will be even more ways of enjoying this operating system. As new ways are discovered, additional articles should be added to these InstallPages.

In the present, the leading implementation of the JOS operating system is jJOS/decaf. This implementation has influenced most of the information presented here.

The preferred boot method for UltimateJOS is now GRUB. [1]

General requirements

jJOS/decaf has general requirements for JOS machine, the machine that actually runs the JOS operating system. The minimum hardware for a JOS machine is expected to change with the introduction of a virtual memory manager. With a virtual memory manager, jJOS/decaf will require less real memory.

  • IBM PC-compatible
  • '486/66Mhz
  • 16MB+ RAM

Options

JOS can be installed in a variety of models. How do you determine which model is right for you? Find the best model by answering the following questions:

  • Do you have two machines on a network?

    If yes, your best option is InstallWithEtherboot.

    If no, Go to the next question.

  • Your options are InstallWithGRUB and InstallWithLinux.

    Do you want to take JOS for a quick test drive?

    If yes, your best option is InstallWithLinux.

    If no, your best option is InstallWithGRUB.
