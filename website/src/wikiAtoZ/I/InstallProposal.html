InstallPages

Article contributed by GilbertHerschberger (8 January 2001).

Summary

There might be a more effective way to install a Java-based operating system. All sister projects could use a common installation process.

  • with binary image of the GRUB boot loader
  • with Linux(r) and Microsoft Windows
  • with your choice of operating system

Procedure

The download of an operating system could be automated into the following procedure.

  1. Start the Boot Disk utility.

    The utility downloads a well-known configuration file and displays a list of available operating systems.

  2. Select an operating system from a list and press the Download button.

The Download button runs through the following check list:

  • If necessary, download the RAWRITE tool. This step is for Windows only.

  • If necessary, download the GRUB boot image and prompt for creating a boot disk.

  • Download the operating system (kernel and virtual machine).

  • Display instructions for booting the operating system.

Creating a boot disk

Here is how a binary image of the GRUB boot loader might work.

  • Files are always downloaded to these well-known directories.

    • On Linux, /usr/local/jos-grub
    • On Windows, C:usrlocaljos-grub

  • Download RAWRITE.

    For Windows only, click the "Download RAWRITE" button to download a pre-compiled RAWRITE program from a well-known location on the Internet. This is an un-compressed, un-encrypted Win32 executable image. The latest release of the RAWRITE tool must be maintained at this location.

    /usr/local/jos-grub/rawrite.exe

  • Download GRUB

    Click the "Download GRUB" button to download a pre-compiled GRUB boot loader from a well-known location on the Internet. This is a un-compressed, un-encrypted boot-able image. The latest release of the loader must be maintained at this location. The "Download GRUB" button downloads the boot image to this well-known directory on a local drive

    /usr/local/jos-grub/bootdisk.dat

  • Create Boot Disk

    Click the "Create Boot Disk" button to create a boot disk. To see how a boot disk is created, see also the Boot Disk utility. The bootdisk.dat file is copied to a floppy disk.

Downloading an operating system

There is more than Java-based operating system. Each operating system kernel and virtual machine should be pre-compiled as a GRUB-compatible executable image. Each un-compressed, un-encrypted image should be downloaded from a well-known location on the Internet. Each operating system should be downloaded to a corresponding directory under /usr/local/jos-grub. For example, JJOS (jJOS + decaf) might be downloaded to the following directory:

/usr/local/jos-grub/<STRONG>jjos</STRONG>

Each operating system has a corresponding standard Java class library. This should be downloaded with the operating system.

  • Select an operating system from a list and press the "Download Operating System" button. The file name of the GRUB-compatible boot image should be displayed. The file name of the standard Java class library should be displayed.

  • Use the instructions found elsewhere to boot your operating system using GRUB.

Configuration

A data-registry compatible file can be useful for associating files with an operating system. Such a configuration file should have a well-known location on the Internet. The configuration file is downloaded by a client-side utility.

[JJOS]
path=jjos
wiki=JJOS
kernel=jjos-grub
module=jjos.zip
