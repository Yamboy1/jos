InstallPages

Article contributed by GilbertHerschberger (26 July 2000).

Introduction

With the latest release of the BootDisk application, use the following steps to install JJOS.

  1. Download components
  2. Install JOS Binary Edition
  3. Create a boot disk, and
  4. Booting JJOS with GRUB.

1. Download components

The following components are required to install JJOS with a Boot Disk application.

  • JOS Binary Distribution

    These are the executable images invoked by GRUB or Etherboot. Download a binary distribution for GRUB, such as JOS1e Binary Distribution (GRUB Boot), from The JOS Binary Distributions page.

  • Boot Disk application

    Download the Boot Disk application, a Java application that helps you create a boot disk. Follow the instructions in DownloadBootDisk.

    Note: A Java Runtime Environment (JRE or JDK) is required to run the Boot Disk application. Almost any JRE will do because the Boot Disk application is compatible with JDK 1.02 from Sun Microsystems. Its visual components are based on AWT, not JFC/Swing.

2. Install JOS Binary Distribution

Using the instructions found inside the distribution, install JJOS. The most important files in the distribution are jjos-grub and jjos.zip.

In this example, we'll install these files in the /usr/local/jos-grub directory. At the hardware level, this directory is on drive 1 and partition 1. Drive and partition may vary from one machine to the next. Later, you'll be asked to explain to GRUB how to find these files by drive, partition and name.

3. Create a boot disk for GRUB

The Boot Disk application is platform independent. It is compatible with both Linux(r) and Microsoft Windows(tm). The following steps are required to create a boot disk:

  1. Insert a floppy disk in /dev/fd0 on Linux, or drive A: on Windows.

    Label this floppy disk as GRUB Boot Disk. Format the floppy disk, if necessary, using a utility provided by your operating system.

  2. Start the Boot Disk application.

    It is an AWT application for maximum compatibility with whatever virtual machine you might have.

    $ <STRONG>java org.jos.gchii.bootdisk1a.BootDisk</STRONG>

  3. Select the GRUB option, if necessary.

    The Option field includes both GRUB and Etherboot disk images. The GRUB option is the default.

  4. Press the Run button.

    When you press the Run button, this program writes out a selected boot disk image. A boot disk image is written directly to /dev/fd0 device on Linux. A boot image is written to a file called BOOTDISK.DAT and copied to a floppy with rawrite.exe on Windows.

  5. Close the Boot Disk application

For Windows NT users only:

  1. Use rawrite.exe to copy BOOTDISK.DAT to your floppy disk in drive A:.

    > <STRONG>rawrite BOOTDISK.DAT A:</STRONG>

4. Boot with GRUB

Boot your computer from your boot disk. When you boot with GRUB, you must enter additional GRUB commands when prompted. The following example runs the init application, written in the Java programming language. It looks for files on drive 1 and partition 1. It uses the jjos-grub and jjos.zip files installed during step 2.

command> <STRONG>kernel (hd1,1)/usr/local/jos-grub/jjos-grub init</STRONG>
command> <STRONG>module (hd1,1)/usr/local/jos-grub/jjos.zip</STRONG>
command> <STRONG>boot</STRONG>
