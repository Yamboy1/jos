Article contributed by GilbertHerschberger (6 March 2000).

Introduction

This installation model requires at least two machines, connected through a network. The first machine is a JOS machine, a machine that actually boots with the JOS operating system. The second machine is a server. Here is how the two machine work together:

  1. Boot the server.

  2. On the server-side, start the BOOTP and TFTP services.

  3. Boot Etherboot on the JOS machine from your JOS Boot Disk.

  4. Etherboot contacts the server's BOOTP service to get its IP address and the name of its boot image.

  5. Etherboot gets a boot image from the server's TFTP service.

  6. Etherboot jumps into the boot image. This is how the JOS machine runs JOS.

Requirements

For this model, your JOS machine can be a less expensive, light-duty machine. It might be a "diskless" workstation. It might be a "network computer".

  • The JOS machine boots from a floppy. You'll need a floppy drive.

  • The JOS machine downloads a boot image from the network. You'll need a network interface card.

  • A hard drive and CD-ROM are optional.

While testing, it is easier if the JOS machine is directly connected to the server (on the same network segment).

Installation

To install with Etherboot, there is a server-side and client-side installation. Most of the software installation is on the server-side.

About the server

Any operating system can be used on the server-side. The server operating system must be configured for TCP/IP, run the BOOTP service, and run the TFTP service.

  1. Download the JOS Binary Distribution 1d. Additional instructions and sample configuration files are found inside.

    • .tar.gz (approximately 4,000,000 bytes)
    • .zip (approximately 4,000,000 bytes)

  2. Untar/unzip it on the server machine.

  3. Copy josbin/jjos-nbi to the /tftpboot directory as configured for your TFTP service. The jjos-nbi file must be readable and executeable by anyone on the network.

  4. Test Option: If you have a TFTP client on your server, you should now be able to download the jjos-nbi file.

  5. Test Option: If you have a third machine with a TFTP client, you should now be able to download the jjos-nbi file across the network.

  6. From your JOS machine, get the MAC address for its network interface card.

  7. Configure your BOOTP service. Using the MAC address of its network interface card, assign your JOS machine an IP address. Select /tftpboot/jjos-nbi as its boot image. Make sure the IP address for the JOS machine is unique on your network. Both your JOS machine and server must be part of the same IP subnet.

  8. Create your own JOS Boot Disk. With your server machine, copy a bootable image to a floppy disk. The network interface card that's in your JOS machine must match your Etherboot image.

    About the JOS machine

    Setting up your JOS machine is easy. Move to your JOS machine:

    • Insert the JOS Boot Disk in the floppy drive;

    • Press the reset button and enjoy!
