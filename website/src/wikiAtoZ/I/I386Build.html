Introduction

The i386 build of jJOS/decaf is part of the JJOS module. It is the ultimate JOS. It is required by InstallWithEtherboot and InstallWithGRUB.

Ultimate JOS

This build represents the ultimate implementation of JOS. Unlike HostBuild, it does not depend on a foreign operating system at runtime. The jJOS i386 kernel is used instead of a foreign kernel.

  • Multiple threads

    Multiple threads of control are provided by decaf. The native thread for JOS is the java.lang.Thread in bytecode.

  • Standard C/C++ library

    Parts of the standard C/C++ library, including memory management functions like alloc() and free(), are implemented for JOS in the jJOS kernel.

  • TCP/IP stack

    The JEPS project is building a suitable TCP/IP stack for JOS. A TCP/IP stack must be implemented in order for a JOS program to connect to other machines on a network.

Foreign JDK

Just like the host build, this build depends on a foreign JDK. A foreign JDK must be used to compile the JVM-specific classes. Although the JVM-specific classes are platform independent, the build process is written to build the classes on Linux.

Mistakenly, I did not install an off-the-shelf JDK on my Linux box. If I had, this build would have used it to compile JVM-specific classes. Mistakenly, I did not install a copy of classes.zip. If I had, this build would have used it to create jjos-nbi.
