Introduction

The Invocation API is part of the Java Native Interface. It enables a machine code program to use one or more virtual machines. A native program is written in machine code, it can use any JNI-compatible virtual machine. Many JNI-compatible virtual machines are available from many different vendors. While they have different strengths and weaknesses, they have the JNI in common.

Create a virtual machine

The CreateJavaVM method is part of the Invocation API. When given startup parameters, like classpath, a pointer to a virtual machine structure is returned. This exposes the contents of the structure directly to the machine code.

The CreateJavaVM handles the creation of a separate thread. More than one virtual machine can be created. Each virtual machine is expected to run in its own native thread.

Enumerating virtual machines

The Invocation API also provides a mechanism to enumerate each virtual machine that has already been created. When two or more virtual machines are available, a native method can find other virtual machines within its process.
