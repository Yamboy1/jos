Introduction

The invokexxx opcodes must invoke both native and non-native methods. It is important for the virtual machine to implement identical semantics, regardless of the native flag.

Independent

The native attribute is independent of public, protected, private and final attributes.

  • Public

    When a native method must be useful to classes from other packages, a public native attribute is recommended.

  • Protected

    When a native method must be useful to other classes through inheritence, a protected native attribute is recommended.

  • default

    When a native method must be useful to classes within the same package, a native attribute alone is recommended.

  • Private

    When a native method is useful only within a class, a private native attribute is recommended.

  • Final

    The native attribute is independent of a final attribute. Like a non-native method, the final attribute prevents a derived class from overriding the method.

Inheritence

When a superclass implements a native method, it is explicitely available through the super keyword.

public class MyExample 
	 extends BaseExample {
  public void example() {
	 System.out.println( "example()" );
	 super.example();
  }
}

When a virtual machine invokes a method, it must enforce the semantics of public, protected, private and final.

The super keyword changes the name of the class. A method is invoked based on a combination of class and method. There is no super opcode. The virtual machine does not support a super keyword. The name of the class, the name of a method and its signature must be provided to the virtual machine.

Example

In the above example, the super keyword is used to invoke super.example(V)V. An object (this) is passed as the first parameter to invokespecial. If the superclass is BaseExample and the derived class is MyExample, MyExample.example() is invoked like this:

  store (this)
  invokespecial MyExample/example(V)V

Since MyExample/example(V)V is non-native, a new stack frame must be created. It translates System.out.println( "example()" ) in the usual way.

Within an instance of MyExample, it invokes super.example() like this:

  store (this)
  invokespecial BaseExample/example(V)V

The code for BaseExample/example(V)V is not shown. It might be native or non-native. When BaseExample/example(V)V is non-native, a new stack frame must be created. The opcodes for BaseExample/example(V)V are part of the BaseExample class.

When BaseExample/example(V)V is native, a virtual machine must translate the request from Java to C. The machine code for BaseExample/example(V)V is part of the virtual machine or a shared library.

Without the class name, it would be impossible to invoke a method of a superclass. By the time a virtual machine sees it, the super keyword is long gone.
