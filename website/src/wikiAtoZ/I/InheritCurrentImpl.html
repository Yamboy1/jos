API; SmartAPIPages

Article contributed by GilbertHerschberger (10 June 1999).

Introduction

I have been using a few other techniques to find the current implementation. One of these other techniques is using inheritence. To change the current implementation, edit the source code, change the super class and recompile. Let's jump right into an example.

Application program

In this scenario, you have an application program that always needs the "current" implementation of a Demo interface.

:
import org.jos.demo;
:
  public void example() {
	 Demo app = new CurrentDemo();
	 app.startDemo();
	 app.stopDemo();
  }

Interface

Here again is the Demo interface:

package org.jos.demo;

public interface Demo {
  public void startDemo();
  public void stopDemo();
}

Basic implementation

When writing the demo API, you might offer a basic implementation of the demo interface within the same package, like this:

package org.jos.demo;

public interface BasicDemo
	 implements Demo {
  public BasicDemo() {
  }

  public void startDemo() {
	 System.out.println( "Basic Demo: Start Run" );
  }

  public void stopDemo() {
	 System.out.println( "Basic Demo: Stop Run" );
  }
}

Current demo

Here is an example of obtaining the current implementation through inheritence. When writing the demo API, you might offer a current implementation of the demo interface within the same package, like this:

public class CurrentDemo
	 extends BasicDemo {
  public CurrentDemo() {
  }
}

Fast implementation

You can write and test a new implementation of the demo interface, like this: 

package org.jos.fastdemo;

import org.jos.demo;

public interface FastDemo
	 implements Demo {
  public FastDemo() {
  }

  public void startDemo() {
	 System.out.println( "+" );
  }

  public void stopDemo() {
	 System.out.println( "-" );
  }
}

Transition

Here is an example of changing the current implementation. You must edit the source code, change the super class and recompile.

<STRONG>import org.jos.fastdemo;</STRONG>

public class CurrentDemo
	 <FONT COLOR="#7F7FEF">// *D! extends BasicDemo {</FONT>
	 <STRONG>extends FastDemo {</STRONG>
  public CurrentDemo() {
  }
}

And run

All the old programs will use the fast demo without recompiling. All your old programs benefit from some new implementation without recompiling. If there are fewer bugs and better performance in a new implementation, you win.

On the other hand, you must (1) edit code and (2) recompile in order to switch the current implementation. Or, you must recompile in one directory and copy a .class file to another.

Smart API

A SmartAPI selects the current implementation without recompiling. Searching for every class that implements an interface can be automated. You should be able to pick the "current" implementation for yourself. A current implementation may be found with just a class name. Both org.jos.demo.BasicDemo and org.jos.fastdemo.FastDemo implement the org.jos.demo.Demo interface. The name of every class that implements the org.jos.demo class file could be stored as a system-wide configuration (text file), like this: 

:
org.jos.demo=org.jos.demo.BasicDemo
org.jos.demo=org.jos.fastdemo.FastDemo*
:
Here, the asterisk (*) indicates the "current" implementation.
