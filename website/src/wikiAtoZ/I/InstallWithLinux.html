Not Linux installation

Linux installation is a complex topic. This article is not about the installation of Linux, but the installation of JOS on top of Linux.

What are the minimum requirements of Linux in order to install JOS?

JOS should not be using any libraries from your system except in its 'kernel emulation', where it uses bits and pieces of the standard C++ library and the math library. Optionally, it uses ncurses and libdl. In general, it depends on how the binary was compiled. [1]

Which Linux distribution(s) should I use?

The distribution you have will probably work. This is a question of installing, not compiling. A binary edition of JOS is available for download and install on a Linux machine.

jJOS/decaf isn't terribly demanding. [1] It should run on many distributions of Linux.

What Linux packages should I install to run JOS? to build JOS?

Install the tar and gzip packages. You'll need to untar the binary distribution of JOS/Grub and grub boot. The list of dependencies might change from time to time. jJOS/decaf requires any library it was compiled against.

To build the host target of jJOS/decaf will require g++, gmake, javac, zip, optionally libncurses, and optionally libdl. The i386 build also needs the jjos-nbi softs, etherboot or grub, and NASM. [1]

Runtime Edition

The runtime (or binary edition) of jJOS/decaf contains only the result of the build process. A C/C++ compiler is not required; no C/C++ code is compiled. Another Java development kit is not required; no Java source code is compiled.

How do I install a runtime/binary distribution of JOS on Linux?

You untar it. The distributor may well include a tarball-root script file to execute jJOS/decaf from the right place and with the default options. [1]

Look for instructions in the binary edition itself.

The host build creates a Linux executable. Where should it be installed?

How many Linux executables -- including shared libraries -- are created by the build process? What are their names?

'jjos' is created by HostBuild; 'jjos-grub' and 'jjos-nbi' are created by I386Build depending on which boot process you selected. [1]

Technical Edition

I understand that Linux is used to build JOS. If I download the JJOS module from CVS, a HostBuild is what I need.

How do I download jJOS/decaf from CVS?

Look for the JJOS module. For more information, see also CVS.

How do I make jJOS?, decaf? JVM-specific classes?

jJOS, decaf and JVM-specific classes are created by a make script.

  • For the host build, use this:

    $ cd JJOS/arch/host/nativecode
$ make
  • For the Etherboot version of i386 build, use this:

    $ cd JJOS/arch/i386/nativecode
$ make jjos-nbi
  • For the GRUB version of i386 build, use this:

    $ cd JJOS/arch/i386/nativecode
$ make jjos-grub
