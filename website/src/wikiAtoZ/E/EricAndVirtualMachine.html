Introduction

The virtual-machine component is an abstract base class written in C++. It provides the runtime-library, classpath, startup-class and argument property ofr concrete virtual machines. It also defines the pure virtual run() method.

Extending the virtual-machine class

The eric tool supports two distinct mechanisms for creating a virtual machine. This is reflected in the C++ class design.
  1. All virtual-machine components are similar because all Java Virtual Machines are similar. The VirtualMachine class in C++ is an abstract base class. It provides common properties and methods.
  2. A virtual machine can be launched using one of the java, javaw, jre and jrew tools. The kick tool is called "kick" because it launches, or kick starts, a virtual machine using another tool. The eric tool provides an equivalent kick mode. The KickVirtualMachine class extends VirtualMachine and defines a run() method that launches a virtual machine using another tool.
  3. A virtual machine can be launched using the InvocationAPI of the JavaNativeInterface (JNI). This is a non-kick mode. No other tool is required. The BasicVirtualMachine class also extends VirtualMachine and defines a run() method that launches a virtual machine using JNI.

The VirtualMachine::createVirtualMachine() method

This methods returns an instance of VirtualMachine appropriate for your custom application. To extend the framework, implement a custom class that extends VirtualMachine, BasicVirtualMachine or KickVirtualMachine. Provide your own implementation of this method to return an instance of your custom class. 
static VirtualMachine *
<nolink>VirtualMachine::createVirtualMachine() {
  return new _custom_VirtualMachine();
}
