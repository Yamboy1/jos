Introduction

The ERIC Project is part of the JOS Project. It is an open-source and free-license software project. It has its own home page on the Internet at http://www.mindspring.com/~gchii/eric/ .

We can begin again to demonstrate the awesome power of the network as an on-demand, as-needed distribution mechanism. This distribution mechanism is compatible with agents, applets, applications and servlets.

Goals

This project provides a service to the JOS Project specifically and the Java community in general. Besides creating useful tools for other people to use, this project helps us reach the following goals.
  • Use the InvocationAPI. The Java Native Interface includes an Invocation API. This API is available to C/C++ programmers on Linux and Microsoft Windows. This API enables a programmer to start a virtual machine within a machine code program.
  • Build a custom class loader. How do use a custom class loader in a browser?
  • Design a package file format. A package file format must be refined so that it supports compression and additional items. The package file format will be fully tested and ready for implementation inside JOS. The package file format will be fully supported with a reusable PackageFileAPI. As a Java programmer, you can build your own tools to build ERIC-compatible package files.
  • Integrate a Java application with a foreign operating system. It should not take great effort to get a mostly-bytecode application to run along side all the other applications you must use on a foreign operating system.
  • Experiment with an invocation interface for decaf.
By using an existing virtual machine that fully implements the Java 2 Platform, the applications built on top of ERIC-related breakthroughs will be immediately available to customers without delay.

About platform-independence

We are working toward a more platform-independent eric tool. The first platform-specific build is for Microsoft Windows. The second platform-specific build is for Linux. With release 10 of ERIC 1c, the Linux build is incomplete. It compiles and links, but does not run. It now depends on the Java Native Interface; the jni.h header file is included. The code that uses this interface on Linux has yet to be written and tested.

Meanwhile, the default behavior from the kick tool has been integrated into the eric tool as the "kick" mode. The kick tool invokes the java tool instead of the Java Native Interface.

  • kick=false By default, the eric tool is not in "kick" mode and uses the Java Native Interface.
  • kick=true When in "kick" mode, the eric tool invokes the java tool like the kick tool.
