Introduction

The eric tool uses a specific method to automatically detect a JavaRuntimeEnvironment. It is not necessary to provide the javahome and runtimelibrary properties in your eric configuration file. Here is a step=by-step description.

Properties

Using the virtual-machine component, the are set. See also EricAndBasicVirtualMachine.

Auto-detect process

The eric tool (or your custom C++ program) sets the runtime-library, classpath, startup-class and arguments properties of the virtual-machine component. Once the properties are set, it invokes the run() method. Within the run() method, here is what happens on the Microsoft Windows platform:
  1. The virtual-machine component reads the Microsoft Windows registry. It only reads the registry. It reads from LOCAL-MACHINE/SOFTWARE.
  2. It gets the CurrentVersion property of the JavaSoft/Java Runtime Environment key. The current-version property is a string indicating the currently installed version, such as "1.1", "1.2", "1.3" or "1.4".
  3. Using the current-version property, it gets the JavaHome and RuntimeLib properties of the JavaSoft/Java Runtime Environment/n.n key, where n.n is the current-version.
  4. If there is no current version, or the current version does not point to a valid key, it uses pre-defined versions in the following sequence:
    1. "1.4"
    2. "1.3"
    3. "1.2"
    4. "1.1"
  5. A runtime library is loaded using the runtime-library property. The JNIGetDefaultArgs() and JNICreateJVM() functions are discovered.
  6. Without regard for the current-version, the JNI 1.1 method is used to pass information to the runtime library. A 1.1-compatible runtime library responds by creating a virtual machine. A 1.2-compatible runtime library responds by returning NULL.
  7. Otherwise, the JNI 1.2 method is used to pass the same information to the runtime library. A 1.2-compatible runtime library responds by creating an instance of a virtual machine.
  8. The main() method of the startup-class is invoked, with argument as a parameter.

About Java Software Development Kit

Ideally, the current version of the Java Software Development Kit (JSDK) should be used when there is no Java Runtime Environment.
