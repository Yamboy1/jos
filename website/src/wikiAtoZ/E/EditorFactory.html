EditorPages

Article contributed by GilbertHerschberger (16 Jul 1999).

Introduction

An editor factory is based on SmartAPI. It is able to create editors based on a uniform resource locator (or URL).

Definition

Immediately, there are two kinds of editors.

  • The first kind of editor is the AWT-compatible editor. An editor must extend java.awt.Component and implement the Editor interface.
  • The second kind of editor is the Swing-compatible editor. An editor must extend the Swing equivalent of java.awt.Component. And since Swing has been given different packages at different times, this editor factory must provide a controller for all those packages.

Editor interface

The editor interface is a tag. It identifies an object as potentially compatible with the editor factory. 

public interface Editor {
}

Editor controller

The editor controller creates an editor based on a given URL. It extends the controller interface. 

public interface EditorController
	 extends Controller {
  public static final PROTOCOL = "x-editor:";
}

AWT editor controller

The AWT-compatible editor requires its own controller. 
public class AWTEditorController
	 implements EditorController {
  public static final SUBPROTOCOL = "awt:";
  public static final SUBSUBPROTOCOL = "class:";
  public static final MATCH = 
		PROTOCOL + SUBPROTOCOL + SUBSUBPROTOCOL;

  public AWTEditorController() {
  }
  public boolean acceptsURL( String url ) {
	 if ( !url.startsWith( MATCH ) ) {
		return false;
	 }

	 return true;
  }
  public Object getResource( String url ) {
	 if ( !acceptsURL( url ) ) {
		return null;
	 }
	 String cn = url.substring( MATCH.length() );

	 Class c = Class.forName( cn );
	 Object o = c.newInstance();
	 if ( !( o instanceof Editor ) ) {
		return null;
	 }
	 if ( !( o instanceof java.awt.Component ) ) {
		return null;
	 }
	 return o;
  }
}
