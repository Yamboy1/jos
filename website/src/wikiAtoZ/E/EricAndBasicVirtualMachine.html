Summary

The virtual-machine component enables your C++ program to reliably start a JNI-compatible JavaVirtualMachine. It is part of the ERICProject. Source code is found in the cpp/eric1d/basicvirtualmachine.cpp file on CjOSCVSOnSourceForge .
  • http://cvs.sourceforge.net/cgi-bin/viewcvs.cgi/cjos/cpp/eric1d/basicvirtualmachine.cpp?rev=1.3&content-type=text/vnd.viewcvs-markup

Introduction

The virtual-machine component is written in C++. It encapsulates the InvocationAPI of the JavaNativeInterface. It enables you to easily enable your C++ program to launch a JNI-compatible JavaVirtualMachine.

Setting properties

The virtual-machine component has the following properties.
  • runtime-library. This is a path to a runtime library. Any JNI-compatible runtime library can be selected. jvm.dll or javai.dll is sometimes used on Microsoft Windows. We continue to work on an implementation of this component on Linux.
  • classpath. This is the java.class.path property. This component does not yet provide a complete classpath property. A more complete classpath property would be a component, which, for example, might hide the details of a platform-specific path separator. It might enable the addition of individual paths, enabling a warning when a path does not actually exist at run-time.
  • startup-class. This is the name of a Java class, using dot notation familiar to Java programmers. This class is a bridge between the virtual-machine component and your main-class.
  • argument. There is only one argument supported. This argument is a query string, as in a URI. It is passed to the startup-class. In turn, the startup-class is responsible for separating the argument, if necessary, into its component parts and passing information to a main-class. This component does not yet provide a complete argument property. A more complete argument component might enable you to set each argument separately, providing URI-encoding and appropriate argument separator.
