Introduction

A Linux-compatible build of the eric tool is still under construction.

Existing jars

It is important for the eric tool to dynamically create a java.class.path property using a well-defined algorythm. Java archives (jars) in the erichome .lib directory are automatically added to the java.class.path property so that you do not need to manually edit a configuration file every time you install a new archive.
  1. Read the names of all files in the erichome .lib directory.
  2. Save names which end with .zip and .jar.
  3. Sort names into alphabetical order so that behavior is well-defined and predictable.
  4. Return names in a single string, separated with a platform-specific separator. On Linux, the separator is a colon (:). On Windows, the separator is a semi-colon (;).

Sorted string list

File names must be stored in a well-defined structure so that names can be sorted with a string-sort method. It is not important for this method to be highly optimized because there will typically be only a handful of jars in the erichome .lib directory.

It might be best to build a class in C++ to encapsulate the behavior of a sorted string list. One string at a time is added to the list. The size of the list grows; items are always returned in alphabetical order. 

 class sorted-string-list {
   public:
     sorted-string-list();
     virtual ~sorted-string-list();

     virtual int getSize() const;
     virtual const char *getItem( int index ) const;
     virtual void addItem( const char *v );

   protected:

   private:
     string-node *ptop;

 };

Adding an item

Each string-node is capable of storing a single string and another string-node. A new string is inserted into the list in alphabetical order, something like this: 
 sorted-string-list::addItem( const char *v ) {
   string-node *pnew = new string-node( v );

   if ( ptop == NULL ) {
     ptop = pnew;
     return;
   }

   string-node *pcurr = ptop;
   for (;;) {
     if ( pcurr->getNext() == NULL ) {
       pcurr-setNext( pnew );
       return;
     }

     if ( pnew.lessThan( pcurr.getNext() ) ) {
       string-node *ptemp = pcurr.getNext();
       pcurr.setNext( pnew );
       pnew.setNext( ptemp );
       return;
     }

     pcurr = pcurr->getNext();
   }
 }

Size property

The size of the sorted-string-list could be calculated or maintained. 
 int sorted-string-list::getSize() {
   string-node *pcurr = ptop;
   for ( int size = 0;;) {
     if ( pcurr == NULL ) {
       return size;
     }
     else {
       size++;
     }

     pcurr = pcurr->getNext();
   }
 }

String node

A string-node is similar to a char-buffer. In addition to the behavior of a char-buffer, a string-node can store another string-node and compare itself to another string-node. 
 class string-node : public char-buffer {
   public:
     string-node();
     virtual ~string-node();

     virtual bool lessThan( const string-node *other ) const {
       if ( other == NULL ) {
         return false;
       }
       if ( other->getPayload() == NULL ) {
         return false;
       }
       return strcmp( getPayload(), other->getPayload() ) < 0;
     }
     virtual const string-node *getNext() const {
       return pnext;
     }
     virtual void setNext( const string-node *v ) {
       pnext = v;
     }

   protected:

   private:
     string-node *pnext;

 };
