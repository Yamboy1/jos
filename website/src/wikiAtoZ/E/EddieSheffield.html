I've recently taken an interest in Jini. I'd like to see a version of JOS developed that would be Embedded/Personal Java compliant and capable of supporting Jini. Being something of a hardware guy at heart my vision is to be able to create a simple system core with a low-cost processor, ROMed JOS/Jini support, RAM, networking, and an easy to design for logical bus. These cores would be extended in various ways - IDE interface, SCSI interface, serial, parallel, USB, etc. - to create low cost boxes providing various network services via Jini. For example:

  • Core + IDE + HD = storage service
  • Core + SCSI + scanner = scanning service
  • Core + parallel + printer = printer service

Perhaps these could even be embedded, for instance on an HP MIO type card to turn the printer into a printing service directly.

Let me know if there's any interest! I picked up a few interested parties on the kernel list recently.
