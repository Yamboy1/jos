Article contributed by GilbertHerschberger (5 August 2000).

Introduction

A boot loader is the first step in running an operating system. The Linux operating system boots from a boot loader called LILO. LILO loads and starts the Linux kernel.

GRUB is required to boot JJOS. GRUB and LILO are similar, but incompatible. The I386Build of JJOS is compatible with Etherboot and GRUB, but not LILO.

A GRUB Boot Disk is a special floppy disk that enables you to switch between Linux(r) and JJOS on a single machine. This reusable disk can be used to boot JJOS and other operating systems.

Procedure

Build your own GRUB Boot Disk. The following instructions assume you're able to log in as root to an existing Linux system.

  1. Install
  2. Prepare
  3. Copy

Caution: These commands are capable of installing a filesystem on your hard drive, wiping out all of the files stored there. Be careful to specify a floppy device.

1. Install

You can install GRUB by compiling it from source code. See also GRUBInstallFromSource. Or, you can install GRUB by downloading the GRUB binary/runtime distribution. See also GRUBBootDiskFromImage.

2. Prepare

Before a boot image can be copied to it, a floppy disk must be prepared with a filesystem.

  • If a floppy disk has already been mounted, unmount and remove it now. A floppy can be unmounted with a Linux command, like this.

    # <STRONG>umount /floppy</STRONG>

  • To insert a floppy disk, insert the floppy disk in the drive, but do not mount it. Find an unused 1.44MB floppy disk. Label it as your GRUB Boot Disk. All information on the floppy disk will be destroyed.

  • Install a filesystem on floppy disk.

    A filesystem must be installed on the floppy disk. If a filesystem has already been installed, go to the next step. Otherwise, the FAT or ext2 filesystem is prefered. [A40] Use one of the mkfs.* commands in Linux, like this:

    # <STRONG>mkfs.ext2 /dev/fd0</STRONG>
    or 
    # <STRONG>mkfs.msdos /dev/fd0</STRONG>

    At this point a floppy disk could be mounted, but do not mount it. If any error messages are displayed while making a filesystem, use another floppy disk.

3. Copy

After a floppy disk is prepared, copy a boot image to it.

  • To change to the GRUB directory, use one of the following Linux commands.

    • After compiling GRUB from source code:

      # <STRONG>cd /usr/local/shared/i386-pc/grub</STRONG>

    • After downloading binary/runtime distribution:

      # <STRONG>cd /usr/local/GRUB</STRONG>

      For more information, see the instructions distributed with the GRUB binary edition.

  • To install GRUB on a floppy disk in /dev/fd0, use the following Linux commands:

    # <STRONG>dd if=stage1 of=/dev/fd0 bs=512 count=1</STRONG>
# <STRONG>dd if=stage2 of=/dev/fd0 bs=512 seek=1</STRONG>

    If any error messages are displayed while installing GRUB on a floppy disk, go back to step 2 and use another floppy disk.

Reference

  • The GRUB homepages is http://www.gnu.org/software/grub/ .

  • Usually you don't want to use a minix filesystem, but rather FAT or ext2. -- John Leuner [A40]
