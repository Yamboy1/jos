Introduction

Maintenance of the glossary for JOS Wiki is part of the cost of doing business. It is overhead. It is grunt work. It doesn't contribute directly to the development of JOS-compatible software or designs.

Without an automated search, the glossary provides a way to find many articles. By displaying a page and using your browser's built in search, it is possible to locate articles based on article name.

Note: There are many articles on JOS Wiki. Each member should make a point of reading all of them. Here is a suggested way to keep track of how much you've read:

  1. Print out the glossary pages, and
  2. Check off each article as you read them.

Glossary

The Glossary is part of JOS Wiki. It is an article index. It has 26 pages, from A to Z.

The glossary is an article index. As a simple dictionary, it should contain a link to every term defined in this collection. Some definitions have been "lost" because an article is unreferenced by another article (and search is disabled).

Each glossary page contains links to other articles. Typically, the content of a page contains nothing but links. It might contain no other text. At the top of each glossary page, there are links to other glossary pages, from A to Z.

Methods

Authors might define a term using an article or the glossary. The preferred method is with an article. With this method, each term is defined with its own article. The glossary provides only a link to the article for each term defined. A definition can be found by typing a term in the Go: box.

The glossary has been maintained using two methods. Previously, a term would be defined on the glossary itself. This causes many problems. Among the foremost problem is how a definition could not be found by typing the word in the Go: box. The previous and preferred method are always separated by a horizontal line.

Changes page

Articles are added and changes. The changes page shows articles that have changed recently. As they age, articles eventually fall off the changes page. Immediately after an article is added, it is shown on the changes page.

Unlike the changes page, the glossary does not show an article immediately after it has been added. To do so would require constant editing of the glossary pages. That would require a lot of unwanted and duplicate effort. Changes to the glossary page would constantly show up on the changes page, which is undesireable.

Regular maintenance

The glossary needs regular maintenance. It should be updated four times each year, in January, April, July and October. At this time this article is being written, its regular maintenance is expected in within next few days.

The glossary is brought up-to-date by downloading the entire changes page from the sfWiki software and reviewing it. Changed articles are separated from new articles. An analysis of the changes page and the previous glossary shows all articles that are new. With the introduction of the sfWiki software, there is no longer a need to save the changes page to disk.

Using normal on-line browsing, each glossary article is updated with new articles. If there are no new articles for a particular letter, the corresponding page is not changed.

Glossary articles are updated in reverse alphabetical order, from Z to A, so that the changes page will display them in alphabetical order.
