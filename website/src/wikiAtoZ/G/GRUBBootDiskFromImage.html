InstallPages

Article contributed by GilbertHerschberger (8 August 2000).

Introduction

A boot disk is a great way to boot JJOS. A boot disk is a special floppy disk with a boot loader installed. It enables you to switch between operating systems temporarily without reconfiguring your existing system. A boot disk can be used to boot JJOS and other operating systems.

Procedure

This procedure might be the easiest way to build your own GRUB Boot Disk. It enables you to take JJOS for a test drive. The following instructions assume you're able to log in as root to an existing Linux system.

  • Download
  • Prepare
  • Copy

Caution: These commands are capable of installing a filesystem on your hard drive, wiping out all of the files stored there. Be careful to specify a floppy device.

1. Download

The first step to install GRUB is to download a binary distribution.

  • Make a new directory.

    $ <STRONG>mkdir ~/bootdisk</STRONG>
$ <STRONG>cd ~/bootdisk</STRONG>

  • Download a binary distribution of GRUB. For example, a distribution is available at the following location:

    <STRONG>ftp://alpha.gnu.org/gnu/grub/old/grub-boot-0.5.95.image</STRONG>

    After downloading grub-boot-0.5.95.image, it should be in your ~/bootdisk directory.

2. Prepare

Before a boot image can be copied to it, a floppy disk must be prepared. A filesystem must be installed.

  • Switch to the root account. The following steps will require superuser privileges.

  • Move to the bootdisk directory, if necessary.

    # <STRONG>cd /home/<EM>user-name</EM>/bootdisk</STRONG>

  • If a floppy disk has already been mounted, unmount and remove it now. A floppy can be unmounted with a Linux command, like this.

    # <STRONG>umount /floppy</STRONG>

  • To insert a floppy disk, insert the floppy disk in the drive, but do not mount it. Find an unused 1.44MB floppy disk. Label it as your GRUB Boot Disk. All information on the floppy disk will be destroyed.

  • Install a filesystem on floppy disk.

    A filesystem must be installed on the floppy disk. If a filesystem has already been installed, go to the next step. Otherwise, the FAT or ext2 filesystem is prefered. [A40] Use one of the mkfs.* commands in Linux, like this:

    # <STRONG>mkfs.ext2 /dev/fd0</STRONG>
    or 
    # <STRONG>mkfs.msdos /dev/fd0</STRONG>

    At this point a floppy disk could be mounted, but do not mount it. If any error messages are displayed while making a filesystem, use another floppy disk.

3. Copy

Once the floppy disk is prepared, a boot image can be copied to it.

  • Switch to the root account. The following steps will require superuser privileges.

  • Move to the bootdisk directory, if necessary.

    # <STRONG>cd /home/<EM>user-name</EM>/bootdisk</STRONG>

  • To install GRUB on a floppy disk in /dev/fd0, use the following Linux command:

    # <STRONG>dd if=grub-boot-0.5.95.image of=/dev/fd0</STRONG>

    If any error messages are displayed while installing GRUB on a floppy disk, go back to step 2 and use another floppy disk.

Reference

  • The GRUB homepages is http://www.gnu.org/software/grub/ .
