Introduction

As you may know, RobertFitzsimons has been generously keeping a log of IRC meetings for the JOS Project. IRC meetings are interactive and up-to-the-minute discussions. They are announced on the JOS Project homepage and typically held on Sunday.

Robert posts Logs for other members to download and read in a plain-text editor. These logs enable attendees to review a discussion sometime later. They enable anyone to review a discussion they may have missed. While logs have been a good thing for the project, it could be better. Some logs have many lines. These are especially difficult to read and research in a text editor. For those who cannot attend, it is difficult to reference a specific comment discussed in a log file. While the log has timestamps for each comment, it is sometimes difficult to reconstruct the timeing of an on-line conversation.

For these and other reasons, a log file could be converted into an HTML page. Other members could download and read logs in an HTML browser. Other HTML pages, such as JOS Wiki articles, could make references to specific comments in the logs. Issues would be more easily researched.

Contest

Robert has tenatively agreed to run a script to convert a log to HTML -- if someone else would write it. Now this sounds like a contest. I hereby challenge all other Java programmers subscribed to this list. Who can write such a program in the fewest lines of code, with the least complexity? Any program must run on Robert Fitzsimons' computer. It must use only open source and free license software.

A program must read a log file and format it into an HTML page (or pages). I'm sure the quality of the HTML page will partly determine which program Robert uses. The program Robert actually uses to format the HTML pages will be considered the "winner". Robert is not required to use any program submitted by this contest and there may be no "winner".

  • For those members who think it is too hard, you might wait until someone else submits a program for review and then improve it. That is the essence of open source development, isn't it?

  • For those members who think this is too easy, you might want to throw something together and be the first to submit a program for review. Help us get the contest going.

  • You do not need to work alone. Form your own team and submit a team effort.

  • The design of your HTML page is your problem. Don't know HTML? Find someone who does. The design of the command line or user interface is your problem.

  • All programs written in the Java programming language should be placed in a package. Do not use the default package! I would recommend a package name something like this:

    org.jos.<EM>your-name</EM>.irclog

    where your-name is your name, your alias or your team's name.

  • To simplify the contest, the name of the main class should be "Main".

Modifications

If you don't like the rules, please make a suggestion. Send a message to the jos-general mailing list. If you don't like the contest, please think of this as an opportunity to support a worthy cause. Obviously, the logs might be more accessable to our members as a one-step download of HTML pages rather than a multi-step download of an archive and text editor. More importantly, we can benefit from the combined ideas of many members and their diverse styles of programming. Besides, you might win the contest.
