Introduction

Starting at the most primative, a line editor comes first. A line editor listens for keystrokes (keystroke events) and offers limited editing features to edit a line of text.

Line property

A line editor manages a line property. Modifying this property automatically repaints the line.

  public String getLine() {
	 return buffer.toString();
  }
  public void setLine( String v ) {
	 onKey( Key.DEL );

	 int iMax = v.length();
	 for ( int i = 0; i < iMax; i++ ) {
		onKey( Key.getKey( v.charAt( i ) ) );
	 }
  }
  public void onKey( Key v ) {
  :
  }

Append

There are specific characters that should be allowed in a line editor. These are the letters A-Z, a-z and digits 0-9. There are a few other characters, too.

  private static final String UPPER = "ABCDEFGHIJKLMNOPQRSTUVWXYZ";
  private static final String LOWER = UPPER.toLowerCase();
  private static final String DIGITS = "0123456789";
  private static final String OTHER1 = " t"'&<>+-*/^";
  private static final String OTHER2 = "`~!@#$%(){}"
  private static final String ALLOWED =
		 UPPER + LOWER + DIGITS + OTHER1 + OTHER2

  public void onKey( Key v ) {
	 if ( key.isAscii() ) {
		int pos = ALLOWED.indexOf( "" + Key.toChar() );
		if ( pos < 0 ) {
		  return;
		}
		buffer.append( Key.toChar() );
		repaint();
	 }
  }

Backspace and delete

The line editor enables you to go back and fix a mistake. The backspace and delete keys remove characters from a line.

  public void onKey( Key v ) {
	 if ( v == Key.BS ) ) {
		buffer = buffer.substring( 0, buffer.length() - 1 );
		repaint();
		return;
	 }
	 if ( v == Key.DEL ) ) {
		buffer = new StringBuffer();
		repaint();
		return;
	 }
	 :
  }

End of line

A line editor might provide two keystrokes for end of line. The first keystroke (CR?) indicates that the user wants to continue. The second (ESC?) indicates the user want to cancel.

  public void onKey( Key v ) {
	 if ( v == Key.CR ) {
		synchronize ( flag ) {
		  flag = Flag.OK;
		}
		return;
	 }
	 if ( v == Key.ESC ) {
		synchronize( flag ) {
		  flag = Flag.CANCEL;
		}
		return;
	 }
	 :
  }

When to use

A line editor can be used by a text-based browser/desktop/shell. A line editor can be used remotely. A keystroke-to-stream and stream-to-keystroke conversion pushes keystrokes across a process barrier.

The repaint() method might be optimized to perform the minimum work possible to represent a line of text. For example, a backspace might be written out to a terminal as BS, SP, BS. A delete might be written out as CR, CLREOL.

When the init process for JOS runs a text-based shell, the shell can use a line editor to gather keystrokes from the keyboard into a single line, which is a more useable form. A shell should process a line at a time. By using a stand-alone line editor, a shell can forget about individual keystrokes.

Special keys

Most of the time, a line editor passes lines to a line listener; it does not pass individual keystrokes. A line listener can update the special keys property of a line editor to force the line editor to broadcast special keys.

  public LineListener getListener() {
	 return listener;
  }
  public void setListener( LineListener v ) {
	 listener = v;
  }
  private LineListener listener;
}

A line editor provides special behavior for special keys (BS, DEL, ESC, CR). A listener can override the behavior of a special key by adding it to the external special key property. A keystroke will be broadcast to a listener instead of being used by the line editor.

Testing

This component can be tested in an off-the-shelf virtual machine. An AWT or Swing component is required. Or, a network connection and terminal is required. The standard in does not provide individual keystrokes because it uses a JVM-specific line editor. The line editor provided by an off-the-shelf JVM does not have a special key property.

public interface LineListener
	 extends KeyListener {
  /**
	* A line event.
	* @param v a single line
	*/
  public void onLine( String v );
}

A key event from AWT/Swing is mapped to a key for the line editor. A line editor does not provide additional cursor control, such as a left or right arrow. The cursor should track with the last character on a line.

A line editor can be reused. After running a line editor, it can be run again. Each time it is run, it broadcasts zero or more keystroke events and exactly one line event. It uses an asynchronous I/O model, like finder package.

This sample program runs until you press the Enter key (CR) or Escape key (ESC).

public class SampleShell
	 extends BasicConsoleProgram
	 implements LineListener {
  public SampleShell() {
  }
  public void run() {
	 Terminal t = new AWTTerminal();
	 LineEditor editor = new BasicLineEditor();
	 editor.setKeyboard( t );
	 editor.setConsole( t );
	 editor.setListener( this );
	 editor.run();
  }
  public void onLine( String v ) {
	 println( v );
  }
  public void onKey( Key v ) {
	 println( "onKey( " + v + " )" );
  }
  public void onClose() {
	 println( "onClose()" );
  }
}
