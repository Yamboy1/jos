Introduction

XDC stands for Extreme Distributed Computing. It was created in eleven days in December 1998 in a project sponsored by Realign Software Inc. ( http://www.realignsoft.com/ ) Like JOS, XDC is open source and free license. For more information, see also the JavaDoc ( http://www.realignsoft.com/products/xdc/ ) available now.

XDC is an entry in the emerging market of personal distributed computing. Distributed computing comes in two sizes: personal and business. While CORBA and JINI is big business distributed computing, CORBA and JINI are out of the reach of many medium and small businesses. Personal distributed computing is less complicated than business distributed computing and can be understood by more programmers.

Attention! We want to contribute XDC to the JOS Project and need your help. Send E-Mail ( gchii@mindspring.com ).

  • Note: The architecture of XDC is similar to ORKA. A custom device written in Java will run inside both XDC and ORKA.

Dynamic storage object

Inside XDC, the dynamic storage object is called a buffer (xdc.Buffer). The methods of a dynamic storage object for Java should provide most of the buffer interface. For more information, see also BasicBuffer in JavaDoc ( http://www.realignsoft.com/products/xdc/xdc1adoc/com.realignsoft.xdc1a.BasicBuffer.html ).
