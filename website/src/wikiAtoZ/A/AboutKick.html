Introduction

The kick tool enables you to launch your Java application without a custom script on Linux or custom batch file on Windows or custom shortcut on Windows.

The kick tool provides additional system properties to enable the upcoming Fork API. This API enables your Java application to clone itself, invoke another Java application with the same classpath and system properties, start-up a Java application in another directory.

Features

It enables you to include all jars in an arbitrary directory using the following syntax:
  • classpath=[directory-name]
In turn, the kick tool invokes the java tool for a Java Development Kit or the jre tool for a Java Runtime Environment. It is compatible with JDK 1.0.x, JDK 1.1.x, JDK 1.2.x and JDK 1.3.x.

At runtime, the kick tool finds the appropriate tools and archives. It provides human-readable messages when none of the following tools can be found:

  1. java
  2. jre
  3. javaw
  4. jrew
It also provides a human-readable message when neither of the following ZIP-compatible archives can be found:
  1. rt.jar
  2. classes.zip
The kick tool dynamically adds the standard rt.jar or classes.zip archive to classpath so that you don't have to.

The kick tool dynamically adds the program directory to classpath so that you can launch your Java application from its original class files. The program directory is where the kick tool itself is stored.

Compared to the java tool

The kick tool is easier and more powerful than the java tool.
  • Like the java tool, it requires a main class. The main class must implement the standard "static main" method.

Compared to the eric tool

The kick tool is simpler, and thus less powerful, than the eric tool.
  • Like the eric tool, you do not need a C/C++ compiler to distribute a custom application. You can rename the kick tool to the name of your Java application. You can rename kick to myApp on Linux. You can rename kick.exe to myApp.exe on Windows. It will read its configuration from myApp.ini.
Unlike the eric tool, it is a console application. It is more like the java tool than the javaw tool.

Unlike the eric tool, it never reads the Microsoft Windows registry. Support for kick shortcut files is planned. Support for kick configuration files is planned.

  • It does not support package files.
  • It does not support downloading packages from an HTTP service.

Properties

The kick tool supports the following properties:
  • javahome. The javahome property enables the kick tool to use a specific Java Development Kit or Java Runtime Environment. Only the last javahome property will be used.
  • classpath. The classpath property enables you to specify additional directories and archives. Use a separate property for each directory or archive. The kick tool itself provides the appropriate platform-specific separator. A warning will be displayed for each directory specified that does not exist. A warning will be displayed for each archive specified that does not exist.
  • mainclass. The mainclass property enables you to specify the main class for your Java application. Only the last mainclass property specified will be used.
