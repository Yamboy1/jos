Introduction to version 1i

This version of JJOS started with version 1f. It was an attempt to refine and improve the BytecodeNativeInterface by integrating it with the decaf virtual machine.

The "i386" build boots from both Ethernet and GRUB. The "host" build of this version runs on Linux.

Module

This version is stored in the jjos1i module on JJOS CVS on SourceForge.

One archive

Unlike version 1f, this version has only one jjos.zip archive. One archive is used for both the "host" and "i386" builds. There are many benefits in using a single archive.
  • All of the Java source code is found in a single directory tree. The Java source code is found in the bytecode directory, instead of the common/bytecode, arch/host/bytecode and arch/i386/bytecode directories.
  • Every class has a unique name. The "host" device drivers are found in the decaf.host package and their class names start with "Host". The "i386" device drivers are found in the decaf.i386 package and their class names start with "I386".
  • Bytecode is compiled from a separate make file. Compiling machine code and bytecode have been separated into different make files. This enables a programmer to compile Java source code on one machine and compile machine code on another.
