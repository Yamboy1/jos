Introduction

This article provides a first look at the Smart API.

Summary

The Smart API is a multiple-stage object factory (MultipleStageFactory), with emphasis on uniform resource identifiers. A multiple-stage object factory finds a factory (in one or more stages) and creates an object (in its last stage). Here, every object is considered a digital resource.

Purpose

The application programming interface (API) itself consists of two classes.
  • org.jos.smartapi4a.Controller This interface is implemented by plug-ins. Each plug-in is an object factory.
  • org.jos.smartapi4a.URI This class is used by applications to create many kinds of digital resources.
  • http://cvs.sourceforge.net/viewcvs.py/cjos/java0/org/jos/smartapi4a/

Plug-ins

Plug-ins are everything. A multiple-stage object factory is useless without plug-ins. Plug-ins define all run-time behavior.

In your custom application, you have a spectrum of choices.

  • The default factory is pre-configured and ready to use.
  • To extend its behavior, you can design a custom controller and add it to the default factory.
  • To override and replace its behavior, you can replace the default factory with a custom factory of your own design. Choose your own configuration mechanism. Configure it with a mix of default and custom plug-ins.

Default factory

The default factory for the Smart API provides a collection of default plug-ins. It demonstrates what this API might do for your system.

For version 2a, the default factory is implemented in the org.jos.gchii.factory1b package.

  • http://cvs.sourceforge.net/viewcvs.py/cjos/java0/org/jos/gchii/factory1b/
For version 4a, the default factory is implemented in the org.jos.gchii.factory1c package.
  • http://cvs.sourceforge.net/viewcvs.py/cjos/java0/org/jos/gchii/factory1c/
Current distributions of Smart API include sample controllers, factories and applications. The UniversalBrowser is an interactive demonstration.

Uniform resource identifier

Uniform resource identifier is defined by W3 ( http://www.w3.org/ ) and was made popular by the ubiquitous Internet HTML browser. It is uniform identifier because it uses explicit schemes for each kind of digital resource and/or network protocol.
