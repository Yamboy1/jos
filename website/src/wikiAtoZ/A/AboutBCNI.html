BytecodeNativeInterface; DownloadBCNI

Article contributed by GilbertHerschberger (6 October 2000).

Introduction

The BytecodeNativeInterface (BCNI) is a well-known object factory. It is intended to create vm-specific objects, if possible. It can be invoked from any method.

API

The application programming interface (API) itself consists of two classes.
  • org.jos.bcni2a.Factory - This class is extended by a virtual machine designer. Each implementation is vm-specific.
  • org.jos.bcni2a.Interface - This interface is used within the standard Java class libraries to find an implementation of an interface.

Purpose

This could be the foundation of a VersionlessVirtualMachine. With the bytecode native interface, a more portable standard Java class library is possible. One virtual machine can be used to develop and test a standard class library for another virtual machine.

This design uses interfaces instead of using abstract classes. In classic virtual machine design, abstract classes are used to define the interface between the standard Java class library and a virtual machine. Otherwise, a virtual machine is version-bound, required to re-implement abstract classes for a specific Java "platform". Or, a virtual machine is tied to machine code, forced to define machine code methods even when bytecode methods are more appropriate.

Current release

The current release is 2a ( DownloadBCNI ).
