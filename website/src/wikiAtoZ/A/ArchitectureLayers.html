Introduction

An operating system is very large puzzle. It is helpful to divide it into layers, from the bottom up. The bottom-most layer is the physical hardware. Higher layers depend upon lower layers.

The first step in a traditional operating system is the definition of architectural layers. While JOS is not a traditional operating system, it is still described in four layers.

Diagram

It would be helpful to see a diagram of architectural layers for JOS. A diagram of layers would help describe the dependencies one layer has upon another. It would also help us understand how much of each layer is done, complete, finished, useable, distributable and doesn't need to be designed.

Three layers

From the traditional operating system view point, an operating system has (at least) the following three layers:

  1. Hardware

    The hardware layer is the part of a computer you can touch. It contains a real processor (or real microprocessor) and assorted supporting components, such as a bus, RAM, ROM, real-time clock, DMA, and various ports, cables and peripherals.

  2. Kernel

    The kernel layer is the part of the software that has exclusive control over the operation of a real microprocessor. It is a scheduler and coordinator for a general purpose computer. From a microprocessor's viewpoint, the kernel is the only program it ever runs.

  3. Program

    The program layer is the part of the software that has limited control over the operation of a real processor. Both device drivers and full applications are part of the program layer because they have limited control over the operation of a real processor.

    Four layers

    JOS is different. It squeezes a Java virtual machine between kernel and program layers. Again, from the traditional operating system viewpoint, JOS has (at least) the following four layers:

    1. Hardware

      The hardware layer is identical to a traditional operating system. Both JOS and a traditional operating system will run on top of the same hardware.

    2. Kernel

      The kernel layer is identical to a traditional operating system. While this layer is identical, the implementation of a JOS kernel is, of course, unlike other operating systems.

    3. Java virtual machine

      A Java virtual machine runs "on top of" the JOS kernel. This is the "extra" assumption JOS has that other operating systems do not have.

    4. Program

      The JOS program layer is slightly different than the traditional program layer. JOS programs have limited control over the operation of a Java virtual machine, not a real processor. Both device drivers and full applications are part of a program layer because they have limited control over the operation of a Java virtual machine.

      Even more layers

      There may be more components required between the Java virtual machine and applications. While these components are necessary, they are part of the Java virtual machine layer, and should be.

      In JOS terms

      From the JOS viewpoint, these four layers are described this way. JOS has (at least) the following four layers:

      1. Hardware/bare metal

        This is again the physical hardware layer.

      2. JOSBox

        What is the JOSBox? It is written in native code for a real processor (or microprocessor).

      3. JOSSystem

        What is JOSSystem? It is written in a mix of Java and native code.

      4. User Applications

        What is a user application layer? It is written in 100% pure Java code.

        Questions

        This article does not have all of the answers. Here are more questions:
