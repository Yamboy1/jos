ProcessPages

Reviewed by GilbertHerschberger (30 November 2000).

Introduction

Multiple bytecode process is in the news again. It seems that everyone wants it. Should we modify all Java applications, or not?

To modify

Some people believe that the right answer for multiple bytecode processes is modifying all existing Java applications. If every Java application were made more polite, they say, all would get along inside a Java Virtual Machine. They blame the Java programmer, saying that our Java applications aren't good enough. If we had better Java applications, they assert, we would not need a multiple bytecode process.

Jonathan Knudsen is an author and developer at O'Reilly & Associates. He has summarized this viewpoint in the following article:

  • Java Application Etiquette

Some of his suggestions are sound, some are amusing.

Not to modify

Others believe no Java application needs to be modified. The reasons for this are clear. [1] Multiple virtual machines and/or a MBCP-compatible virtual machine could solve the problem. Is it worthwhile to prepare your Java application now, or not?

Unexpected things happen. It is not possible to create a Java application that is so polite it closes all of its system resources automatically without help from a machine code or bytecode process.

Java programmers are doing their best to write portable Java applications. They have a long list of important things they must keep in mind to have a Java application that works. I believe it is most inappropriate to blame Java programmers for features missing from the Java environment. If the Java Virtual Machine Specification allowed for multiple bytecode processes, the virtual machine could free all system resources when an application dies.

Therefore, it is not worthwhile to modify existing Java applications to fit inside the new model. It might be worthwhile to change the way new Java applications are written. Most programmers should build a Java program, not Java application. Java programs get along within a single virtual machine. For more information on Java programs, see also ProgramQuotes .

Correction

Both a Java application and a Java program are required. An application needs a process to itself, either a machine code or bytecode process will do. It is not enough to build Java applications with a view to sharing a virtual machine with other applications. We want control over starting, stopping and monitoring applications. Unless a virtual machine is explicitely aware of each application, it cannot keep track of its system resources. It is not necessary for an application to close its own system resources, such as windows, files, sockets. That is what the ultimate environment must do.
