Introduction

We are about to dust off the decaf virtual machine and create a new version. The new version will be called "3a". If you would like to join our effort or make suggestions, join us on our jos-general MailingList.
  • Unlike previous versions, decaf will be called "decaf" and not "JJOS".
  • The C/C++/ASM source code for the jjos kernel should be maintained separately and made available to this product as a static library.
  • The Java source code should be maintained separately and made available to this product as a ZIP-compatible archive.
  • Unlike previous versions, changes to this version will be enclosed in #ifdefs so that the base version can be recompiled. Platform-specific and build-specific code will also be enclosed in #ifdefs that are more readable and based on the availability of each feature.

In C++

In general a C++ class should have a corresponding toCStr() method to convert it to a print-able ASCIIz string or a toUnicode() method to convert it to a Unicode string. This should make it easier to debug a virtual machine.

Something must be done about primative arrays. A byte array should be an array of 8-bit values. A char array should be an array of 16-bit values. A short array should be an array of 16-bit values. An int array should be an array of 32-bit values.

BaseArray

  • public cint32 getSize() const;
  • Int32Array extends BaseArray
    • public cint32 getInt( cint32 n ) const;
    • public void setInt( cint32 n, cint32 v );
    • ByteArray extends Int32Array
    • CharArray extends Int32Array
    • ShortArray extends Int32Array
    • IntArray extends Int32Array
    • FloatArray extends Int32Array
  • Int64Array extends BaseArray
    • public cint64 getInt( cint32 n ) const;
    • public void setInt( cint32 n, cint64 v );
    • LongArray extends Int64Array
    • DoubleArray extends Int64Array
