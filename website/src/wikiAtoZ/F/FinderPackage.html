AsynchronousIO

Article contributed by GilbertHerschberger (13 March 2000).

Introduction

A finder package is distributed with the SmartAPI - Version 2a product. As I write this, it's full name is org.jos.finder1a.

Finding a file

To find directories, your class must implement DirectoryListener. To find files, your class must implement FileListener. To find both directories and files, your class must implement both DirectoryListener and FileListener.

A basic finder searches recursively through a directory tree when the show property is set to "all".

A finder is a digital resource. Create a finder with the x-finder:file: scheme and the SmartAPI. Plug your class into the finder and run it.

public class Demo
	 implements FileListener {
  public void example() {
	 String s = "x-finder:file:?pattern=*.java&show=all";
	 URI uri = new URI( s );
	 Finder f = (Finder) uri.getObject();
	 if ( f == null ) {
		return;
	 }
	 f.setListener( this );
	 f.run();
  }
  public void onFile( String v ) {
	 // insert custom code here.
  }
  public void onResult( int v ) {
	 // insert custom code here.
  }
}

Asynchronous

Unlike java.io.File, this model uses asynchonous I/O. The finder notifies the listener -- immediately -- whenever it finds a file. The finder notified the listener -- immediately -- when it comes to the end of the file list. A listener interface is used to define a well-known callback function. A finder calls this asynchonously, whenever it finds a file.

Polling is not required. A listener does not poll the finder to check if it has found any more files.

Find application

Version 2 of the Find application uses the finder package to search for files. To start the Find application, use this location:

run:program:org.jos.gchii.find2a.Find
