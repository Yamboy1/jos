This extends the discussion of Factory and FactoryExample2.

Take the factory to the next level: the common URL factory. The common URL factory provides methods to add any other factory.

It is important for a common URL factory to implement the interface of all factories in the catagory.

Here is a common component factory with URL support.

public class CommonComponentFactory
	 implements ComponentFactory {
  public CommonComponentFactory() {
  }
  public void addComponentFactoryByName( String v ) {
	 :
  }
  public void addComponentFactory( ComponentFactory v ) {
	 int n = manager.getCount();
	 manager.setItem( n, v );
  }
  public void removeComponentFactory( ComponentFactory v ) {
	 :
  }
  public boolean acceptsURL( String url ) {
	 int iMax = manager.getCount();
	 for ( int i = 0; i < iMax; i++ ) {
		ComponentFactory factory = manager.getItem( i );
		if ( factory.acceptsURL( url ) ) {
		  return true;
		}
	 }
	 return false;
  }
  public Component getComponent( String url ) {
	 int iMax = manager.getCount();
	 for ( int i = 0; i < iMax; i++ ) {
		ComponentFactory factory = manager.getItem( i );
		Component c = factory.getComponent( url );
		if ( c != null ) {
		  return c;
		}
	 }
	 return null;
  }

  private ComponentFactoryManager manager =
		new ComponentFactoryManager();
}

A common component factory must support a well-known (and predictable) method to select other factories. The above factory uses registered factories on a first-come, first-serve method. Each factory is used to created a component in the order in which it was registered.

Yes, it is possible to have two URLs from two factories create the same component.

Yes, it is possible to have two factories create two entirely different components when given the same URL.

Yes, it is possible to cascade common factories so that one common factory is registered with another.

Yes, the addComponentFactoryByName() method gives us a factory of factories, because the "name" of a factory is a URL.
