Introduction

A Factory has a few requirements. The first requirement is nicely done in Java.

Item 1

Objects created by a factory set must have something in common.

The Java interface is exactly what we need for something in common.

In the editor factory, FactoryExample, the factory returns a Panel, not a specific editor. SimpleEditorPanelFactory returns a java.awt.Panel, not a SimpleEditorPanel.

In the component factory, FactoryExample2, the factory returns a java.awt.Component.

Factories should be designed in such a way to use an existing build-in Java interface or class, if possible.

Otherwise, factories should be designed around a custom interface. Any factory should return an object which implements a custom interface.

Non-alternative

Factories must create objects with something in common. In the worst case scenario, a factory in Java would return an object, java.lang.Object. It would defeat the whole purpose of a factory to have each requestor go through trying to identify the object's specific class after it was created by a factory.
