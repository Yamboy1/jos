ForkAPIPages

Article contributed by GilbertHerschberger (27 August 2002).

Introduction

The complexity of a classpath should be encapsulated in a classpath bean.

Example

The following example illustrates how to use the classpath bean. 
  public void example() {
    ClasspathBean b = new ClasspathBean();
    b.addDirectory( "directory-1" );
    b.addDirectory( "directory-2" );
    b.addDirectory( "directory-3" );
    :
    System.out.println( b.toString() );
  }
This example prints "directory-1:directory-2:directory-3" on Linux and "directory-1;directory-2;directory-3" on Microsoft Windows.

Bean

The following methods are part of the classpath bean.
  • public void clear() This method clears a list of directories. A new or clear'd classpath is equal to "".
  • public int getCount() This method returns the number of directories in a classpath.
  • public String getDirectory( int index ) This method returns the nth directory.
  • public Enumeration getList() This method returns an enumeration of the directory list.
  • public void addDirectory( String v ) This method appends a directory. This may increment count. Duplicate directories are eliminated.
  • public String toString() This method returns a classpath as a single string. A platform-specific separator should be used to separate directories.
This bean encapsulates the complexity of parsing a classpath from a string. The parseClasspath() method might return an instance of the classpath bean.
  • public static ClasspathBean parseClasspath( String v )
The bean has a default constructor.
  • public ClasspathBean()
One classpath can be appended to another. Duplicate directories are be eliminated.

This bean can be used outside the scope of the Fork API and should be implemented in a separate package.

Changes to Fork API

With a classpath bean, the Fork API can be simplified.
  • public Classpath getClasspath() This method gets the classpath property.
  • public void setClasspath( Classpath v ) This method sets the classpath property.
To append a directory to classpath, use getClasspath().addDirectory().
