Introduction

This extends the discussion of Factory and FactoryExample.

In the previous example, a factory was created for each of the individual editors. What happens when you have hundreds or thousands of editors? Having one factory per class breaks down.

The factory is extended with the idea of a uniform resource locator (or URL). With the URL, one factory supports many classes.

Here is a component factory with URL support. 
public interface ComponentFactory {
  public boolean acceptsURL( String url );
  public Component getComponent( String url );
}

Here is a program that uses an AWT component factory: 
public class Example {
  public static void main( String[] args ) {
	 String cn = "x-component:awt:Button?OK";
	 ComponentFactory factory =
		  new AWTComponentFactory();
	 Component c = factory.getComponent( cn );
	 :
  }
}

Here is an AWT component factory: 
public class AWTComponentFactory
	 extends ComponentFactory {
  public AWTComponentFactory() {
  }
  public boolean acceptsURL( String url ) {
	 if ( url.startsWith( "x-component:awt:" ) ) {
		return true;
	 }
  }
  public Component getComponent( String url ) {
	 if ( !acceptsURL( url ) {
		return null;
	 }
	 if ( url.endsWith( "Button?OK" ) ) {
		return new Button( "OK" );
	 }
  }
}
