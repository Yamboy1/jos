ForkAPIPages

Article contributed by GilbertHerschberger (10 June 2002).

Introduction

This API enables a Java program to launch a new instance of the current application, with options to modify its parameters. It provides a mechanism to launch a Java application in a platform-independent way.

Package

A well-known BytecodeProcess interface and BytecodeProcessFactory class are defined in the org.jos.fork1a package.

How to clone an application

The simplest form of launching a new application is by running a default bytecode process object. The following example illustrates how to do this: 
  public void example() {
    BytecodeProcess p = BytecodeProcessFactory.getBytecodeProcess();
    p.run();
  }

How to specify a directory

The current directory can be an important attribute of a Java application. It is supported through the user.dir system property. The following example illustrates how to start a new instance of the current application in a new directory: 
  public void example() {
    BytecodeProcess p = BytecodeProcessFactory.getBytecodeProcess();
    p.setDirectory( _some-directory_ );
    p.run();
  }
The directory property of a BytecodeProcess object becomes the user.dir system property of the target application.
