AboutSmartAPIUtilities

Article contributed by GilbertHerschberger (3 January 2001).

Introduction

The Finder Browser utility is part of Smart API Utilities. It enables you to experiment with the x-finder: scheme. It accepts a finder URI and, if possible, displays its directories, files and result code.

  • View source

Location

To start this utility, enter the following location. It works with both the UniversalBrowser and Program Browser utility.

run:program:org.jos.gchii.finderbrowser1a.FinderBrowser

Mechanism

File Finder and Zip Finder controllers plug into the Finder Factory. In turn, the Finder Factory plugs into the Object Factory. In turn, the Object Factory plugs into the Smart API.

When given a location, this utility passes it to the Smart API. The Smart API passes it to the Object Factory. In turn, the Object Factory passes it to the Finder Factory. In turn, the Finder Factory passes it to the File Finder or Zip Finder controller. If a location is a request for a file-based finder, the File Finder controller returns a File Finder, of the org.jos.gchii.filefinder3a package. If a location is a request for an archive-based finder, the Zip Finder controller returns a Zip Finder, of the org.jos.gchii.zipfinder1c package.

Other finders are possible, and encouraged. Each finder controller should implement a unique sub-scheme.
