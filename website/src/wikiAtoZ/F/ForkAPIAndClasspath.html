ForkAPIPages

Introduction

Let's take another look at the classpath property of a bytecode process.
  • This property is expected to be more platform independent within the Fork API.
  • There is some redundency. This property is available through two methods. The setClasspath() method and the setSystemProperty() method can set the same java.class.path property. While the setClasspath() method is platform-independent, the setSystemProperty() method is not.

Example

Here is an example of how a program can set the classpath property using a BytecodeProcess interface. 
  public void example() {
    URI uri = new URI( "x-fork:" );
    BytecodeProcess p = (BytecodeProcess) uri.getObject();
    if ( p == null ) {
      return;
    }

    String[] list = {
        "./lib/classes.zip",
        ".",
        "./lib/cjos-smartapi4a.jar",
        };
    p.setClasspath( list );
    :
  }

Signature

This is the signature of the setClasspath() method. 
  public abstract void setClasspath( String[] v );

Possible implementation

Each element of the string array is expected to be a directory. Since each directory is a separate string, the implementation of org.jos.fork1a.BytecodeProcess provides a platform-specific directory separator.

If the platform-specific separator is a colon (:), for example, the following snippet illustrates how an implementation can set the java.class.path property. 

  public void setClasspath( String[] v ) {
    int iMax = v.length;
    StringBuffer b = new StringBuffer();
    b.append( v[ 0 ];
    for ( int i = 1; i < iMax; i++ ) {
      b.append( ":" );
      b.append( v[ i ] );
    }
    setSystemProperty( "java.class.path", b.toString() );
  }
The getClasspath() method must return a string array. It gets the java.class.path system property. It separates this property using a platform-specific directory separator. 
  public abstract String[] getClasspath();

Appending to classpath

A directory can be appended to the existing classpath using the appendClasspath() method. Here is a possible implementation. 
  public void appendClasspath( String v ) {
    String s = getSystemProperty( "java.class.path" );
    setSystemProperty( s + ":" + v );
  }
