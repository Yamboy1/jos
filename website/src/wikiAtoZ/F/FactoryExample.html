Introduction

A Factory is a useful technique for creating a new object at runtime. Here is an example of using this with Versioning.

Imagine we have these three packages, each with its own customEditor class.

  • org.jos.editor1a
  • org.jos.editor1b
  • org.jos.editor1c

How do you load each editor?

A editor factory interface would look something like this: 
public interface EditorFactory {
  public Panel getEditor();
}

Here is a program which creates any compatible editor, based on a command line option: 
public class Example {
  public static void main( String[] args ) {
	 String cn = args[ 0 ] + "Factory";
	 EditorFactory factory = CommonEditorFactory.getFactory( cn );
	 Panel p = factory.getEditor();
:
  }
}

Here is the first factory: 
package org.jos.editor1a;

public class SimpleEditorFactory
	 implements EditorFactory {

	 return new SimpleEditorPanel();

An central editor factory can create a dynamic object and get an editor, like this: 
public class CommonEditorFactory 
	 implements EditorFactory {
  public static EditorFactory getFactory( String cn ) {
	 Class c = Class.forName( cn );
	 Object o = c.newInstance();
	 if ( !( o instanceof EditorFactory ) ) {
		return null;
	 }
	 return (EditorFactory) o;
  }

  public Panel getEditor() {
	 // *A! ask user to select from a list of editors.
	 String cn = "org.jos.editor1a.SimpleEditorFactory";
	 EditorFactory f = getFactory( cn ).
	 return f.getEditor();
  }
}
