SmartApiPages

Article contributed by GilbertHerschberger (17 August 2003).

Introduction

'CjOS Smart API' ( SmartApi ) is a multiple stage object factory. JNDI is the Java Naming and Directory Interface API ( JNDI ). This article illustrates the interaction of these two object factories.

Data source

Both of these object factories provide a 'data source' concept as a mechanism for the Java Database Connectivity API ( JDBC ). A data source provides a level of indirection between an application and its database connection. This level of indirection enables your application to use a well-known data source without knowing the specific details of a JDBC driver and its URL. It also enables your application to increase its efficiency by using a pool of database connections.

More about JNDI

The JNDI API is optional. It is not required. Still, there are some applications which would benefit from a data source and connection pool but do not need JNDI.

jdbc:datasource: scheme

The Data Source Controller is a plug-in to the Smart API. By default, it implements the jdbc:datasource: scheme, especially when JNDI is not available.

Future

Because the Data Source Controller is plug-in, it can be replaced or supplemented at runtime. A new JNDI-aware controller should be developed which implements the same jdbc:datasource: scheme. When it is available, the new controller should return a connection using JNDI.

The following example illustrates this using a fictional JOSDEMO database.

  1. Your application uses the jdbc:datasource:JOSDEMO location to get a JDBC connection.
  2. When available, the JNDI-aware Data Source Controller creates an initial context and gets the object for java:com/ext/jdbc/JOSDEMO.
  3. Otherwise, the Data Source Controller creates a JDBC connection using its own configuration.

Example

The following source code illustrates how to get a connection: 
  public Connection example( String dsn ) {
    URI uri = new URI( "jdbc:datasource:" + dsn );
    return (Connection) uri.getObject();
  }
Within the JNDI-aware controller, the following source code illustrates how a connection is obtained: 
  public Object getObject( String v ) {
    String dsn = ...
    Context initialContext = new InitialContext();
    try {
      Context context = initialContext.getContext( "java:/com/ext/jdbc" );
      return (Connection) context.getObject( dsn );
    }
    catch( Throwable e ) {
      return e;
    }
  }
By convention, a JDBC connection is com/ext/jdbc/something. Of course, this is not always the case.
