Introduction

The connection from one machine to another is an important part of locating a digital resource. The location of a digital resource includes where it is stored or where it is running. A uniform resource locator (URL) is a standard mechanism to locate digital resources when there are one or more machines.

Example

This example demonstrates how to use the socket: subscheme.

  public void example() {
	 URI uri = new URI( "io:socket://quest:23/" );
	 IOStream ios = (IOStream) uri.getObject();
	 if ( ios == null ) {
		return;
	 }
	 :
  }

Socket

The socket is one of the fundamental concepts in a URL mechanism. A socket is form of I/O stream.

Socket controller

The socket controller is a plug-in for SmartAPI. The socket: subscheme always returns an instance of org.jos.program2a.IOStream, or null.

io:socket://<host>:<port>/

The general form of a host connection is something like this:

<scheme> ':'
  [ <subscheme> ':' ... ]
  [ '//' 
	 [ <user> [ ':' <password> ] '@' ]
	 <host> [ ':' <port> ] '/' ]
  [ <uri> ]
  [ '?' <querystring> ]
  [ '#' <fragment> ]

The user, password, uri options cannot apply to the socket: subscheme. The host and port options are always required. The scheme is io:. The subscheme is socket:. Here is the result:

'io:socket:'
  '//' <host> [ ':' <port> ] ]
  [ '?' <querystring> ]
  [ '#' <fragment> ]

Socket scheme

Unlike the socket: subscheme, the socket: scheme always returns an instance of java.net.Socket or null.

socket:
  '//' <host> [ ':' <port> ] ]
  [ '?' <querystring> ]
  [ '#' <fragment> ]

The controller for the socket: subscheme actually uses the socket: scheme.

  public Object getObject( v ) {
	 String scheme = "io:";
	 String subscheme = "socket:";

	 if ( !v.startsWith( scheme + subscheme ) ) {
		return null;
	 }

	 URI uri = new URI( v.substring( scheme.length() );
	 Socket socket = (Socket) uri.getObject();
	 if ( socket == null ) {
		return null;
	 }

	 return new SocketIOStream( socket );
  }

Properties

Additional properties can be set using the query string. The following properties may be supported by your socket controller.

  • nodelay

    No delay is a boolean property. When used, the no delay property is set by a socket controller. For more information, see also the setTcpNoDelay() method of the java.net.Socket class.

  • linger

    Linger is an integer property. When used, the linger property is set by a socket controller. For more information, see also the setSoLinger() method of the java.net.Socket. class.

  • timeout

    Timeout is an integer property. When used, the timeout property is set by a socket controller. For more information, see also the setSoTimeout() method of the java.net.Socket class.
