SmartProtocolPages; RegistryExamples

Article contributed by GilbertHerschberger (21 September 1999).

Introduction

The SmartRegistry uses a registry protocol. A client agent, especially a Registry Browser, uses this protocol to load registries from a given URI. A registry factory loads registries and only registries.

Protocol

The following table contains a registry protocol and its well-known interfaces.

Protocol Interface Description
x-registry: 
null
Registry
PersistentRegistry

This protocol expects a registry name. The registry factory is responsible for loading a registry. It encapsulates the process of reading a registry file, creating a proxy to an existing registry and connecting to a registry across the network.

A client agent must handle a registry path, literally a fragment string. The path used to set an iterator to a specific registry node. Since the whole registry must loaded to get part of it, the factory ignores a fragment string.

Subprotocols

The following table contains some of the registry subprotocols and their well-known interfaces. When a file extension is used, a subprotocol is optional. Likewise, when a subprotocol is used, a file extension is optional.

Subprotocol Interface or Extension Description 
conf:
.conf

null
Registry

This protocol loads a file using the configuration file format, taken from UNIX(r) and Linux(r). The rest of the URI is passed directly to an input stream factory.

 
ini:
.ini

null
Registry

This subprotocol loads a file using the initialization file format, a Window(tm) form of configuration file. The rest of the URI is passed directly to an input stream factory.

 
property:
.properties

null
Registry

This subprotocol loads a file using the property file format, from the Java(tm) platform. The rest of the URI is passed directly to an input stream factory.

 
set:
.set

null
Registry

This subprotocol loads a file using the "set" file format, a over-simplified form of initialization, configuration or property file. The rest of the URI is passed directly to an input stream factory.

 
persistent:

null
PersistentRegistry

This is a special subprotocol. It turns any Registry -- from other subprotocols -- into a persistent registry (PersistentRegistry), which is a partial implementation of RegistryInterator. Every change to a persistent registry is saved, implicitely and automatically. The rest of the URI must start with another registry subprotocol.

 
new:

null
Registry

This is a special subprotocol. It returns a new registry. The implementation of the registry interface is determined by the current configuration of the registry factory.
