Introduction

This is your introduction to the Signature services. This is something of a proposal. This is something of a strategic plan. If you'd like to volunteer, you can build a service with your Signature on it.

Similar

All services are similar. There are a lot of things in common. A lot of work has been done to create reusable components for TCP and UDP services.

Smart API

The SmartAPI provides all of the services with access to all kinds of digital resources. For example, the TCP service is available through a universal resource locator (or URI), like this:

x-service:tcp:?port=80&count=5

A custom TCP service extends BaseTCPService, like this:

public class <EM>Custom</EM>Service
	 <STRONG>extends BaseTCPService</STRONG> {
  public <EM>Custom</EM>Service() {
  }
  <STRONG>public void onSocket( Socket v )</STRONG> {
  :
  }
}

Plan

Here is the strategic plan. A member of the JOS project goes throught the following steps:

  • pick your favorite UDP or TCP protocol;

  • create a package with your name on it;

    This is your signature.

  • extend BaseUDPService or BaseTCPService;

    These general purpose classes will be written and distributed to everyone.

  • publish the code for others to run (test and debug);

  • After a service has been fully tested and debugged, we'll discuss how a service might fit into the org.jos namespace.

By extending BaseXXXService, all of the services will be compatible with the UniversalBrowser. By keeping instance variables non-static, multiple instances of a service can be created to listen on different ports within the same virtual machine. By using the C-style enumeration for Java, constants required by each protocol might be reusable in other services.
