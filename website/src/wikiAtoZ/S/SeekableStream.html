Introduction

A seek-able stream is able to set the seek address of the next read and next write. Neither java.io.InputStream nor java.io.OutputStream are seek-able. It is a dilemma to extend these streams to match the more robust model of java.io.RandomAccessFile.

Streams created by a java.io.Socket are not seek-able either. A protocol must exist on top of a write-only stream to pass seek information to a peer.

About SeekableInputStream class

This class is an extension of java.io.InputStream it adds the ability to seek to a position within the input stream. Like a random-access stream, this stream is (probably) infinitely re-readable.

This class implements the Seekable interface.

About SeekableOutputStream class

This class is an extension of java.io.OutputStream it adds the ability to seek to a position within the output stream. Like a random-access stream, this stream is (probably) infinitely re-writeable.

This class implements the SeekableStream interface.

About SeekableStream interface

Some streams are seek-able, and some are not. When a stream is seek-able, it should implement the SeekableStream interface. (The java.io.RandomAccessFile is seek-able; but, it does not yet implement this interface.)

There are two key methods in the seek-able interface:

  • getPosition(); and
  • void setPosition();

Here is the source code for this interface:

public interface SeekableStream {
  /**
	* Gets Position property.
	*/
  public abstract long getPosition()
		throws IOException;

  /**
	* Sets Position property.
	*/
  public abstract void setPosition( long v )
		throws IOException;

  /**
	* Sets Position property.
	*/
  public abstract void setPosition( long v, SeekOption o )
		throws IOException;
}

About SeekOption interface

There are three seek options: BEGIN, CURRENT and END. This is wrapped up in an object-oriented enumeration, something like this:

public class SeekOption {
  public static final SeekOption BEGIN = new SeekOption( 0 );
  public static final SeekOption CURRENT = new SeekOption( 1 );
  public static final SeekOption END = new SeekOption( 2 );

  private SeekOption( int v ) {
	 id = v;
  }
  :
  private int id;
}

Related to file subsystem

Seek-able streams may be important to the needed functionality of a file subsystem. A random-access file is hardly adequate for seek-able streams that are not local files.

With a seek-able stream socket, a remote hard drive can implement the same interface as a local hard drive. While programs run on one machine, some of their disk access can be redirected to another. Therefore, a light-weight machine can use the hard drive of a heavy-weight machine.

In the serverfarm-in-a-box architecture, every machine inside the serverfarm must be on a separate subnet. All the machines connected to this subnet behave as if they were part of one big machine.

With the processor-independent nature of bytecode, processors can be mixed and matched in a serverfarm, and should be.

About SeekableIOStream interface

Like an I/O stream interface, this interface manages a pair of streams, one input stream and one output stream.

public interface SeekableIOStream {
  public abstract SeekableInputStream getInputStream()
		throws IOException;
  public abstract SeekableOutputStream getOutputStream()
		throws IOException;
  public void close()
		throws IOException;
}

Summary

A seek-able stream should have always been a well-known interface. For maximum backward compatibility, it will be written into a package of its own.
