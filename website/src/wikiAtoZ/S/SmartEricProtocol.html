Article contributed by GilbertHerschberger (31 July 2000).

Introduction

One virtual machine can invoke another. Or inside an MPCL-compatible virtual machine, one bytecode process can invoke another. The eric tool provides a vm-independent way for one virtual machine to invoke another. It is easier to invoke another virtual machine without getting into the vm-specific problems.

The eric: protocol provides a pure Java model for the eric tool. In theory, it is possible to achieve the benefits of the eric tool by invoking the java tool.

eric:?location=x
eric:new?location=x
eric:desktop.eric
eric:file:desktop.eric
eric:in:file:desktop.eric

Description

The eric: protocol is plug-in for the SmartAPI.

eric:<EM>file-spec</EM>
where file-spec uses a subprotocol, such as one of the following:

  • file:
  • ftp:
  • http:
  • etc.

Since the http: protocol is supported, ERIC shortcut files can be stored on an Internet/intranet HTTP service.

Step by step

The following example returns an instance of ERICShortcut and prints its properties.

  public void example() {
	 URI uri = new URI( "eric:file:desktop.eric" );
	 ERICShortcut sc = (ERICShortcut) uri.getObject();
	 if ( sc != null ) {
		println( "path=" + sc.getPath() );
		println( "shell=" + sc.getShell() );
		println( "location=" + sc.getLocation() );
	 }
  }

ERICController implements the eric: protocol. Here are the steps the uses to return an instance of ERICShortcut.

  1. Creates an instance of BasicERICShortcut,
  2. Reads file-spec into a data Registry object, using the x-registry: protocol,
  3. Copies the appropriate properties from Registry to BasicERICShortcut,
  4. Returns BasicERICShortcut to the calling method.

Shortcut file

An ERIC shortcut file is like a properties file. For more information, see also ERICProject. It can contain one or more of the following properties:

  • path
  • shell
  • location

The ERICShortcut interface maps these properties.

public interface ERICShortcut {
  public abstract String getPath();
  public abstract void setPath( String v );
  public abstract String getShell();
  public abstract void setShell( String v );
  public abstract String getLocation();
  public abstract void setLocation( String v );
}

The BasicERICShortcut class implements the ERICShortcut interface.

public class BasicERICShortcut 
	 implements ERICShortcut {
  public BasicERICShortcut() {
  }
  public String getPath() {
	 return path;
  }
  public void setPath( String v ) {
	 path = v;
  }
  public String getShell() {
	 return shell;
  }
  public void setShell( String v ) {
	 shell = v;
  }
  public String getLocation() {
	 return location;
  }
  public void setLocation( String v ) {
	 location = v;
  }
  private String path;
  private String shell;
  private String location;
}

Compatibility

An instance of ERICShortcut is compatible with both the eric and java tools. A new virtual machine is created with the java tool and the eric tool. A new primordial class loader should be created within an MPCL-compatible virtual machine.

About eric tool

The eric tool automatically figures out the correct CLASSPATH for a new virtual machine. With an ERICShortcut object, it is easy to invoke the eric tool.

  public void example( ERICShortcut v ) {
	 StringBuffer cmd = new StringBuffer();
	 cmd.append( "run:os:eric.exe" );
	 cmd.append( " path=" + encode( v.getPath() ) );
	 cmd.append( " shell=" + encode( v.getShell() ) );
	 cmd.append( " location=" + encode( v.getLocation() ) );

	 URI uri = new URI( cmd.toString );
	 Program p = (Program) uri.getObject();
	 if ( p != null ) {
		p.run();
	 }
  }

About java tool

Although it is more difficult, it is possible to invoke an ERIC shortcut with the java tool. Both getJavaTool() and getClasspath() is dependent on custom configuration.

  public void example( ERICShortcut v ) {
	 StringBuffer cmd = new StringBuffer();
	 cmd.append( "run:os:" + getJavaTool() );
	 cmd.append( " -classpath=" + getClasspath() );
	 cmd.append( " org.jos.gchii.eric1c.ERICShell" );
	 cmd.append( " cache=./cache" );
	 cmd.append( "&path=" + encode( v.getPath() );
	 cmd.append( "&shell=" + encode( v.getShell() ) );
	 cmd.append( "&location=" + encode( v.getLocation() ) );

	 URI uri = new URI( cmd.toString );
	 Program p = (Program) uri.getObject();
	 if ( p != null ) {
		p.run();
	 }
  }

eric: subprotocol

The run: protocol has a eric: subprotocol. The eric: subprotocol always returns a Program or null.

  public void example() {
	 URI uri = new URI( "run:eric:file:desktop.eric" );
	 Program p = (Program) uri.getObject();
	 if ( p != null ) {
		p.run();
	 }
  }
