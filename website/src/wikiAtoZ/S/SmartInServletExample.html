Introduction

The servlet: subscheme is proposed. It will part of the in: scheme. It will always return an instance of java.io.InputStream (or null). It connects to a servlet environment either directly or through the HTTP network protocol.

This subscheme is not to be confused with the servlet: scheme, which always returns an instance of javax.servlet.Servlet or null. For more information on the servlet: scheme, see also SmartServletExample .

Example location

In this example, the org.jos.demo.DemoServlet class extends javax.servlet.http.HttpServlet. With MiniServer running on port 8080, the servlet is usually invoked in an HTTP browser, like this: 
http://localhost:8080/servlet/org.jos.demo.DemoServlet
Once the servlet: subscheme is installed as a SmartAPI plug-in, the same servlet can be invoked like this: 
in:servlet:org.jos.demo.DemoServlet

Example program

The following sample method reads the page generated by the DemoServlet class. 
public class Example 
    implements Program {
  public void run() {
    String cn = "org.jos.demo.DemoServlet";
    URI uri = new URI( "in:data:servlet:" + cn );
    DataInputStream input = (DataInputStream) uri.getObject();
    if ( input == null ) {
      return;
    }

    onInput( input );

    try {
      input.close();
    }
    catch( IOException e ) {
    }
  }

  protected void onInput( DataInputStream input ) {
    String line;
    for (;;) {
      try {
        line = input.readLine();
      }
      catch( IOException e ) {
        return;
      }

      if ( line == null ) {
        break;
      }

      println( line );
    }
  }

  protected void println( String v ) {
    System.out.println( v );
  }
The servlet: scheme returns an instance of java.io.InputStream. It is translated into http://localhost:8080/servlet/ or another host and/or port.

The data: scheme turns an instance of java.io.InputStream into an instance of java.io.DataInputStream.
