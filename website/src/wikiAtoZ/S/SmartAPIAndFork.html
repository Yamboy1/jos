ForkAPIPages; SmartAPIPages

Article contributed by GilbertHerschberger (5 June 2002).

Introduction

The Fork API is defined in the org.jos.fork1a package. A factory is needed to create the platform-dependent instance of its well-known BytecodeProcess interface. But the implementation of the factory itself should not be platform-dependent.

Implementation

A BasicBytecodeProcess class may be implemented in the org.jos.gchii.smartfork1a package. The proposed x-fork: scheme should return a copy of the current bytecode process.

Plan

The following class may be implemented in the org.jos.fork1a package so that a Java application can obtain a copy of the current bytecode process. 
public class BytecodeProcessTool {
  public static BytecodeProcess getBytecodeProcess() {
    URI uri = new URI( "x-fork:" );
    return (BytecodeProcess) uri.getObject();
  }
  private BytecodeProcessTool() {
  }
}
