SmartAPIPages

Article contributed by GilbertHerschberger (9 December 1999).

Introduction

While Smart API and MIME are similar, they are not the same.

Example from Java class library

Just as java.net.URL is used to get the content of a document, the client agent is responsible for presenting a document.

How Smart API works

A string is passed to each controller in the Smart API. A controller determines if it can construct an object from its interpretation of the string. The object is passed back to the requestor, without considering a presentation.

public interface Controller {
  public boolean acceptsURI( String v );
  public Object getObject( String v );
}

How client agent should work

Once you have an object, another mechanism must take over inside a client agent. A universal client agent is responsible for rendering all kinds of objects and presenting them. Inside a client agent, a viewer might be loaded and initialized once at the beginning of a session. This is a mechanism to find a viewer that can accept an object: 

public interface PlugIn {
  public boolean acceptsObject( Object v );
  public Object getObject();
  public void setObject( Object v );
}

On the other hand, a universal client agent might support all kinds of plug ins, loading/unloading plug ins as needed. This kind of client agent requires a plug-in factory.

public interface PlugInFactory {
  public boolean acceptsObject( Object v );
  public PlugIn getPlugIn( Object v );
}

Once a plug-in has been created, a universal client agent uses its acceptObject() method to use it again. When none of the loaded plug ins accept an object is only time the PlugInFactory get involved.

About TELNET client

Personally, I don't want my browser to reuse my TELNET client. I would rather have a new window created with each invocation of a telnet: location. This means that some plug-ins are reusable and some are not. A plug-in factory returns a plug-in with the object already set. When a plug-in is not reusable, it accepts no more objects.
