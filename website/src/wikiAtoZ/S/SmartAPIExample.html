Article contributed by GilbertHerschberger (3 September 1999).

Introduction

A well-known interface or well-known base class is typically required to build a meaningful SmartAPI. While you could build a controller to return instances of java.lang.Object, it is more important to build a controller for implementations of a well-known interface and classes derived from a well-known base class.

Runnable

Limping upon the this example again, java.lang.Runnable is a well-known interface. It is part of the core Java class libraries. You can have many packages. You can have many classes in each package. Each class has an opportunity to implement the java.lang.Runnable interface.

If you were to build a RunnableController for a runnable: protocol, you would be doing everyone a favor. They could load all things runnable with a simple URI.

  public void example() {
	 String s = "runnable:org.jos.demo.DemoRunnable";
	 Runnable r = (Runnable) new URI( s ).getObject();
	 :
  }

The purpose of a specialized controller is implement a URI scheme, and make sure an object is runnable. It must filter out all those classes that do not implement the java.lang.Runnable interface.

A specialized controller might be able to create runnable objects from any package, not just a few. It can select objects, as we've shown here, by class name. More importantly, it can select objects with less precision (by alias and/or properties).
