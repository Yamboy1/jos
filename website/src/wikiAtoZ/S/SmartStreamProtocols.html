Introduction

A stream factory primarily creates input and output streams. This article discusses some of the protocols (schemes) supported by a stream factory.

The smart stream factory is compatible with the smart object factory. Typically, the smart object factory is configured to use the stream factory.

Digital resource

A stream is another digital resource. A stream has a uniform resource identifier (URI). The stream factory creates objects from a URI.

Subprotocols

The following subprotocols are used within the smart stream factory. The smart stream factory supports subprotocol chaining, where more than one non-terminal subprotocols can be used. Where appropriate, both in: and out: protocols define similar subprotocols.

Non-Filter Subprotocols
Subprotocol Interface Description
file: 
null
java.io.InputStream
java.io.OutputStream

The smart stream factory does not use java.net.URL for the file: protocol. It uses a platform-independent path and supports a default directory.

The input stream factory returns an instance of java.io.FileInputStream, which implements the java.io.InputStream interface.

The output stream factory returns an instance of java.io.FileOutputStream, which implements the java.io.OutputStream interface.
http: 
null
java.io.InputStream
java.io.OutputStream

The smart stream factory first gets an instance of java.net.URL, and then uses its getInputStream() or getOutputStream() method.

The input stream factory uses getInputStream() to return an instance of java.io.InputStream.

The output stream factory uses getOutputStream() to return an instance of java.io.OutputStream.

This is a filter (or non-terminal) subprotocol. It decompresses an input stream or compresses an output stream using the GZIP file format. This subprotocol requires the Java 1 Platform (or higher).

The output stream factory returns an instance of java.util.zip.GZIPOutputStream, which implements the java.io.OutputStream interface.

This is a filter (or non-terminal) subprotocol. It marshalls an input stream or unmarshalls an output stream using the Java 1 Platform serialization. This subprotocol requires the Java 1 Platform (or higher).

The input stream factory returns an instance of java.io.ObjectInputStream.

The output stream factory returns an instance of java.io.ObjectOutputStream.

null
InputStream

Proposed This is a non-terminal subprotocol. This subprotocol returns the results of a servlet request as an input stream. The input stream factory replaces servlet: with http://localhost:8080/servlet/ by default. Another host and/or another port can be configured.

This is not to be confused with the servlet: protocol, which returns an instance of javax.servlet.Servlet. When a servlet runner is plugged directly into the SmartAPI, a protocol stack is not required.

For more information, see also SmartInServletExample .
