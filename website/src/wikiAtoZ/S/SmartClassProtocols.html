Introduction

A class loader is used to create class objects. While a class loader is adequate, the smart class factory has the potential to be much better. For example, the smart class factory might be able to load a class from an archive file on another computer. A controller is created for each kind of bytecode storage mechanism. A class factory creates classes and only classes.

The smart connection factory is future oriented. Anyone can create a bytecode storage mechanism and add it to their smart class factory. It builds a foundation for the bytecode factory and bytecode network service (or application server).

Example

The following example illustrates the simplicity using the smart class factory to create a class:

  public void example() {
	 URI uri = new URI( "class:java.awt.Button" );
	 Class c = (Class) uri.getObject();
	 if ( c == null ) {
		return;
	 }
  }
