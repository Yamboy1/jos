Introduction

It should be no surprise. The servlet browser is similar to the applet browser, just a servlets are similar to applets. What is a servlet browser? And how can I build one?

Definition

An servlet browser is a browser that specializes in displaying servlets. Unlike servletrunner, which requires a separate HTML browser, a servlet browser is all-in-one. It has a browser's location field as direct input into a host servlet environment. When you type in a location and press the Run button, a servlet browser displays servlet output.

HTTP GET

A HTTP GET request is the most frequently used request and easiest to implement. The following steps are performed without a pause, without a timeout.

  1. Context

    The servlet browser or one of its components must implement the methods of a servlet context.

  2. Configure

    After a servlet is loaded, it is configured by setting up a custom servlet config and calling its init() method. The extra parameters for a servlet are passed through a servlet config.

  3. Request

    A request object is created. The request URL comes from the location field.

  4. Response

    A response object is created. The output stream is connected to the servlet browser's output window.

  5. Service

    The service() method is called with a request and a response. The servlet reads its request and sends output through the response to the output window.

  6. Destroy

    Immediately following a service request, a servlet is destroyed. The servlet's destroy method is called and the servlet is unloaded.

Output options

The output window can contain a plain-text console or an HTML console. With a plain-text console, the output from all servlets is displayed in plain-text. With an HTML console, the output from an HTML servlet is rendered.

Parameters

You can experiment with servlet parameters. Parameters are passed to a servlet through the request object.

servlet:org.jos.demo.DemoServlet?name-1=value-1

Clearing the display

Each press of the Run button clears the output window. Again, a servlet is created, used and destroyed with press of the Run button.

Building your browser

You can assemble a servlet browser from the Java AWT (or Swing) and release 1b of the SmartAPI.

  • servlet panel

    A servlet panel is a specialed panel for displaying running servlets and their output properly.

When the Run button is pressed, the location is parsed to find the name of the servlet class. The servlet class is passed to the smart object factory to create a servlet. The location is passed to the servlet as part of the request. The servlet browser only handles servlets properly.

Future

A servlet browser could be improved by providing more control over the lifetime of a servlet. The creation, initialization and destruction of a servlet should be independent from the Run cycle. A servlet browser should offer a start/stop servlet mechanism.
