SmartAPI; SmartStreamProtocols; SmartAPI4aStreamFactory

Article contributed by GilbertHerschberger (20 July 1999, revised 2 July 2002).

Introduction

While working on the registry API, I created a simple stream factory. I can imagine a better stream factory already.

Java I/O

The Java I/O package leaves a lot to be desired. While there are many different classes in the package, they aren't so easy to use.

Stream factory

You should be able to open a buffered file input stream as a print-writer with the minimal amount of code. Why not use a URL for a file resource? Why not pick the filters for your resource with subprotocols? 
  public void example() {
    String url = "stream:print:buffer:file:/usr/local/example";
    StreamFactory factory = StreamFactory.getFactory();
    PrintWriter writer = (PrintWriter) factory.getStream( url );
  }
So, you don't have to re-write the same code. 
  public void example() {
    String url = "stream:print:buffer:http:/host/download/example";
    StreamFactory factory = StreamFactory.getFactory();
    PrintWriter writer = (PrintWriter) factory.getStream( url );
  }
All of the different streams should be supported, including the RandomAccessFile. Compare that to this, even without the try and catch: 
  public void example() {
    URL url = new URL( "http:/host/download/example" );
    InputStream input = url.getInputStream();
    BufferedWriter buffer = new BufferedWriter( input );
    PrintWriter writer = new PrintWriter( buffer );
  }
During the development of the stream factory, the stream protocol was changed from stream: to in: and out:. In order to get an input stream, you'll need to use the in: protocol. In order to create an output stream, you'll need to use the out: protocol.
  • in: - Creates an object based on an input stream.
  • out: - Creates an object based on an output stream.
For subprotocols, the in: protocol supports
  • buffer: - Inserts a java.io.BufferedInputStream.
  • data: - Inserts a java.io.DataInputStream.
  • file: - Opens a file for input, converting a platform-independent file name to platform-dependent.
  • ftp: - Opens a file, using the FTP protocol. This is a short-cut to url:ftp:.
  • http: - Opens a file, using the HTTP protocol. This is a short-cut to url:http:.
  • token: - Creates a java.io.StreamTokenizer for an input stream. This does not return an instance of java.io.InputStream.
  • url: - Opens an input stream using java.net.URL. The stream factory supports any protocol provided by java.net.URL.
For subprotocols, the out: protocol supports
  • buffer: - Inserts a java.io.BufferedOutputStream.
  • data: - Inserts a java.io.DataOutputStream.
  • file: - Opens a file for output, converting a platform-independent file name to platform-dependent.
  • print: - Inserts a java.io.PrintStream.

URI Example

Here is a simple example for creating a program that reads one text file and writes another. 
  public void example() {
    String source = "in:data:buffer:file:/usr/local/example-in";
    DataInputStream in =
        (DataInputStream) new URI( source ).getObject();
    if ( in == null ) {
      return;
    }

    String target = "out:print:buffer:file:/usr/local/example-out";
    PrintStream out =
        (PrintStream) new URI( target ).getObject();
    if ( out == null ) {
      try {
        in.close();
      }
      catch( IOException e ) {
      }
      return;
    }

    String line;
    for (;;) {
      try {
        line = in.readLine();
      }
      catch( IOException e ) {
        break;
      }
      if ( line == null ) {
        break;
      }

      out.println( line );
    }

    try {
      out.close();
    }
    catch( IOException e ) {
    }

    try {
      in.close();
    }
    catch( IOException e ) {
    }
  }
