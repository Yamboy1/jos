SmartAPIPages

Article contributed by GilbertHerschberger (30 August 2001).

Introduction

Here is an example of an interactive session with the Smart API Service.
  1. Start the Smart API Service. It listens on port 8023 by default.
  2. Start a TELNET client.
  3. Connect to port 8023.
You'll see something like the following message when you connect: 
service: Smart API Service, version x.x.x
package: gchii.tools.smartapiservicexx
login:
Enter your account name and password when prompted. If your account information is current, the following prompt will be displayed: 
smartapi>
Like the UniversalBrowser utility, the Smart API Service is a command-line interface to the SmartAPI. Type any location.
  • When you enter the location of a web page, its contents will be displayed (but as plain text in a TELNET client).
  • When you enter the location of a text file, its contents will be displayed.
  • When you enter the location of a native OS program, a Java application and a JOS program, its standard out (if any) will be redirected to your TELNET client.
And like the Universal Browser utility, the Smart API Service always displays a "Running..." message before it invokes a location and a "Done." message afterward. All text between the "Running..." and "Done." messages are generated by the location.

Example 1

The following location invokes the Smart List program, a program which displays the current configuration of the SmartAPI. 
smartapi> run:program:org.jos.gchii.smartlist1a.SmartListProgram
Running...
Factory: org.jos.gchii.factory1b.Factory
:
Done.
smartapi>

Example 2

The following location invokes the Static Mailman program. See also AboutStaticMailman. 
smartapi> run:program:gchii.tools.staticmailman1a.StaticMailmanProgram
Running...
Static Mailman Program (1a)
site=sometext
path=admin
path=arch
path=apps
:
Result Code 0.
Done.
smartapi>

On a server

You can run the Smart API Service on another machine. The other machine is known as a server.

When used in this configuration, remember that a location is invoked from the perspective of the service, not the client. Some locations display graphics. If so, these graphics will be displayed on the server machine, not the client. Test a location with the Universal Browser utility.

Single virtual machine

The Smart API Service enables you to run many programs in a single virtual machine. Here is how this can be an advantage when you're running network-based software.
  • Control concurrency. By consistently running your program as a service (on a well-known port), you can use simple synchonization to control concurrency. Each instance of a program runs in a separate thread.
  • Remote administration. This service enables you to run any program, both native and bytecode, from a remote location.
  • Sometimes a full HTTP service, with a Java servlet environment, is overkill for your network application. The Smart API Service has a light-weight footprint and configuration, but delivers heavy-weight results.

About the smartapi tool

To start this service on Microsoft Windows,
  1. Click (or double-click) on the smartapi.exe program. The smartapi.ini file contains its configuration. The Smart API Service window is displayed.
  2. If necessary, configure the program. This service can be configured to listen on another port. This service can be configured to start automatically.
  3. Click on the Run button.
To stop this service,
  1. Click the Stop button. Or, click the close (X) button.
