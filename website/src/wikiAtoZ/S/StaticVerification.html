Introduction

I must deal with thousands of custom classes on a daily basis. An inconsistent package name is one of the most annoying things because it cannot be seen with a directory listing. A class looks as if it exists; but, a virtual machine cannot load it. An inconsistent package name is when a class file is stored in the wrong directory.

Tool

I am thinking of a new tool for static bytecode verification. Since something is better than nothing, it is just a start. Static verification can eventually perform more powerful tests. It would use Pure Reflection for C. Given a directory, it would scan for all .class files. It would open each class file and make sure that its actual package was the expected package. It would print the name of every class that has an inconsistent package name.

bcverify d=<EM>some-directory</EM>

Starting with some-directory, this tool would scan for class files. With the show=all option, it would scan subdirectories too. For each class file name, it would open the class file and extract its class name.

rc_HANDLE h = rc_openClassFile( cf );
char *cn = rc_getThisClassName( h );
