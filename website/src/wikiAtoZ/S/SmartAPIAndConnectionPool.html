SmartAPIPages

Article contributed by GilbertHerschberger (27 January 2002).

Introduction

The following subprotocols have been suggested.

Prompt subprotocol

Within the JDBC, some drivers prompt an end-user for a name/password and some don't. For example, the JDBC-ODBC bridge does not provide a prompt, but expects user and password parameters.

To make prompting more consistent across JDBC drivers, the following subprotocol has been proposed:

  • jdbc:prompt:text
To illustrate the point, let's use the following example:
  • jdbc:odbc:JOSDEMO?user=x&password=y
If you would like to prompt for user and password, insert the prompt: subprotocol into the location, like this:
  • jdbc:prompt:odbc:JOSDEMO
The prompt: subprotocol will display a generic dialog box, with both a name and password field. Type in your user name and password and press the OK button. If I type x in the User field and y in the Password field, the prompt: subprotocol will create the following connection:
  • jdbc:odbc:JOSDEMO?user=x&password=y
Other parameters will be passed unchanged. To illustrate, the prompt: subprotocol would accept this location:
  • jdbc:prompt:odbc:JOSDEMO?someparameter=z
and turn it into something like this:
  • jdbc:odbc:JOSDEMO?someparameter=z&user=x&password=y
When a user presses the Cancel button, the prompt: subprotocol returns null. No exception is thrown.

Pool subprotocol

Within JDBC, some connections are automatically pooled by the driver and some are not. For example, the JDBC-ODBC bridge does not provide an automatic connection pool.

In order to open a connection from a built-in connection pool, the following pool subprotocol has been suggested:

  • jdbc:pool:text
The pool: subprotocol will attempt to return a pooled-connection object from a built-in connection pool. A pooled-connection implements the java.sql.Connection interface and uses another connection. When closed, a pooled-connection closes the other connection only if reference count drops to zero.

Combining subprotocols

The prompt: and pool: subprotocols can be used together. When used together, it should not matter which is used first.
  • jdbc:prompt:pool:odbc:JOSDEMO
If I type x into the User field and y into the password field and that creates a connection, jdbc:odbc:JOSDEMO?user=x&password=y is inserted into the connection pool. The connection pool itself may contain connection-holder objects, enabling reference counting.
