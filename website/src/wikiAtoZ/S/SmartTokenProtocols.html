SmartProtocolPages

Article contributed by GilbertHerschberger (29 September 1999).

Introduction

A parser, like SourceServer's Downloader, could uses this protocol to load and tokenize an input stream. A stream tokenizer factory creates stream tokenizers and only stream tokenizers.

Protocol

The following table contains a stream tokenizer protocol and its well-known interface.

Protocol Interface Description 
token:
x-token:

null
java.io.StreamTokenizer

This protocol expects a smart input stream URI. The stream tokenizer factory is responsible for loading an input stream and passing it to the constructor of a stream tokenizer.

This factory handles a query string. The query string is used to pass parameters to a stream tokenizer. The following parameters have been partially implemented:

  • eolIsSignificant

    This parameter is used if it has a value of true or false.

  • commentChar

    The first character of this parameter is used as a comment character.

Example

The following example demonstrates how to build a stream tokenizer directly from the Source Server's main page directly, across the Internet.

  public void example() {
	 URI uri = new URI( "x-token:buffer:" +
		  "http://www.jos.org/source/" +
		  "?eolIsSignificant=true&commentChar=," );
	 StreamTokenizer st = (StreamTokenizer) uri.getObject();
	 if ( st == null ) {
		System.out.println( "Document not found." );
	 }
  }

Limitation

This factory uses a query string for the stream tokenizer. Since the factory use a query string, it cannot be used to completely identify an input stream URI. A future solution might include a URI pipe or embedded URI.
