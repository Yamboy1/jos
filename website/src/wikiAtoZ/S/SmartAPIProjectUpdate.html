Smart API - Project Update

Created: 99.07.30Last Updated: 99.12.21Project Page: SmartAPIPagesMailingList: arch@jos.org

Project Summary:

The purpose of this project is to build an extraordinary toolkit for building application programming interfaces (or APIs). This toolkit supports the techniques recommended for a future-oriented operating system. While an object factory creates an object, a smart object factory creates an object from any API. Object selection is open-ended, supporting both selection by name and selection by attributes. In addition to default plug-ins, this toolkit helps anyone develop (and debug) a plug-in.

21 December 1999

There are far too many new combinations of cool features to list them all. The new desktop browser uses the new AWT-compatible Paintable interface and paint studio components. The deric tool, from ERICProject, uses a byte array controller to grab a class file. The exception tool has a format method to format any exception (Throwable).

Grabber

With a byte array controller, your program can grab a file as a byte array, something like this: 

  public void example1() {
	 URI uri = new URI( "in:byte:file:example.txt" );
	 byte[] buf = (byte[]) uri.getObject();
  }
  public void example2() {
	 URI uri = new URI( "in:byte:http://www.jos.org/example.txt" );
	 byte[] buf = (byte[]) uri.getObject();
  }

Copy tool

Add a whole new dimension to a copy command. Once you have the in: and out: schemes from the Smart API, you can copy from any scheme to any scheme. With so many different schemes, it is impossible to provide an exhaustive list of each useful combination.

Many programmers can use the copy tool without knowing exactly how it works. The from parameter is used as a URI to create an input stream. The to parameter is used as a URI to create an output stream. A copy program won't recognize classic OS wildcards, such as the asterisk (*) and question mark (?).

  public void example() {
	 String s = "in:file:demo1.gif";
	 String t = "out:file:demo2.gif";
	 try {
		CopyTool.getTool().copy( s, t );
	 }
	 catch( IOException e ) {
	 }
  }

A copy program has two parameters, from and to. The from parameter is used as a URI to create an input stream. The to parameter is used as a URI to create an output stream. A copy program won't recognize classic OS wildcards, such as the asterisk (*) and question mark (?).

Do all this interactively, too. The GUI front-end to the copy program enables you to type a source URI in the from field and a target URI in the to field.

Copy a file in the current directory.

in:file:demo1.gif --> out:file:demo2.gif

Copy a file into a directory that does not exist (yet). Output directories are created automatically.

in:file:demo1.gif --> out:file:backup/19991221/demo1.gif

Copy a file out of an archive.

in:[file:src.zip]/org/jos/demo/Demo.java -->
	 out:file:Demo.java

Copy a file directly from the network to your local machine.

http://www.jos.org/download/tools.tgz -->
	  out:file:tools.tgz

The copy tool has a number of different methods: 

  public void copy( String s, String t )
		throws IOException;

  public void copy( InputStream input, String t )
		throws IOException;

  public void copy( String s, OutputStream output )
		throws IOException;

  public void copy( InputStream input, OutputStream output )
		throws IOException;

9 December 1999

Smart API is not equal to MIME. The purpose of the Smart API is very different from the purpose of MIME. After a long week of additional discovery, MIME must be another subsystem needed by an operating system to complement a smart API. For more information on the difference between the Smart API and MIME, see also SmartAPIAndMIME.

Many of the old-style controllers are inefficient. Most of them use the instanceof operator to determine if the object implements the right interface. Of course, the instanceof operator requires an old-style controller to create an object first.

A new-style controller is possible with a ClassTool. This class tool implements instanceof as an instanceOf() method. The signature of this method is something like this: 

public interface ClassTool {
  public abstract boolean instanceOf( Class c, String[] list );
}

The instanceOf method recursively scans a class, all of its interfaces and super classes. It returns true if any interface/class name in list is implemented. It is not used to determine which interface/class name is implemented.

31 July 1999

  • New: The source code for SmartAPI - Release 1b is now available for download (See also SmartAPIPages).
  • New: SmartAPI - Release 1b includes working versions of RegistryIterator and RegistryIteratorController. This makes it easy to build a registry browser (and eventually a registry editor).
  • New: Release 1b includes the acceptURI() and getObject() methods for the smartapi.URI class.
    • Release 1a Patch: You can add an acceptURI() and getObject() method to the smartapi.URI class. These methods should pass a URI string to a corresponding method of the smartapi.ObjectFactory.
  • New: You can browse any .ini file through your registry interface. The PlainTextBrowser has been released as part of SmartAPI - Release 1b. It is an interactive demonstration of the smart object factory. It is useful for experimenting and debugging. The first release of this browser has full support for Registry API - Release 1b.
  • Bug and bug fix: The PlainTextBrowser has already identified the following bug in release 1a.
    • Bug: The smartapi.ObjectFactory tries to be type-safe, like it is supposed to. Unfortunately, some of the controllers implement Controller when they should have implemented ObjectController. The quick fix is to modify ObjectFactory.addController(), making it not so type-safe for now. The right fix is to identify and rework some controllers, like streamfactory.StreamController, to implement ObjectController like they are supposed to.
    • Bug Fix: Release 1b contains the right fix for ObjectControllers.
  • Help Wanted: Your feedback is appreciated. We're still working on bringing together a MatureAPI. We are looking for more people to join the registry group and build more plug-ins. We still need help for smart factories missing from Java. We need a smart factory for AWT components, for each release of Swing components, for JDBC connections, and for many other Java "APIs".

-- GilbertHerschberger

1 August 1999

Upgrading from release 1a to 1b changed the default constructors for these smart factories:

  • smartapi.ClassFactory,
  • smartapi.ObjectFactory,
  • streamfactory.StreamController,
  • streamfactory.InputStreamFactory, and
  • streamfactory.OutputStreamFactory.

Default constructors for a smart factory should use addController() for all classes within the same packages as the smart factory, not addControllerFor(). The purpose of addControllerFor() is to load controllers for other packages that may or may not be installed.

Bug and bug fix: Download release 1b with patch. There is a bug in the default constructor of smartapi1b.ClassFactory. It was not upgraded properly. It attempts to load a controller from release 1a by name, which can't work. It should have added a controller object. To patch release 1b by hand, modify the default constructor like this.

  protected ClassFactory() {
	 <FONT COLOR="#7F7F7F">/* *D!
	 String s =
		  "com.realignsoft.smartapi1a.SystemClassController"
	 addControllerFor( s );
	 */</FONT>
	 <STRONG>addController( new SystemClassController() );</STRONG>
  }

This patch also contains updated test programs for smartapi and streamfactory. If only I had updated and run the test programs for release 1b, there would have been no need for a patch. The ClassFactory_Test would have detected the this bug.

On the other hand, this is a good example of when to use a patch, as described in the rules of SmartAPIProposal. No new classes were added or dropped. No new methods were added or dropped. -- GilbertHerschberger
