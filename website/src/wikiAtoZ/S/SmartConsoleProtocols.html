Article contributed by GilbertHerschberger (21 September 1999).

Introduction

The console studio provides quite a few implementations of the Console interface. It provides a console factory, which implements the x-console: protocol.

Future

While the rest of the URL is expected to contain a class name, this is not the future of the console factory.

Subprotocols

The console factory uses additional controllers to support the following subprotocols:

  • system:

    This subprotocol returns a console attached to a standard input or output stream.

    • standard in - x-console:system:in
    • standard out - x-console:system:out

  • awt:

    This subprotocol returns a visual console, compatible with the AWT API. The factory expects the name of a visual console, literally a Java(tm) class name. The factory interprets a query string to set the following properties (if possible):

    • enabled
    • title
    • visible

  • stream:

    This subprotocol returns a stream console. The rest of the URL is passed directly to the output stream factory.

  • log:

    This subprotocol, like stream, returns a stream console. The rest of the URL is passed directly to the output stream factory. A log is different than a stream because a log automatically adds the date and time to each message.
