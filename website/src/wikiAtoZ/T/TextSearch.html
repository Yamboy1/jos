Introduction

A text search is the process of searching through files for matching text. When you know the text but not the name of the file, a full text search might help.

While files contain the given text?

Specifically, this program prints the name of all files with the given text. Files that do not contain the text are skipped.

Text search utility

A text search utility is part of the SmartAPI 2a library. As a utility, it is written using the Abstract Windowing Toolkit (AWT) for maximum compatibility.

This utility is written in only a few lines of code. It should be easy to maintain and figure out exactly how it works. It uses Finder and its the asynchonous I/O model. Finder searches for files, and optionally, searched for files in subdirectories.

This utility has both a visual and non-visual component. The TextSearchPanel is visual component. It can be integrated into other AWT-style utilities. The TextSearchProgram class is a non-visual component. It can be invoked from any other method.

This utility was written using the Java programming language. It is as platform independent as possible. It is open source and free license software.

Procedure

To search for files which contain given text, use the following procedure.

  1. Start text search utility. Use one of the following methods.

    • To start the text search using the UniversalBrowser, use the following location:

      <STRONG>run:program:org.jos.gchii.textsearch1a.TextSearch</STRONG>

    • To start the text search using the java tool, use something like the following command:

      $ <STRONG>java org.jos.gchii.textsearch1a.TextSearch</STRONG>

  2. Type in a directory name in the directory field. Files in this directory are searched.

  3. Type in a file name pattern in the pattern field. The default pattern is *.txt.

  4. Type in the matching text. This is an exact match, so basically all characters are taken literally.

  5. Chose one of the show options. This option affects files in subdirectories. When show is selected, subdirectories are included in the search. Otherwise, subdirectories are ignored.

  6. Press the Run button. The text search prints files that contain the matching text.

    Exact match

    As you can probably see from the source code, the utility prints the name of a file only when it finds an exact match. It is case sensitive. It does not match whole words. It uses a single match. The carriage return/newline (CR) and line feed (LF) characters cannot be matched. This code could be expanded to include any of the following options:

    • case-sensitive
    • whole words
    • multiple match
