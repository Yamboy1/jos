ProcessPages

Article contributed by GilbertHerschberger (1 June 2000).

Introduction

The real world already provides us with Java applications that should be able to run as an independent bytecode process. Most of those applications are unaware they are hosted by our Java-based operating system.

Case A Series

The following program should terminate the new bytecode process, without affecting any other bytecode process.

  • Test Case A100

    public class TestA100 {
  public static void main( String[] args ) {
  }
}

    This bytecode process must terminate when its main thread ends.

  • Test Case A101

    public class TestA101 {
  public static void main( String[] args ) {
	 System.exit( 0 );
  }
}

  • Test Case A102

    public class TestA102 {
  public static void main( String[] args ) {
	 System.exit( 1 );
  }
}

    (This isn't exactly a worst case.) This reminds me that a return code must be returned to the calling process.

  • Test Case A201

    public class TestA201 {
  public static void main( String[] args ) {
	 throw new RuntimeException();
  }
}

    This bytecode process must terminate when its main thread ends, even with a runtime exception.

  • Test Case A202

    public class TestA202 {
  public static void main( String[] args ) {
	 throw new VirtualMachineError();
  }
}

    This bytecode process must be isolated from all others, even when a virtual machine error is thrown like this. Errors should terminate this bytecode process, but not a virtual machine.

  • Test Case A203

    public class TestA203 {
  public static void main( String[] args ) {
	 NonExistantClass.getTool();
  }
}

    This bytecode process must be isolated from all others, especially a class-not-found exception like this. Exceptions should terminate this bytecode process, but not a virtual machine. This test includes all cases where a class cannot be resolved with its superclass and interfaces.

  • Test Case A204

    public class TestA204 {
  public static void main( String[] args ) {
	 InvalidClass.getTool();
  }
}

    This bytecode process must be isolated from all others. Errors like this should terminate this bytecode process, but not a virtual machine.

  • Test Case A205

    public class TestA205 {
  static {
	 System.loadLibrary( "nonExistantLibrary" );
  }
  public static void main( String[] args ) {
  }
}

    A link errors like this should terminate this bytecode process, but not a virtual machine.

  • Test Case A301

    public class TestA301 {
  public static void main( String[] args ) {
	 0 / 0;
  }
}

    Divide by zero exceptions should not terminate a virtual machine.

  • Test Case A302

    public class TestA302 {
  public static final int x = 0 / 0;
  public static void main( String[] args ) {
  }
}

    This divide by zero exception should not terminate a virtual machine.

  • Test Case A303

    public class TestA303 {
  static {
	 0 / 0;
  }
  public static void main( String[] args ) {
  }
}

    This divide by zero exception should not terminate a virtual machine.
