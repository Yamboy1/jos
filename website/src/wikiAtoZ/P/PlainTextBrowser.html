SmartAPI; PlainTextBrowserExamples

Article contributed by GilbertHerschberger (28 July 1999).

Introduction

Earlier, I may have hinted at a plain-text browser. What is a plain-text browser? And how can I build one?

Definition

A plain-text browser is a browser in its simplest form. Like its complex cousin, a plain-text browser has a location field. When you type in a location and press the Run button, the plain-text browser displays a document always in plain text.

Clearing the display

Each press of the Run button should clear the display.

Support scrolling

While a GUI plain-text browser has scrollbars, a character-based interface should automatically repaint the screen like the more command (or the less command). A character-based browser should be compatible with telnet.

Support fragments

And when you use a fragment in a URI, it scrolls forward to the anchor (if possible). The fragment part of a URI must not be passed to the smart object factory.

Linking

In order to support hot-links in a plain-text browser, you have at least these options:

  • Pick a hot-link from a list

    When you press a button on the toolbar, a list of all hot-links in the plain-text document are displayed. The browser recognizes links in the form of <URL:link> and others.

  • Link at cursor

    After you move your cursor to a line and column, you press a button on the toolbar. The browser tries to extract a link from that part of the plain-text document.

  • Text selection

    After you select text in a plain-text document, you press a button on the toolbar. The browser uses the selected text as a link.

Building your browser

You can assemble a plain-text browser from the Java AWT (or Swing) and release 1a of the SmartAPI. You can use a list or text area component as your plain-text display.

  • List component

    You can use a list component to display the plain text. The list component is able to hold large documents. The smallest area that can be selected is an entire line.

  • Text area component

    You can also use a text area component. The text-area component doesn't display large documents. A non-editable text area provides a way to select text. You can copy selected text into the clipboard. You can use selected text as a new location.

When the run button is pressed, the location is passed directly to the smart object factory. Based on the class of the object returned, a document can be output to the display.

The plain text browser has already proved its worth. It quickly demonstrates what the object factory can and cannot do.

In the last 24 hours, I was disappointed three times by the Java API. First, the java.net.URL doesn't seem to recognize anything other than the http: scheme. While it's great that it recognizes http:, I thought there were additional packages in standard distribution for ftp:, news:, and mailto:. I just assumed they'd be there because I've seen them consistently in package names.

Second, I was surprised that java.io.StreamTokenizer doesn't have a close() method. So, how do you close the stream tokenizer once you've opened it? Maybe a stream tokenizer isn't the right class to break an input stream into tokens. I want a StringTokenizer for an input stream.

Backward compatibility is important to your programs. I was again disappointed in Sun's "reasonable incompatibility" approach. The java.sql.DatabaseMetaData interface has been changed in a nasty way. While an early release defined getProductName() and getProductVersion() to throw an SQLException, the later release does not. So all programmers writing new code will not be prepared for it when an old implementation throws this exception. After you upgrade, your compiler will mislead you, telling you that the SQLException is never thrown by these methods when, in fact, it can be thrown. -- GilbertHerschberger (30 July 1999)
