I feel like a kid in a candy store: So many new things to try. -- GilbertHerschberger (31 July 1999)

Introduction

This is something I've been wanting to do for a long time. I put together the first plain text browser that accepts URIs and passes them on to the smart object factory. This is a front-end to the smart object factory and demonstrates the power of the SmartAPI. I added a bridge from the plain text browser to the registry API. Now, I can test more of the registry API, too.

File: protocol

Try these URIs for the location in your plain text browser. 

<STRONG>file:</STRONG>/usr/local/example
<STRONG>in:</STRONG>file:/usr/local/example

in:<STRONG>buffer:</STRONG>/usr/local/example
in:buffer:file:/usr/local/example

in:<STRONG>data:</STRONG>/usr/local/example
in:data:file:/usr/local/example

in:<STRONG>token:</STRONG>/usr/local/example
in:token:file:/usr/local/example

in:token:data:buffer:file:/usr/local/example

windows only:
file:///C|/autoexec.bat
in:file:c:autoexec.bat
in:file:c:/autoexec.bat

x-registry protocol

When you have .ini files, you can browse them already. 

<STRONG>x-registry:</STRONG>file:run.ini
x-registry:file:demo1.ini#/section-1

x-registry:file:/usr/local/example.ini
x-registry:ini:file:/usr/local/example

x-registry:file:c:windowswin.ini
x-registry:file:c:windowssystem.ini#/Password Lists

About registry keys

the registry.getKeys() method does not have to return its keys in any particular order. A key either exists or it doesn't. It seems to me that a good registry browser/editor should be able to sort keys in alphabetical order.

Token input stream

Because the java.io.StreamTokenizer wasn't anywhere close to what I wanted, I wrote TokenInputStream. Both the controller and stream have a lot more modifications before they will be "finished".

By default, the token input stream uses a space, TAB, CR, and LF as its delimiters and shows the delimeters in the browser. A query string should be passed to the controller to specify delimiters, delimiter sets (c, cpp, java, etc), and choose show/noshow for delimiters.
