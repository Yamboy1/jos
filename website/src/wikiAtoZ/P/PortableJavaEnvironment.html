Introduction

A portable Java environment is desireable. It runs the same way on any platform. It includes the platform-specific tools that are part of the Java Development Kit. It is portable because it runs on any off-the-shelf virtual machine.

The portable Java environment is defined this way:

[virtual display*]			[foreign application launcher*]
	 |								|		 
	 |								|	[Java application launcher]
  [window manager*]			 |		 /
	  						[portable application launcher]
								 /
		 					  /
		[portable desktop application]

* - platform-specific components are valid.

Desktop

A desktop is an application. It runs on top of (portable) window manager. It uses a portable application launcher, capable of launching both foreign applications and Java applications.

Launcher

A portable application launcher is a non-visual package. It provides a unified model for launching a foreign application and a Java application.

  • foreign application

    While the foreign application launcher is platform-specific plug-in, the portable application launcher works the same way on every platform. A JNI-compatible virtual machine is a specific platform. An MPCL-compatible virtual machine is-a specific platform.

    The portable application launcher provides a better model of applications than the exec() method of java.lang.Runtime(). And yet, it depends upon the exec() method of java.lang.Runtime. This portable application launcher is something in the style of the run:os: scheme of SmartAPI.

  • Java application

    The portable application launcher dispatches a request to run a Java application to the Java application launcher. The Java application launcher provides a model of a Java application. It might create a new virtual machine in a new machine code process. It might create a new primordial class loader in a MPCL-compatible virtual machine. Either way, a Java application launcher depends on a foreign application launcher.
