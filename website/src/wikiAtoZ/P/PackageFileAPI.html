ERICProject

Article contributed by GilbertHerschberger (17 December 1999).

Introduction

This API is part of the Execute, Run, Invoke and Call (ERIC) Project. It defines a well-known API for reading and writing a package file.

Package file builder

A package file builder is responsible for building a single package file. As classes are added to a builder, additional validation is used to make certain a class belongs in a package.

Let's use the java.awt.Button class as an example. A package name is passed to a package builder. All classes in a package must belong to the given package. A package file builder is expected to compare a package name with the absolute name of a class. If a class cannot be added to a package, a modifier exception is thrown.

  public void example() {
	 String pn = "java.awt";
	 String[] list = {
		  "java.awt.Component",
		  "java.awt.Button",
		  "java.awt.Frame",
		  "java.awt.Window",
		  "java.awt.Canvas",
		  "java.awt.Font",
		  "java.awt.List",
		  "java.awt.TextArea",
		  "java.awt.TextField",
		  "java.awt.Color"
		  };

	 PackageFileBuilder builder =
		  new BasicPackageFileBuilder( pn );

	 builder.setCompressionMethod( CompressionMethod.GZIP );

	 int iMax = list.length;
	 for ( int i = 0; i < iMax; i++ ) {
		try {
		  builder.addClass( item, grabClass( item ) );
		}
		catch( ModifierException e ) {
		  _print( e );
		}
	 }

	 try {
		builder.saveAs( "file:" + pn + ".package" );
	 }
	 catch( IOException e ) {
		_print( e );
	 }
  }

Reading a package file

A package file is easy to read with this framework.

  public void example() {
	 String s = "http://www.jos.org/jpkg/java.awt.package";
	 PackageFile pf;

	 try {
		pf = new BasicPackageFile( s );
	 }
	 catch( MalformedPackageFileException e ) {
		_print( e );
	 }

	 byte[] pf.getClassData( "java.awt.Canvas" );
  }

Modifier exception

When a package builder is modified, there is always a possibility of a modifier exception. This API will use a unique modifier exception that's compatible with personal distributed computing and called org.jos.packagefile1a.dev.ModifierException.

A package builder does not throw a malformed-package-file exception.

Header interface

A header has properties. These properties are modified through an interface called Header. Each kind of header is an extension of BaseHeader, which implements Header. This class is common to the package reader and package writer, so it is part of the org.jos.packagefile1a package.

package org.jos.packagefile1a;

public interface Header {
  public abstract boolean isMagic( byte[] v );
  public abstract byte[] getMagic();
  public abstract byte[] getData();
  public abstract int getSize();
}

While reading a package, the package input stream needs to know how big each header is expected to be. The getSize() method provides a size of a header in octets.

While writing a package, the package output stream needs to get the entire header -- properties and all -- in a single call. This is the purpose of the getData() method. It returns an single array of octets (bytes) with properties encoded within the array.

BaseHeader class

To make it easier for a custom header to work within this API, a base class is provided.

package org.jos.packagefile1a.BaseHeader;

public abstract class BaseHeader
	 implements Header {
  public BaseHeader() {
  }
  public BaseHeader( byte[] v ) {
	 setMagic( v );
  }
  public BaseHeader( String v ) {
	 setMagic( v );
  }
  public boolean isMagic( byte[] v ) {
	 int iMax = magic.length;
	 if ( iMax != v.length ) {
		return false;
	 }
	 for ( int i = 0; i < iMax; i++ ) {
		if ( magic[ i ] != v[ i ] ) {
		  return false;
		}
	 }
	 return true;
  }
  public byte[] getMagic() {
	 byte[] buf = new byte[ magic.length ];
	 System.arraycopy( magic, 0, buf, 0, magic.length );
	 return buf;
  }
  protected void setMagic( byte[] v ) {
	 magic = v;
  }
  protected void setMagic( String v ) {
	 byte[] buf = v.length() + 1;
	 int t = 0;
	 buf[ t++ ] = (byte) 26;
	 int iMax = v.length();
	 for ( int i = 0; i < iMax; i++ ) {
		buf[ t++ ] = (byte) v.charAt( i );
	 }
	 setMagic( buf );
  }
  private byte[] magic = null;
}

FileFormatHeader class

The file format header is always plain-text at the beginning of a package file.

public org.jos.packagefile1a;

public class FileFormatHeader
	 extends BaseHeader {
  public FileFormatHeader() {
	 super( "filE" );
  }
  public long getLastModified() {
	 return lastModified;
  }
  public void setLastModified( long v ) {
	 lastModified = v;
  }
  public CompressionMethod getCompressionMethod() {
	 return compressionMethod;
  }
  public void setCompressionMethod( CompressionMethod v ) {
	 compressionMethod = v;
  }
  private long lastModified;
  private CompressionMethod compressionMethod = nul;
}
