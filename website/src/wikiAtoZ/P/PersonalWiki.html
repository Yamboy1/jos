Article contributed by GilbertHerschberger (10 May 2001).

Introduction

The PersonalWiki software is intended to demonstrate the servlet-transaction model, the relationship between serlvet-transactions and programs and how to reuse a wiki-text renderer to create static HTML pages.

It is not intended to replace the sfWiki software on the Internet. Unlike the sfWiki software, the PersonalWiki software expects an author to be an HTML author. It renders only a small subset of the wiki format. It does not support multiple forums. Since it is intended to run only on your local machine, it does not yet require author accounts. It is written at the level of Java 1 (JDK 1.1.6-compatibility).

The Subprogram API should be combined with the servlet-transaction model to create a transaction program.

Servlet-transaction model

Experience has shown that -- unlike other software -- a servlet should represent the application level. Each servlet-based application should consist of a single servlet. This has many benefits. A servlet-transaction model is stronger than the servlet model alone. A servlet becomes responsible for selecting an appropriate transaction. A servlet examines the incoming request and quickly delegates the request to a transaction. A transaction does the grunt work. Each transaction is dedicated to a single kind of request. When transactions are similar, inheritence is employed to isolate the similarities in a base class.

A servlet needs a servlet environment. A servlet and all of its transactions share one custom classloader. A transaction combines one request and one response.

A servlet-based application should use as few servlets as possible. Servlets are expensive to load, install, etc..

Servlet transactions are difficult to design at the servlet level. Another concept is needed to encapsulate the request-response pair. This concept is a "transaction".

A transaction object is created with every incoming HTTP request. A transaction has parameters and path from the original HttpServletRequest object. The transaction has a console from the original HttpServletResponse object.

Subprogram

The transaction sounds a lot like a program. A program has parameters and path from the original location. A program has a console provided by the subprogram environment. A new instance of a program is created with each request. The program is Runnable.

A transaction sounds a lot like a program because it is a kind-of program. A servlet transaction is Runnable.

Benefits

When a transaction is-a program, it is capable of being reused in ways that servlets and other servlet-transactions cannot possibly be used. For example, the Subprogram API enables the output from a program to be redirected to a file. When a servlet-transaction is a program, its output can be redirected to a file. Write one program that can output directly to a browser and output to a static HTML file.
Version 1a is the first implementation of Personal Wiki. It is explicitly versioned in the gchii.tools.personalwiki1a package. It is capable of rendering wiki text exclusively with HTML markup. Support for wiki markup will be added later.

This version provides an Index transaction. An index is generated for one letter at a time. Click on the A link to display a list of all articles which begin with the letter A. An Index transaction is not a search. While a search would read article content, an index only reads article names.

This version stores articles in a series of 26+ directories from A thru Z. Articles which begin with A are stored in the A directory; articles which begin with B are stored in the B directory; and so on. This increases the performance of Personal Wiki without resorting to the overhead of an SQL server. The Article class is responsible for the article name and corresponding file name.

  • It renders four hyphens (-) as <HR SIZE="!" NOSHADE>.
  • It supports [three spaces]* as an indent.
  • It supports [six spaces]* as an double indent.
  • It is not yet capable of editing or saving an article.
-- GilbertHerschberger (12 June 2001).
Version 1b is an upgrade of version 1a. It is explicitly versioned in the gchii.tools.personalwiki1b package. It is renders HTML markup, a subset of wiki markup and cross-references article names. Support for more wiki markup will be added.

This version also stores articles in a series of 26+ directories.

Look closely at the title of an article. See how an extra space is inserted before each uppercase letter to make the title more legible. The Article class is responsible for the article name and corresponding file name and "title". A title has spaces between words in an article name.

  • WebHome becomes Web Home;
  • WhosWho becomes Whos Who;
  • WhyNotMultiProcessJVM becomes Why Not Multi Process JVM; and
  • PersonalWiki becomes Personal Wiki.
The Index transaction has been improved to display an article title, rather than article name. It no longer displays the directory part of an article's location.

This version introduces the Text transaction. This transaction displays the wiki text as is in an HTML page. This feature enables one personal wiki servlet to render pages from another in real-time.

This version has added support for wiki markup from both sfWiki and QuickiWiki.

  • It renders text surrounded by the percent sign (%) as an article name.
  • It renders an indent from the first characters on a line.
    • a single asterisk (*)
    • [three spaces]*
    • [tab]*
  • It renders a double indent from the first characters on a line.
    • a double asterisk (*)
    • [six spaces]*
    • [tab][tab]*
  • It renders strong from a pair of three single quotes (').
  • It renders emphasis from a pair of two single quotes (').
  • It automatically disables wiki markup
    • within an anchor tag with an HTML option,
    • within a preformatted section (the PRE tag), and
    • within a code section (the CODE tag).
  • For all-caps article names only, a link is provided if and only if the corresponding article exists. It does not provide a link to an edit page.
  • When rendering an article name in the title of an HTML page, a space is inserted between "words" to make the article title more read-able.
  • For all existing articles, an article name is a link to an article. When rendering an article name in the link, a space is added between "words" to make it more read-able.
  • For a missing article, a prefix-style link recommended by TheWikiWay is used. A link to an edit page is displayed as a question mark. The article name itself is rendered as plain text without a space added.
  • It does not yet recognize the NOLINK tag.
  • It does not yet render e-mail addresses or other locations, such as ftp: or http:.
  • While it produces an edit link for missing articles, it is not yet capable of editing or saving an article. It does not render an Edit button. It does not render an edit form or Save button.
-- GilbertHerschberger (12 June 2001).
Version 1d is an upgrade of Personal Wiki 1c.
  • It loads an external template. The template is stored in a wiki article called WikiTemplate so that it can be edited just like any other article. The template expires every 30 seconds.
  • The root directory of wiki text is now partially implemented as a servlet parameter. All wiki text is saved in a root/wiki-text directory.
  • Subdirectories A to Z are created automatically as new articles are saved.
  • New articles can be created with a click.
  • This version introduces a new read-only mode. An edit button is never displayed in this mode. Wiki text cannot be saved.
  • Although it has not yet been tested, it might be possible to store wiki text in a ZIP-compatible archive. The root parameter would point to the archive instead of a directory.
-- GilbertHerschberger (25 February 2002).
