DepretiatedMethods [sic];

Article contributed by GilbertHerschberger (30 August 1999).

Introduction

We need a preprocessor. In spite of the fact a preprocessor has been abused in C/C++, we still need a preprocessor for Java. Some things need to be done that are far too difficult to do without one. Instead of supporting no preprocessor, Sun Microsystems should have provided a standard preprocessor API.

Porting

I'm stuck with porting from one Java platform to another. At least it is possible with a C/C++ preprocessor to maintain a port from one platform to another. What about Java? It is nearly impossible. The standard Java compiler doesn't have a preprocessor. It doesn't acknowledge the need to port Java code. When porting from Java 0 to Java 1 to Java 2, it is desparately needed.

I should be able write something like this: 

#java2
import java.swing.*;
#else
import javax.swing.*;
#endif

Not just forward

When I write code for other people to compile, I am supposed to write code that's source-code compatible with their compiler. Some people use Java 1 Platform. Some people use Java 2. Some people still use Java 0 Platform for creating applets that run in an Internet browser. I need to write code that other people can compile successfully.

  #java1
  public void show() {
	 setVisible( true );
  }
  public void hide() {
	 setVisible( false );
  }
  #else
  #endif

Within a method, I need to write something like this: 

  public void example() {
	 #java1
	 add( "directory", directoryPanel );
	 #else
	 add( directoryPanel );
	 #endif
  }

#javaxxx

This directive resolves to true when the compiler uses the given platform or higher. Otherwise, resolves as false. #java1 is for Java 1 Platform; #java2 is for Java 2 Platform; #java3 is for Java 3 Platform; and so on.

#else

This directive separates the lines of code to compile on false.

#endif

This directive marks the end of conditional compiling.

Package directive

When a directory is configured for a specific base package, it should be inserted into the source code automatically. To set the package statement based on the current configuration, use the #package directive. 

#package.smartapi1b;

You should be able to re-compile the same source code for different Java packages. As a consultant, my customers want their code to fit inside their package naming convention. I have vault of Java source code. I need to give them my code in the packages they prefer.

#atleastxxx

This directive stops an old compiler from compiling new code. #atleast1 is for Java 1 Platform; #atleast2 is for Java 2 Platform; #atleast3 is for Java 3 Platform; and so on.

When a Java 1 Platform compiler sees the #atleast2 directive, it will stop compiling immediately with an error message. #atleast2 is equivalent to 

#java2
#else
#abort <EM>message</EM>
#endif
