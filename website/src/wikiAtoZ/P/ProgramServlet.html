Introduction

An arbitrary program can be invoked by the program servlet. The ProgramServlet class is part of the gchii.tools.programservlet1b package. This package depends on the Java Servlet Development Kit (JSDK). This class is installed in a servlet environment.

Programs are SmartAPI-compatible and invoked on the server-side. Since run:main: invokes any Java application, many output-only programs can be reused in the servlet environment. Since run:os: is connected to the exec() method of java.lang.Runtime, many output-only machine code programs can be reused as well.

How it works

The output from a console program is piped to a client program, usually an HTML browser. The servlet expects a single location parameter. If the location parameter is missing, the following HTML form is displayed.

Location: run:program:org.jos.demo.Demo

An arbitrary location should be a uniform resource identifier for a program using the run: scheme. This parameter is passed directly to the Smart API. If org.jos.demo.Demo is a console program, the program servlet returns a page that contains its preformatted output.

Servlet name

When the program servlet is installed on localhost at port 8080, the location in an HTML browser is this:

http://localhost:8080/servlet/gchii.tools.programservlet1b.ProgramServlet
