Introduction

When the cost of a computer is between US$100,000 and US$1,000,000, few organizations can afford to own more than one. When it is between US$1,000 and US$100,000, few organizations can afford one for each person. What happens when the cost of a computer is between US$1 and US$1,000? An organization can afford 100 or more computers per person.

Multiple machines

I want to run 100 applications at the same time. How can I do that? It is possible for each computer to run one application. In order to have 100 applications running at the same time, I need 100 computers.

Process

When a computer is expensive, few people want to buy a one. They resist. Why should they buy a new computer when they don't have to? If it were possible, people would like to run more than one application on the computer they already have.

When an operating system is complex, few people want to learn a new one. They resist. Why should they abandon their old operating system when they don't have to? If it were possible, people would like to run new applications on an old operating system. They would applaud the product, like Java Technologies, that gives new life to old operating systems.

A process maintains the illusion that a program is running alone on a processor.

Resources are limited. Even an expensive computer has limited resources. An operating system must optimize the use of limited resources. For example, a system-wide mechanism for virtual memory maintains the illusion that an application might have more memory available regardless of how many other applications are running.

Inter-machine communication

Inter-machine communication is communication between machines. When each application runs on its own machine, the only inter-process communication is inter-machine communication. 100 machines might be wired together one a network. A network interface card is only one of many ways two machines share their data. Data is easy to share.

The model of inter-machine communication is scaleable. It remains the best form of inter-process communication even when many applications are running on one machine.

Summary

An MPCL-compatible virtual machine, with multiple primordial class loaders, maintains the illusion that an application is running on its own virtual machine. It must be able to share data between processes long before it can share objects. The java.net package provides all the support we need for sharing data between two independent processes.
