PlatformAPIPages

Article contributed by GilbertHerschberger (6 December 1999).

Introduction

Runtime and distribution are two aspects of JOS as a viable software platform. While runtime is dependent on distribution, distribution options are tempered by runtime limitations. We should further define JOS from both perspectives.

Run-time platform

The operating system software must do something. The first aspect of a platform is its runtime characteristics.

How does JOS perform?

At runtime, JOS must run application software in a consistent and preditable manner. A kernel, virtual machine, process and program are run-time components. For example, the platform must create a process consistently so that each process is as independent as possible from all other processes. A program must be invoked consistently so that we can reuse software at the operating system level. Classpath must be managed consistenly. Class loaders must be managed consistently.

As a platform, JOS requires a JOSBox. An operating system provides a predictable environment for compatible applications. Programmers must be able to develop a JOS-compatible application on one machine and customers run it on another.

Distribution platform

The minimum and maximum implementations of this platform are distribution issues. JOSCore alone is a minimum installation. JOSCore plus every JOS-compatible extension every written would be a maximum implementation.

How is JOS installed?

You are required to install everything inside JOSCore. JOSCore is the minimum requirements of a JOS platform. When JOS is distributed, there are a minimum of components that are absolutely, positively required in order for the operating system to be called JOS. The minimum requirements are an issue of distribution. The minimum requirements of JOS is called JOSCore. The minimum requirements will be defined and refined as part of the JOSCore project.

By itself, a minimum distribution of the platform should not be expected to do much. That is why we have JOSExtension. A JOS-compatible extension uses the extension mechanism as defined by JOSBox. While JOSBox determines the mechanism all JOS-compatible extensions must use, a JOSExtension is an implementation of that mechanism. The most interesting part of the platform is found inside JOS extensions. Extensions are distributed along with JOSCore to provide you a choice when you install JOS on your own JOS machine. You are not required to install any extension.
