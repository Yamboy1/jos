BulkMake

Article contributed by GilbertHerschberger (18 August 2000, updated 6 December 2001).

Introduction

The Package Archive utility is part of the BulkMake(tm) product. Like all of the tools in BulkMake, this tool is PackageOriented.

Purpose

The purpose of this tool is to archive a package. Nothing less than a package is archived. This is not a file-at-a-time tool. When given an archive name, this tool adds source code and bytecode for each package to a pair of archives. Source code goes in one archive, and bytecode goes in the other. The tool always starts with the current directory and searches the file subsystem for source code and class files.
  • Matching bytecode, or a .class file, is added to an archive file called archive-name.zip.
  • Matching source code, or a .java file, is added to a an archive file called archive-name-src.zip.
New in version 2b! In patch mode, the tool starts with the current directory and searches the file subsystem for source code and class files which have changed.
  • New bytecode, or a .class file, is added to an archive file called archive-name-patch.zip.
  • New source code, or a .java file, is added to a an archive file called archive-name-src-patch.zip.

Interactive

This is the recommended way to use this tool interactively.
  • Start the BulkMake application,
  • Click the Archive button,
  • Type a name in the archive field,
  • Check or uncheck the patch field, and
  • Press the Run button.
The package tool creates an archive. Since the BulkMake application has its own console, output from the package archive tool is displayed there.

Command line

The package archive tool is available from the command line, and thus available to any method in any other application. Here are some important details about this tool.

The name of the this version package archive tool is: 

gchii.tools.packagearchive2b.PackageArchiveTool
The program recognizes the following parameters:
  • archive name. This parameter is used as the base name for the archive file. To create archives josdemo.zip and josdemo-src.zip, use the following parameter:
    archive=josdemo
  • patch. This parameter is optional. It is used to set patch mode. By default, this tool creates a full archive; patch mode is false. To create a incremental archive, set patch mode to true with the following parameter:
    patch=true

Configuration

The package archive tool always uses the jar tool to create an archive file. The jar tool is an external tool. To configure the jar tool interactively, the following steps are recommended:
  • Start the BulkMake application,
  • Click the Configure button,
  • Type a name in the Jar Tool field,
  • Press the Save button.
