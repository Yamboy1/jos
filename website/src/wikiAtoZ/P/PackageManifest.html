Introduction

A package manifest file contains the names of every package implemented in a collection. Where does this file come from? How should it be used?

Today

Package files are part of the ERICProject and have a .package extension. Originally, the deric index was optional.

The Deric utility creates a package manifest file. The manifest file is compatible with the data registry API and is always called dericindex.txt. Here is a recent example:

  • http://www.jos.org/redist/jpkg/dericindex.txt

The Erica utility uses this manifest file to download packages from a remote source, such as a file, FTP or HTTP service.

About FTP and HTTP

Compared to local disk, it is timeconsuming to check the existance of a remote package file, through a file, FTP and HTTP service. Adding a remote source to your package path has the adverse side-effect of slowing down an application. The worst case scenario is trying to load a package which does not exist.

Rather than check the existance of a remote package file through a file, FTP or HTTP service, it is possible to download a package manifest file. Through this manifest, the existance of a package can be performed more quickly.

Summary

The package manifest file comes from the Deric utility. It should be used to download package files (like the Erica utility) and to check the existance of a remote package file.
