BytecodePages

Article contributed by GilbertHerschberger (2 May 2000).

Introduction

An equivalent of the javap tool is needed. It must be written in pure bytecode and follow the Program API. It should use pure reflection to display the decoded version of a bytecode method.

Not only does it provide useful information about any Java compatible class, it also demonstrates the benefits of pure reflection.

  • This tool will be platform independent. It will run in any version of Java.

  • This tool will not use practical reflection. It will not create any instance of java.lang.Class for a given class file.

  • This tool is a first in a series of tools. This set might have a static verifier and other tools.

Note: The javap tool is part of a Java Development Kit (JDK) from Sun Microsystems.

Static verification

Each Code attribute can be converted from its raw form to a bytecode table. When an opcode jumps to a non-existant offset, the code is invalid and should not be used.
