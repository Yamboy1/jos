The size of each opcode is always 1 more than the size of its corresponding operand. The new Operand class is a Java-style enumeration of all possible operands. The new OpcodeSize class depends on Operand rather than an explicit integer size. -- GilbertHerschberger (15 November 2000)

An opcode table must deal with wide opcodes, too. On the one hand, a wide opcode should have its own identifier, with a WIDE_ prefix. On the other hand, both the wide opcode itself and the next opcode must be used to look up an opcode in the opcode table.

  • Narrow opcodes have a range from 0 to 255.
  • Wide opcodes might have a range from 256 to 511.

An opcode table should also contain the number of elements popped from the stack. This number can be used generically to pop elements off the stack before calling an opcode method. Parameters are passed to an opcode method using both operands and the stack. The type of parameter expected is also closely associated with an opcode. -- GilbertHerschberger (2 May 2000).

The stack frame could be passed as a parameter to an opcode method. The stack frame itself could have an exception field, so that exceptions can be passed from one opcode method to another.

Here is an example of a typedef for an opcode method in C++.

void (*TOpcodeMethod)( TStackframe *stack );

Further, if there were an 512-item array of opcode methods (indexed by opcode), a method could be invoked by compiled machine code.

  unsigned char opcode = *stack.pc;
  if ( opcode == WIDE ) {
	 opcode += 256;
  }
  stack.pc += list[ opcode ].getLength();
  TOpcodeMethod method = list[ opcode ].getMethod();
  method( stack );

More needs to be done to integrate pure reflection with a static class verifier. -- GilbertHerschberger (25 May 2000).
