The following article was contributed by GilbertHerschberger (3 July 2004).

Introduction

This article re-introduces Java properties. The Standard Class Library for Java contains support for the properties file format. The java.util.Properties class enables you to decode a properties file.

The properties file format is a simple, non-hiearchial format.

About XML

The following example illustrates a simple XML document. 

<?xml version="1.0"?>
<list>
  <item1>text1</item1>
  <item2>text2</item2>
  <item3>text3</item3>
</list>

As a review, an XML document is a collection of elements. In this example, list, item1, item2 and item3 are names of elements. A useful XML document must have exactly one root element. In this example, the name of the root element is "list".

Mapping to properties

Our example document can be read as if it were a properties file. This is an equivalent properties file: 

list.item1=text1
list.item2=text2
list.item3=text3

Here are some of the rules: # Properties are equivalent to text nodes of an XML document. Comments are ignored. Attributes are ignored. An element that contains a mix of both text and other elements is ignored. # For an element that contains only text, the property name includes the name of its parent, recursively, separated by a dot. In other words, the name of a property is a dot-separated, fully qualified name of an element. The prefix of every property will be the name of the root document because a good document always has a document root. # In XML, a document might contain multiple elements with the same name. These elements should be mapped; but, properties are not multi-valued. Mapping for multi-valued properties is not yet supported.

properties: scheme

The properties: scheme enables you to load properties from a digital resource. It always returns an instance of java.util.Properties, or null. The properties: scheme expects a subscheme to provide a resource to convert into an instance of Properties.

A local properties file called "josdemo.properties", for example, can be obtained with the following URI:

  • properties:file:josdemo.properties
A local XML file called "josdemo.xml" can be obtained with the following URI:
  • properties:document:xml:file:josdemo.xml
The file: scheme gets an instance of java.io.InputStream. In turn, the document:xml: scheme gets an instance of org.w3c.dom.Document. In turn, the properties: scheme maps the document to properties and returns an instance of java.util.Properties.

Limits

Of course, the XML document is far more expressive than a properties file. Our mapping has limitations.

Checklist

The Properties browser is an AWT-compatible utility. It displays the content of an java.util.Properties object.
  • The Properties browser should be able to display a local properties file with the file: scheme.
  • The Properties browser should be able to display a remote properties file with the ftp: and http: schemes.
  • The Properties browser should be able to display any properties file on the classpath with the resource: scheme. The resource: scheme uses the Java API to open an input stream on a file inside a directory or jar.
  • The Properties browser should be able to display any XML document as a properties file using the document:xml: scheme.
