ERICProject; DownloadMiniServer; MiniServerAndEric

Article contributed by GilbertHerschberger (4 January 2000)

Attention servlet developers: Take it with you wherever you go. MiniServer(tm) is about 450,000 bytes and fits on a standard 1.44MB floppy. That leaves enough room on a floppy for your sample servlets. This HTTP service starts quickly and is easy to configure. No command-line parameters. Automatic classpath and JDK/JRE support.

Introduction

MiniServer(tm) is a servlet-only HTTP service. It is a servlet environment. It is a GUI replacement for the servletrunner tool. It is fully integrated into your native operating system because it is based on the eric tool, not the java tool.

(As currently distributed, you must provide your own copy of the jsdk.jar from the Java Servlet Development Kit (JSDK).)

About MiniServer(tm) for Windows(tm)

To start this service,
  1. Double-click on the minisrv.exe program;
  2. Enter directory and port; and
  3. Press the Run button.
To stop this service,
  1. Press the Stop button, or close the MiniServer(tm) window.
    • (The program runs as long as the window is open.)
To start the non-visual tool,
  1. Click (or double-click) the minisrvd.exe program. Note: The non-visual tool shares the configuration of the visual tool. Use the visual tool to configure it or edit the miniserverapplication.ini file.
    • (The program runs until you stop it with End Task.)

About MiniServer(tm) for Linux(r)

MiniServer(tm) for Linux(r) is coming soon. It awaits the final release of version 1d.

About ERIC

The minisrv tool was created without touching a C/C++ compiler. It is a byte-for-byte copy of the eric tool. The eric tool is available as part of the ERIC(tm) Runtime Edition.
