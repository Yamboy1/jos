Introduction

MPCL is part of the JOS Project. It is part of our process-in-bytecode research. It stands for multiple primordial class loader. In general, there are two kinds of MPCL:

  • using machine code exclusively (MPCL/M) and
  • using machine code and bytecode (MPCL/B)

Using machine code

A virtual machine can hide the primordial class loader from a Java application. The getClassLoader() method returns null for system classes.

Internal to a virtual machine, each class has a private property to identify its primordial class loader.

Using bytecode

A virtual machine can expose the primordial class loader. The getClassLoader() method returns an instance of java.lang.ClassLoader for all classes, including system classes.

Internal to a virtual machine, a primordial class loader in machine code must be mapped to a custom class loader in bytecode.
