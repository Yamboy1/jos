Introduction

When a property is inaccessable, an accessor exception should be thrown. An exception indicates an error has occurred. Unlike C, there should be no special return values.

It is better to make the accessor exception an exception, not a runtime exception. There are two reasons for this.

  1. In practice, you must write exception handling for registry accessors anyway. The content of a registry is dynamic and unpredictable. There is no guarantee that a key exists in any specific registry object.
  2. Each method of the registry interface is optional. Even the registry adapter is a valid implementation of the registry interface. Even though exceptions are irritating to some programmers, the explicit declaration of the accessor exception makes it possible to build predictable code.

For more information, see also the design of the java.sql package, with its SQLException.

Example

What happens when there is no item for a given key? What happens when the item with a given key is definitely not boolean? An accessor exception must be thrown when there is no item for a given key; or, an item for a give key cannot be converted to the expected type. Let's examine the getBoolean() method to illustrate the need for an accessor exception.

  1. The getBoolean() method could be declared like this: 
      public boolean getBoolean( String key );

    This method is limited to return true or false. Which one of these values could indicate an error? Both true and false are valid. Even though it is not declared, this method can throw a runtime exception. registry.AccessorException could extend java.lang.RuntimeException, instead of java.lang.Exception.

  2. Or, the getBoolean() method could be declared like this: 
      public boolean getBoolean( String key )
	 throws AccessorException;

    Declaring the accessor exception makes it explicit. Everytime an accessor method is used, you must put it inside a try..catch block.

ModifierException

The modifier exception is provided for the similar reasons. The modifier exception is thown when a registry implementation doesn't support the type of value; or a item with a given key cannot be modified (read-only).

public class ModifierException
	 extends Exception {
  public ModifierException() {
  }

  public ModifierException( String v ) {
	 super( v );
  }
}

Example

So, the setXXX() methods are declared to throw a modifier exception. The setBoolean() method is declared something like this: 

  public void setBoolean( String key, boolean v )
		throws ModifierException;
