Help Wanted

This package needs help, too. We need Java code to import an XML file into a generic registry. We also need an implementation of the registry interface to fully simulate all of the behavior of an XML file. Who knows XML? Who will work on this Java package?

Questions

  • Is it possible to import an XML file into a registry and export it perfectly? Does the import file match the export file byte-for-byte?
  • What XML translator should be used? Should an XML translator be used?
  • Is it possible to create an implemenation of the registry interface that provides seemless persistance? Every setXXX() method must re-write the XML file so that everything saved in the registry is saved to disk.

XML-to-registry redesigned

The general x-registry: scheme returns an instance of gchii.library.registry1c.Registry. This model supports hiearchial configuration information with an unlimited level of depth.

The x-registry:xml: scheme is still planned. It should return an instance of gchii.library.registry1c.Registry from an XML document. An XML-to-registry conversion program maps the information in an XML document to a registry model.

For the XML component, the document object model (DOM), or org.w3c.dom.Document, is useful for working with small XML documents. The new document:xml: scheme now returns an instance of org.w3c.dom.Document from an XML document. With this scheme, it might be easier to now develop an XML-to-registry conversion program: * An XML-to-registry conversion program converts an instance of org.w3c.dom.Document, from document:xml:, to gchii.library.registry1c.Registry.

For more information, see also SmartApiDocumentXmlScheme .
