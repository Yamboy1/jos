RegistryDesign; RegistryURL

Article contributed by GilbertHerschberger (4 June 1999).

Introduction

Here are a few of the things I would like to see in a registry editor.

  • registry browser
  • edit any registry anywhere.
  • use the registry API.
  • provide value editors for all primative Java types
  • provide value editor for a registry interface

Example

A registry editor should look like a browser, or better, run as a add-in to a browser:
Back Forward Reload Home Search Print Stop
Location: x-registry:net:#hosts/host1/users/user1/editor 
Key-1			  Value-1
Key-2			  Value-2
Key-3			  Value-3
Key-4			  Value-4
Key-5			  Value-5
:
Key-n			  Value-n
Status

Reload

The contents of a registry should be cached locally. If you want to see the latest content, the reload button forces a new copy of a registry.

Property editor

The registry editor should work just like a property editor for a Java Bean. Each kind of value should have its own value editor. When you click on a value, you can change it with a value editor. Opening a value that's a registry type changes the location to the new registry.

Since each item in a registry has a type, a mechanism should be provided to display data type on demand (like mouse over); display data type with color-coding; display data type icons. There should be a mechanism to change the data type of any item on demand. If possible, data should be automatically converted by the editor from one type to another.

Location history

As you explore the registry, the location changes. The back button goes back to the previous location in location history. The forward button moves forward in location history.

Bookmarks or favorites

The registry editor should provide bookmarks, so that you can come back to your favorite registry in a hurry.

Save as...

The registry editor should provide an easy way to save the current registry to another URL. This is like a File/Save As.. option on a pulldown menu.
