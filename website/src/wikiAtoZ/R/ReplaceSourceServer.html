ProductsPages; AboutSource

Introduction

Pages on jos.org server are obsolete and misleading. They are obsolete because source code from Source Server is now available through CJOS CVS ( CJOSProject ) on SourceForge. They are misleading because they suggest that no code has been contributed to this project since 1998. No programmer should attempt to upload their code through the Source Server.
  • http://www.jos.org/source/

Plan

  • Create a corresponding archive in .tgz and .zip format.
  • Replace index.shtml with index.html within the /sites/jos/www/source/ directory on jos.org server. Remove all scripts within the /sites/jos/www/source/ss/ directory on jos.org server.
  • A link might be provided for both .java.txt and .raw.txt files.

2 February 2001

  • Preview of a new index is http://www.jos.org/redist/srcserv/source.html .
  • Created permanent record of original files from /jos/jos-www/source/ on CD-R.
  • Wrote four Java programs to convert Source Server files to static HTML pages. This program reads from one directory tree and writes to another. It reads from /jos/jos-www/source/ and writes to /jos1i/srcserv/. The resulting static pages look identical, but will not require server-side includes. They can be viewed on both CD-R and distributed as a self-contained archive. These pages are a new feature of the JOS Technical Edition ( AboutJOSTechnicalEdition ).
  • The Upload Source feature is disabled on replacement pages. The Go To Wiki feature has been revised. Although JOS Wiki has been running on the metamech.com server for more than a year, the Source Server pages mistakenly pointed to jos.org. Replacement pages correctly point to JOS Wiki on metamech.com.
  • Links to an author within the source code have been modified. On replacement pages, an author link point to an author's article on metamech.com instead of jos.org. .java files are stored with a .java.txt extension. .raw files are stored with a .raw.txt extension.
  • The SourceServer and PrettyPrinter pages on JOS Wiki had broken links. When written, these articles were installed on jos.org, not metamech.com. Turned each broken link into an HTML comment.
-- GilbertHerschberger

31 January 2001

  • Replacement for Source Server can be found at the following location on the Internet:
    • http://www.jos.org/redist/srcserv/
  • Completed the following programs to convert Source Server pages to static HTML. No server-side includes are required. These pages are now the Source Tree feature, part of the JOS Technical Edition.
    • PageProgram Converts a .java.html page.
    • PageTreeProgram Invokes PageProgram for every .java.html file in the source directory tree.
    • IndexProgram Converts an .shtml page, resolving every server-side include.
    • IndexTreeProgram Invokes IndexProgram for every .shtml file in the source directory tree.
-- GilbertHerschberger

29 January 2001

  • Downloaded /sites/jos/www/source/ directory tree from jos.org.
    • Installed WS_FTP95 from ipswitch.com ( http://www.ipswitch.com/ ).
    • Configured WS_FTP95 for JOS Project on jos.org server on the Internet.
    • Created /jos/jos-www/, a local directory on world, one of my Linux servers.
    • Downloaded every file from source and all of its subdirectories.
  • Source Server appears to be dependent on dynamic features of an HTTP server. Many pages are stored with an .shtml extension. This requires an HTTP server to support server-side includes. The include files are
    • .packages
    • .classes
    • .exceptions
    • .errors
-- GilbertHerschberger
