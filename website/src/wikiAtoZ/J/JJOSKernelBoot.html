Article contributed by GilbertHerschberger (14 May 2001).

Introduction

Release 10 of JJOS 1i was able to boot and run a little. Later, changes in the JJOS CVS on SourceForge seem to prevent it from booting. It never displays the "KERNEL: boot" or "KERNEL: A20 OK" messages.

What went wrong? Is it something in jJOS, decaf or the standard C library? It would be far easier to determine what went wrong if we could run kernel test programs. We should have a kernel regression test handy to demonstrate the machine-code features available in jJOS kernel.

Test program

Rather than boot the jJOS kernel with the decaf virtual machine, I would like to boot jJOS with a kernel test program. This would help me understand what kind of C/C++ code works inside jJOS. I'm sure that at one point in time, this was possible. A stand-alone boot would demonstrate how to write to a console in machine code.

For what I want to do, the kernelentrypoint() function does not invoke decaf_main(). I would like to decide at compile-time which program (kernel module) to use. Ideally, this would be decided at link time.

Benefit

A test program would be able to demonstrate a machine code console, heap management and parts of the standard C library without starting the decaf virtual machine.
