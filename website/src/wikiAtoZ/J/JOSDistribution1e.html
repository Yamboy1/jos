19 April 2000

  • The first release of BytecodeResource should be available soon. I am testing each piece as I work through decoding a class file in C++. This has implications beyond bytecode stored as a resource in the DATA segment of an executable image. It means that a complete Java application can be distributed as a machine code executable, combining a virtual machine and both standard and custom bytecode. There is no classpath required for its primordial class loader.

  • Release 2 of WebSource 3b is available for immediate download.

Ryan Heise reported a bug where %WebSource 3b - Release 1 creates an empty package index. The GeneratorProgram was invoked, but immediately threw a null pointer exception. No HTML pages were generated for each .java file, giving an empty package index.

Release 2 is a patch to ConfigurationTool and WebSourceProgram. In release 1, A few methods of ConfigurationTool threw a null pointer exception when its configuration file was missing. The null pointer exception is caught by the exec() method. In release 2, a missing configuration is handled more gracefully.

In release 1, WebSourceProgram did not check the result code returned by the exec() method. This meant that no error messages were displayed when a subprogram failed. In release 2, an appropriate error message is displayed when a subprogram returns a non-zero result code or throws an exception. -- GilbertHerschberger

  • Release 4 of ERIC 1b is available for immediate download

High quality software is my priority. Software has to work. The more often software is used, the higher quality it will be. I build software that I use. I depend on my own software to get my work done. I distribute and document it after I've had a chance to use it.

Today, I happened to be working on another computer. This computer is usually powered down. It is not my usual computer. It does not have all my tools. Then, unexpectedly, I needed to run a PackageFileDoc program I'd written in the Java programming language. Fortunately, I had already installed the Java Runtime Environment and ERIC for Windows(tm). I was pleasantly surprised. I created an ERIC shortcut from scratch using a plain-text editor. I ran the shortcut from a command line with eric shortcut.eric. ERIC downloaded packages files from the network HTTP server and ran the program I wanted. ERIC saved me from distraction.

I took a few extra days to test my release candidate. I found and fixed two high priority defects. ProgramContext was always meant to work with EmbeddedURIs. Now that URITool is available, I fixed it.

But earlier, I had invented a work-around for MainProgramController because of the defect in ProgramContext. With this release, the work-around has been removed. -- GilbertHerschberger

27 February 2000

A limited version of JOS Technical Edition - Release 1e is available for download. My website contains a download directory of my favorite archives. Selected links are also available on this page. We are working toward a goal where the full version is available for download.

  • View source

    Our website has a new feature. It is the syntax-highlighted pages of source code. Links between classes are automatically generated by WebSource. Start at the beginning with products or packages.

    This feature is experimental. It duplicates part of the functionality of Source Server at http://www.jos.org/source/ . Only a snapshot archive of the Source Server will remain. It will be shut down. It will be taken off-line. It will be removed from the www.jos.org server.

  • CD-ROM

    Currently, a full version of JOS Technical Edition - Release 1e is only available on CD-ROM. What's on the CD-ROM? Here is a preview. (This preview has links that are known to be broken.)

  • WebSource

    Unfortunately, I haven't had time yet to package WebSource for distribution. It created these static, syntax-highlighted HTML pages. It is a free license and open source product. It was written in the Java programming language. It uses the SmartAPI (of course!) to read .java files and write .src.html files.

    WebSource does not reformat the original code. Unlike PrettyPrinter, it shows the source code as the programmer wrote it. Unlike Source Server, it automatically generates links between classes.

Take a look at the new pages. Send comments or questions to me, GilbertHerschberger .
