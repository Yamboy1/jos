Article contributed by GilbertHerschberger (15 June 2000).

Introduction

Here is a partial description of what I've done to optimize SuSE Linux 6.0 for Java. It is my entry-level recommendation for Jannux(tm).

Base

For a base installation, I have made at least the following assumptions:
  • To use an off-the-shelf, machine code, multi-tasking Linux kernel as is, without further customization. It has both a real and virtual memory manager.
  • To use LILO as a boot loader for Linux.
  • For a file system, fdisk and mkexfs2, to use the machine code tools provided by SuSE. For cd, ls, rm, rmdir, mkdir, etc. to use the machine code tools provided by SuSE.
  • To use machine code device drivers provided by SuSE.
  • To use the Linux executable format (ELF) for dynamic shared libraries, such as java.dll.
  • To install whatever Java Development Kit (JDK) is provided by SuSE. This JDK is most often used as a Java Runtime Environment (JRE) since I compile most of my Java source code under Microsoft Windows 95.
  • To use built-in TCP/IP. (Every one of my machines have a network interface card.)
  • I use a simple editor I wrote in Java. Sometimes, I use a vi-compatible editor.
  • I use whatever default shell is provided by SuSE. I also use YaST from SuSE.

Client-side

Sometimes a base Linux machine is further refined as a workstation on the client-side.
  • To install a C/C++ compiler (gcc) to compile additional software to machine code.
  • To install make provided by SuSE.
  • For CVS, I use jCVS from the Giant Java Tree. I do not use the machine code version provided by SuSE.
  • For X, I use the XFree86 software provided by SuSE. Once X is configured, I use AWT/Swing on a workstation.
  • In addition, I have installed jJOS/decaf in host mode on one Linux workstation (it does not yet compile). I have also created an Etherboot boot diskette to boot jJOS/decaf on a workstation without Linux. Plus, I have installed NASM and binutils provided by SuSE. I'm fairly certain I installed Etherboot provided by the JOS Project.

Server-side

Sometimes a base Linux machine is further refined as a server.
  • For file and print services using SMB, install Samba.
  • For an HTTP service, install Apache. I have not yet configured Apache for Java servlets. Instead, I use another Java application for servlets on a separate TCP port.
  • For TELNET, FTP, TFTP and BOOTP services, I use the products that are provided by SuSE. I configured inetd. I have configured bootpd. I have installed jJOS/decaf and configured tftpd.
  • I do not configure X on a server. I do not run AWT/Swing applications on a server.
  • I configure my Linux servers through telnet provided by either Microsoft or SuSE. My servers do not have a monitor, mouse or keyboard. Some do not have a floppy drive or floppy drive controller.

Optimized

Here are some of the things that I have done to optimize Linux for Java. It's not much.
  • I have installed the UniversalBrowser on both Linux and Windows. I spend most of my time using it instead of another browser/desktop/shell.
  • I have created a desktop file with a short list of Linux configuration files.
  • Sometimes, I am able to use Find from the SmartAPI rather than find provided by SuSE.

Missing

Optimizing for Linux sometimes means much machine code distributed with Linux can be ignored.
  • No desktop (like KDE).
  • No X applications.
  • No games.
  • No SQL server.
  • No TCL. No Python. No CGI scripting.
  • No mail. No newsgroups.
  • No dial-up networking (neither SLIP nor PPP).
  • No other HTML browser.

Future

This configuration works for me today. I would like to see Java applications explore what has never been done before. I would like to see more Java applications that are as rugged and reliable as their C/C++ counterparts.
Install the linclude package under C/C++ development. For a typically JJOS developer, the Linux(r) include files must be installed. Without them, GRUB will not compile. -- GilbertHerschberger (8 August 2000).
Now, I also have a workstation with Caldera OpenLinux. I haven't yet had the time to optimize it for Java. How I miss the UniversalBrowser! Rather than optimize, I'm spending my time on a Linux-compatible eric tool. -- GilbertHerschberger (23 August 2001).
