Article contributed again by GilbertHerschberger (31 May 2001).

Introduction

While debugging bytecode running inside JJOS 1i, it would be helpful for a programmer to know the name of the current method. The name of the current method could be used in opcode methods when tracing a thread. The name of the current method is also needed for a Java-style stack trace.

A frame must be created for each invocation of a bytecode method. A frame interprets the Code attribute of the method and handles exceptions. Frames are part of a stack. Frames are pushed onto the stack when a bytecode method is invoked. Frames are popped from the stack when a method returns or an unhandled exception is thrown.

Stack trace

This proposal may make it possible for JJOS 1i to print a Java-style stack trace. A Java-style stack trace should print (a) the current class name, (b) the current method name and (c) the program counter (myPC) for each frame on the stack. A Java-style stack trace should replace the C-style stack trace that's currently implemented.

A frame already has a class property and program counter property; but, it does not yet have a resolved method property. Without reworking the frame class, a stack trace cannot find the current method name.

Changes to frame

Look at the frame's constructor. Currently, the calling function must extract the Code attribute from a resolved method and pass this to a frame. Since the calling function extracts the Code attribute on behalf of the frame's constructor, other properties of a resolved method are lost--including the name of the current method.

In C++, a resolved method should be a property of a frame. A resolved method should be passed to a frame and stored in this field. A resolved method property should be read-only.

  • The getMethod() method should return the resolved method of a frame. Its signature is resolve_method *getMethod() const;.
  • The getClass() method should return the resolved method's class. It is redundent (isn't it) to pass a resolved method and a class to a frame when a resolve method has a class property. Its signature is java_class *getClass() const;.
  • The frame's constructor should be modified to extract the Code property from the resolved method. When constructing a frame, the calling function can pass the resolved method and current thread.
