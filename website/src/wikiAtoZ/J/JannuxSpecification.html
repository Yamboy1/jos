Introduction

Programmers have often wondered if Linux could be optimized for Java. Just think of the possibilities! What parts of Linux would we keep? What parts would we want to replace? This article starts an effort to organize Java programmers to find an optimized version of Linux for Java development.

Process

For now, the most important part of this specification is the long-term process of creating and maintaining it. There are few assumptions about the purpose of the resulting operating system. Both server- and client-side recommendations are welcome.

  • Scope

    We will provide documentation, such as installation and configuration check lists. We have no intention of re-inventing the supply chain for Linux. We do not want to get directly involved in the distribution or development of the Linux operating system. Other people can do that.

    We will point the way. We have no intention of re-distributing all of the Java applications that might be recommended for optimizing Linux. Other people can do that, too.

  • Starting point

    Download or purchase any off-the-shelf distribution of Linux. This is the most direct way to optimize Linux for Java. The time to optimize Linux for Java is when you install and configure Linux on your Jannux(tm)-compatible machine.

  • Recommendations

    The Jannux(tm) Specification is a directory of websites. Add your recommendations to this specification by writing to the applications mailing list. In the coming months, we'll ask Java programmers for specific recommendations for optimizing Linux for Java.

  • License

    The Jannux(tm) Specification itself is covered by the LGPL license, unless otherwise stated. This enables the open source community to create and maintain a specification.

    We fully expect the Jannux(tm) Specification to recommend both commercial and non-commercial products. Individual programmers may have a license preference. This is a personal decision on a product-by-product basis. To assist in license, each product should be clearly marked as commercial and/or non-commercial.

De-facto standard

Many programmers have already optimized Linux for Java. How did they do it? What products did they chose? We look forward to documenting a de-facto standard, a standard that many programmers already use.

Future

This is an open specification. In the future, Jannux(tm) will be what you make of it. How can the de-facto standard be improved? Which products written in Java are better than their C/C++ equivalent? Help your fellow Java programmers to optimize the Linux platform.

Summary

Optimizing Linux for Java is a great idea. It's a classic. The time has come to organize. Help the community do something about it.
