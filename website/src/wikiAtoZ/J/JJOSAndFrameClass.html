Introduction

The opcode methods for the decaf virtual machine are implemented in a C++ class called frame. Each opcode has a corresponding method in the frame class. An instance of frame is the context in which the Code attribute is interpreted. It might be said that the frame class is the core interpreter for decaf.

Implementations

The frame class is implemented differently in version 1e, 1f, 1i and 2a. In version 1e, most of the opcode methods are defined in the common/decaf/interp.cc file. Although no opcode method can "throw" a throwable exception object, each returns a boolean (bool) value and has a java_object parameter.

In version 1i, the opcode methods were simplified. Opcode methods can "throw" a throwable exception object using the frame::setException() method. Many opcode methods do not yet throw an exception. Rather, they use the stub macro called THROW_ EXCEPTION_HERE. In the common/decaf directory, the opcode methods were written into the following source code files:

  • interp.cc,
  • interpAtoH.cc,
  • interpI.cc,
  • interpJtoZ.cc,
  • invoke.cc, and
  • invokeop.cc.
In version 2a, the opcode methods should be simplified as they were in version 1i. Unlike previous versions, the source code for the frame class in C++ is found in the common/frame directory. Most of the opcode methods will be written into the following source code files:
  • opcodea.cpp
  • opcodec.cpp
  • opcoded.cpp
  • opcodef.cpp
  • opcodeg.cpp
  • opcodei.cpp
  • opcodej.cpp
  • opcodep.cpp
  • opcoder.cpp
  • opcodes.cpp
Important methods of the frame class have been given their own source file. The frame::monitorenter() and frame::monitorexit() opcode methods are found in monitor.cpp. The frame::getfield() opcode method is found in getfield.cpp. The frame::dynLink() method is found in dynlink.cpp.
