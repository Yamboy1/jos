November 25, 1999

Release 3 is available for Smart API - Version 2a. Download it now.

  • The Desktop browser provides a way to "browse" a desktop file. It is yet another browser in a long list of potential browsers. From the desktop, click on an icon (or whatever) to set a new location. The default desktop typically provides icons for all your desktop files. Put your desktop file(s) on the Internet/intranet to share your desktop with others. With the desktop browser at one end and and the internet package manager at the other, the traditional concept of installation has been retired.

  • Automatic browser selection has been added to the Universal Browser. This makes a strange new experience a little less overwhelming. A universal browser takes time to get used to. Automatic selection is limited to off-the-top-of-my-head protocols. Expect it to be refined. The Universal Browser is a browser of browsers. It brings many different browsers under a common umbrella.

  • This distribution is one step closer to bringing JOS and Jose together. The program browser runs José(tm) applications from Releases 1b, 1c and 1d. To run the José(tm) shell, here's the URI for the Program Browser (or Universal Browser):

    run:jose:com.realignsoft.jose1d.RunApplication

  • The Connection Browser has been upgraded to accept four fragments to find out other things about a JDBC connection. They are:

    • catalogs - uses getCatalogs()
    • procedures - uses getProcedures()
    • schemas - uses getSchemas()
    • tables - uses getTables()

  • A few packages were omitted from previous source and binary distributions. The org.jos.gchii.programbrowser.bridge is a backward-compatibility bridge for program browser. Without a bridge package, a program browser would be useless. With this distribution, program browser can run programs for all four releases of the program API.

  • This is a good time to take the Universal Browser for a test drive. A public static void main( String[] ) method has been added to the Browser class, so that the Universal Browser is backward compatible with the java tool. Start the Universal Browser with a OS command, something like this:

    $ java org.jos.gchii.browser1a.Browser

    As a Java application, you can pass a location to the Universal Browser as a command line argument, like this: 

    $ java org.jos.gchii.browser1a.Browser http://www.jos.org

    Universal Browser is both a Java application and a José(tm) application. From the location field of the Universal Browser, start another instance with this URI: 

    run:jose:org.jos.gchii.browser1a.Browser

For everyone with a preferrence Linux/UNIX, this distribution includes a .tgz file for Java source code. -- GilbertHerschberger

November 24, 1999

Distribution 1 is available for BCNI API - Release 1a. This is an improvement over the Factory class discussed on the architecture mailing list. It includes a Factory class in the org.jos.bcni1a package. The Factory class is the native interface to JVM-specific classes. It also includes a BCNIController class in the org.jos.gchii.smartbcni1a package. The BCNIController class is a Smart API controller for Factory. Download it now.

Distribution 2 is available for Smart API - Release 2a. This release includes the MainController class in the org.jos.gchii.smartmain1a package. MainController enables the program browser to run any class that implements public static void main( String[] ). MainController uses practical reflection available starting with Java 1 Platform. Download it now.

Unfortunately, I had to disable "JOS Wiki - Off Line" temporarily. It is over 3MB and I needed the space today. -- GilbertHerschberger

November 19, 1999

Loosly-coupled class libraries should be the foundation of the JOS Project. Why? Because JOS isn't like all those other operating systems. JOS is extremely extendable, as individual machines plug-in only the subsystems appropriate to a customized environment.

The SmartAPI is aware of, but not dependent on, other subsystems. It is stable and expandable. It runs perfectly on a Java 0 platform. It is independent and friendly with Java 1 and 2 Platforms. It is poised, read to run on whatever comes next. A distribution of the Smart API has been collected, pre-compiled and pre-archived. It is available now:

BCNI API - Version 1a

For more information, see also DownloadBCNI.

Smart API - Version 1a

For more information, see also DownloadSmartAPI.

Smart API - Version 1b

For more information, see also DownloadSmartAPI.

Smart API - Version 2a

For more information, see also DownloadSmartAPI.

These are for your evaluation only. The UniversalBrowser and all of its plug-ins are available in Version 2a. -- GilbertHerschberger
