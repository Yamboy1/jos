Introduction

The newarray opcode enables a Java method to create a new primative array. This method should be available in machine code, so that any C++ method can create a primative array. And, so that any C++ method can create an instance of java/lang/String.

Rewrite

A header file should contain separate methods to create each of the primative array types, including the char array. These methods should return an instance of jobject. For a char array, the native getClass().getName() method should return "[C".

Where is the native char array class? It should be an instance of jclass.

The [C class is an instance of java/lang/Class. It is not loaded, but provided by the Java Virtual Machine. It implements the length() method, which returns the length of a primative array. This example prints "length=100". 

  public void example() {
    char[] x = new char[ 100 ];
    println( "length=" + x.length() );
  }

And then, the frame::newarray() method should be rewritten so that it invokes separate methods to create a primative array. 

// cpp/src/frame/createarray.h

extern "C" {
  jobject mcCreateByteArray( JEnv e, jint v );
  jobject mcCreateCharArray( JEnv e, jint v );
  jobject mcCreateShortArray( JEnv e, jint v );
  jobject mcCreateIntArray( JEnv e, jint v );
  jobject mcCreateFloatArray( JEnv e, jint v );
  jobject mcCreateLongArray( JEnv e, jint v );
  jobject mcCreateDoubleArray( JEnv e, jint v );
  jobject mcCreateObjectArray( JEnv e, jint v );
}

Creating a string

When creating a string for an environment, first create the char array with the mcCreateCharArray() method. When converting from native C-style string, convert from ASCII (or rather, UTF8) to unicode. The char array contains the data for the java/lang/String class.

The payload field in a string is a reference to a char array. Methods of the java/lang/String class, such as substring() and indexOf(), expect non-null payload.

Printing a string

When printing a string, characters should come from the char array, not from the string itself.

Item size

Each item in a char array should be 16 bits, or two octets. An item should not require 32 bits of storage.

Java environment

Within a kernel, there may be one or more Java environment. Each Java environment has heaps (and threads). When a Java environment is destroyed, its heaps are destroyed. The first Java environment is a primary domain. A thread has a stack. A stack has stack frames. This environment is passed to other methods in C++ so that all objects belongs to an environment. A thread belongs to an environment so that when a Java environment is destroyed, all of its threads are destroyed.

Primary domain

The primary domain is the only domain capable of direct access to the hardware. Therefore, the primary-domain class in C++ extends the domain class. The domain class provides the basic behavior for all domains. The secondary-domain class extends the domain class with specific behavior for all secondary domains. A secondary domain cannot directly access the hardware. Rather, it can interact with the primary domain drivers.

Custom application

The command line should provide a mechanism to invoke a custom application. An operating system should invoke the primary domain start-up application and power-on self-test before invoking a custom application in a secondary domain.

Heap walk

When every object on the heap is an instance of java/lang/Object, it becomes possible to "walk" the heap and discover all objects derived from an arbitrary class. To stop all threads, for example, a C++ method can walk the heap to find all instances of java/lang/Thread and invoke its stop() method. To finalize a heap, a method can walk the heap and invoke the finalize() method of every object. A heap does not need to be contiguous. A "heap" property of a memory block determines membership in a heap.

When a memory block and a Java object are one and the same, "heap" is a field of the java/lang/Object class. Otherwise, "heap" is a field in a memory block and a Java object is another field.
