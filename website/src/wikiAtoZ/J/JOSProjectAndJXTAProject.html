Introduction

Here is a comparison of the government of the JOS Project and the government of the JXTA Project ( JXTAProject ).

Government

Like the JXTA Project, this project also makes its decisions in the open, on a public mailing list. It is important for all members to have a voice in the direction and the policy of the project. A smaller group makes binding decisions. The smaller group is called a "board" in the JXTA Project and "voting members" in the JOS Project.

Like the JXTA Project, members of this project play roles. The following roles are almost identical between the JXTA Project and this project:

  • Developer
  • Contributor
  • Committer
  • Project Owner
While the JXTA Project has a board of directors, the JOS Project has a small group of voting members. Voting members provide the official policy of the project.

And unlike the JXTA Project, this project is not sponsored by Sun Microsystems nor any other commercial organization.

Project Owner

Each "sister" project has one owner, usually the person who started the project. The project owner has full control over such implementation details as programming language, development platform, software license and distribution. In the CJOSProject, for example, I am the project owner.

The JOS Project is working on a Java-based operating system. This is a difficult subject. It is difficult, if not impossible, to determine in the beginning which implementation will be the right one. Unlike the JXTA Project, the JOS Project has limited its members. We do not endorse only a single version of the Java Software Development Kit; no kit is forbidden. We do not recommend only one programming language; no language is forbidden. We might not officially recommend a single software license; no license is forbidden. We do not recommend one development platform; no platform is forbidden. Rather, we delegate these and all other implementation details to a sister project and its project owner.

Committer

A committer must integrate the feedback (and possibly source code) into the official product distribution. For sister projects on SourceForge, there is tight security using secure socket layer and commit authorization to CVS. No member has commit access to all sister projects. A committer can only make changes to the projects they are authorized.

Naturally, the project owner is the primary committer. As needed, a project owner can set up accounts for other committers. In the JJOSProject, for example, both ToddMiller and I are committers.

Contributor

Others have contributed their feedback in the development of defect-free software. Some have literally contributed source code. Thanks to all of you who have contributed in the last four years!

Contributors are not required to be members. Contributors can contribute annonymously.

Developer

Our sister projects are set up to enable annonymous FTP access to current distributions. Most are set up to enable annonymous CVS access, too. Both members and non-members download the product and use it.

Developers are not required to be members. Developers can use the software annonymously.
