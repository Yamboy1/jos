Article contributed by GilbertHerschberger (18 June 2001).

Introduction

The latest release of JJOS 1i is release 12. It seems to have a problem within the frame::dynLink() method in C++. This method is used to dynamically link to an interface method or a method in a superclass.

Example

In the Java programming language, the decaf.trace1a.BasicTrace class implements the jos.platform.Trace interface. The interpreter is having difficulty invoking the setState() method of BasicTrace in the following assembler instruction:

invokeinterface jos.platform.Trace.setState(Z)V

The object on the stack is an instance of BasicTrace. In C++, the frame::invokeinterface() method should create a frame for the BasicTrace.setState() method, not the Trace.setState() method.

The dynLink() method in C++

According to the comments, invoking an interface method is the responsibility of the frame::dynLink() method in C++.

Why does this method return a java_class?

It seems to me that this method should return a resolved_method from the right java_class.

The java_class::lookupMethod() method returns a pointer to a resolved method with the given key. A key includes both the method name and descriptor. 

  constant_utf8 *name = rm->getName();
  constant_utf8 *desc = rm->getDescriptor();

Here, rm is the original abstract method. If a method is abstract, its Code attribute should be ignored. A standard compiler does not generate a Code attribute for an abstract method.

The name and descriptor for the original method must be compared to each method in a class. Traditionally, methods are compared in reverse order. Recursion is used to find a resolved method in a superclass. 

  for ( int x = xMax; x >= xMin; x-- ) {
    resolved_method *m = getMethod( x );
    constant_utf8 *n = m->getName();
    if ( !( *n == *name ) ) {
      // method with a different name
      continue;
    }

    constant_utf8 *d = m->getDescriptor();
    if ( !( *d == *desc ) ) {
      // method with same name, different descriptor
      continue;
    }

    if ( m->accessFlags() & ACC_ABSTRACT ) {
      return getSuperclass()->lookupMethod( rm );
    }

    return m;
  }

The java_class::getMethod() method returns a resolved method for a given index. It is implemented in C++ something like this: 

  resolved_method *
  java_class::getMethod( jju16 methodref ) {
    if ( methods == NULL ) {
      return NULL;
    }

    MethodInfo *mi = methods->operator[]( methodref );
    if ( mi == NULL ) {
      return NULL;
    }

    return mi->_to_resolved_method();
  }

Class property

In C++, the getClass() method replaces the classPtr() method. It replaces the direct use of the cur_class field. It returns an pointer to an instance of the java_class class.

Name property

In C++, the getName() method replaces the direct use of the class_name field. It returns a pointer to an instance of the constant_uft8 class.

c_str() method

In C++, the c_str() method has been added to the resolved_method class. It replaces separate calls to c_str() on the java_class and resolved_method. The following example illustrates how to print a method's name. 

  void example( resolved_method *rm ) {
    printf( "  %s method.n", rm->c_str() );
  }

This was the old way: 

  void example( resolved_method *rm ) {
    printf( "  %s.%s method.n",
        rm->getClass()->getName()->c_str();
        rm->getName()->c_str();
        );
  }
