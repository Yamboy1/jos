May 31, 2000

  • Release 1 of BulkMake(tm) 2a is now available for immediate download. This product contains a package-oriented tools for copying, compiling and archiving. This is the first version to support compiler configuration without modifying the source code. It depends on SmartAPI.

    • The location for Package Copier is run:program:gchii.tools.packagecopier2a.PackageCopier.

    • The location for Package Compiler is run:program:gchii.tools.packagecompiler2a.PackageCompiler.

    • The location for Package Archive is run:program:gchii.tools.packagearchive2a.PackageArchive.

I've been using BulkMake(tm) for two years. It automatically searches for .java files and does not require an IDE-specific project file. -- GilbertHerschberger

May 9, 2000

  • Release 9 of the Smart API 2a is now available for immediate download. Does it contain both the org.jos.finder1a and org.jos.gchii.find2a packages? It should. In addition, it contains the org.jos.gchii.smartlist1a and org.jos.gchii.smartprocess1a packages. The Smart List program displays the current configuration of the Smart API. The Smart Process plug-in adds the process: scheme, as described in SmartProcessProtocol .

Next, I'll work on a snapshot of JOS Wiki with all of the new articles introduced last quarter. Look forward to an update of the Glossary and pages A thru Z.

Improved coordination

In theory, binary and technical edition should match exactly. This might be possible with this version. With this version, we have another opportunity to coordinate these editions. When a technical edition is used as the basis of a binary edition, a binary and technical editions would match exactly.

This has been difficult to do because of DOS vs. UNIX problems in the technical edition. This was very close to being solved in release 2 of version 1e.
