BytecodePages

Article contributed by GilbertHerschberger (4 December 2000, revised 9 August 2002).

Introduction

A post-compile verification of a class file can be performed statically or dynamically. This rule can be verified statically.

Class Constant Pool Index

Each constant pool entry must be one of a limited number of types. The Constant.CLASS is one type of constant pool entry. A constant pool index (CP) is a reference to a constant pool entry. Zero is always an invalid constant pool index.

Class CPs

At least, the following CPs must point to a corresponding Constant.CLASS entry in the constant pool.
  • ThisClassCP
  • SuperClassCP
  • InterfaceCP[]
  • ThrowsCP[]

This class CP

This field is found immediately after the 16-bit class access field. It is used to determine the spec (and therefore, name) of this class.

Superclass CP

This field is found immediately following the ThisClassCP field. It is used to determine the spec of a superclass. Exclusively for the java/lang/Object class, this field is intentionally an invalid CP of zero (0) because it has no superclass.

If this class is an interface, the superclass spec must be equal to java/lang/Object.

Dynamic rule:> If this class is not an interface, the superclass must likewise not be an interface.

Interface CP array

This array is found immediately following the 16-bit interface count field. It is used to determine the spec of each interface implemented by this class.

Dynamic rule: The name of each class in this field must be an interface, not a class or abstract class.

Throws CP array

This field is found inside an optional Exception attribute of a method. It is an array of constant pool indexes. Each index must point to a Constant.CLASS entry.

Dynamic rule: Each class in this array must implement the java/lang/Throwable interface.

Verifying a CLASS entry

Each Constant.CLASS entry has a CP field. This field must be a valid CP for a UTF8 entry. Only certain characters are allowed in a class spec. This class is valid only if these UTF8 entries contain a valid class spec.

Dynamic Rule: A class must exist for each Constant.CLASS entry.
