About JDBC

Introduction

This section examines the purpose of JDBC and its contribution to the Database Design Studio.

Purpose

Java Database Connectivity (JDBC) is a platform-independent framework or API. It is written in the Java programming language and available to your Java-based programs. It is based on a plug-in architecture.

Plug-in

There are many implementations of a database. Thus, it is not practical for a common framework itself to provide a protocol for every database. The common framework would become bloated with classes that your application might never use.

This is a good case for a plug-in architecture. A common framework provides an interface for a custom plug-in. A custom plug-in provides an implementation to connect to a specific database.

Driver

Within JDBC, a plug-in is called a JDBC driver.

How used

The Database Design Studio is a collection of platform-independent, database-independent tools. Its tools examine a database, collect information about a database (meta-data, download the content of a database (data). A tool is written in the Java programming language to maintain platform-independence. A tool depends on JDCB to maintain database-independence.

About XML

Introduction

This section examines the purpose of XML and its contribution to the Database Design Studio.

Purpose

An eXtended Markup Language ( XML ) file is a platform-independent mechanism to store data. From a broad point of view, it is a common information file format. A collection of XML files is a information-base.

XML itself does not provide a data-oriented access mechanism, such as SQL. But as an information-base, everything that can be stored in a database can be stored in XML.

From JDCB to XML

Introduction

This section provides more information on the mechanism that can be used to export a database from a JDBC-compatible database to an XML document.

Data source

An export-to-xml mechanism needs only a JDBC data source. Using a given data source, everything about a database can be discovered.
  1. Table List. A meta-data method provides a list of user-defined tables.
  2. Column List. For each table, a meta-data method provides a list of columns.
  3. Column Definition. For each column, meta-data methods provide more information about the definition of a column.
  4. Rows. A result-set provides every row in a table.
  5. Fields. A result-set method provides the data.
Information gathered through the JDBC data source is written to an XML document.
  • The database tag provides information about a database (data source).
  • The table tag provides information about a table, including data and meta-data.
  • The meta-data tag provides information about the structure of a table.
  • The data tag provides information about the content of a table.
With meta-master, the name of an XML document reflects the name of a data source.

Conversion choice

When a field contains a simple value, it is (usually) stored as a property of the dds:record tag. 
  <dds:record f1="a" f2="b" f3="c"/>

When a field contains a complex value, it is stored as a separate tag. 

  <dds:record f1="a" f2="b">
    <dds:field name="f3">c</dds:field>
  </dds:record>

All fields are stored as text in an XML document. The dds:column tag defines the SQL type for a database column and an appropriate conversion is selected once for a table.

From date field to XML

A date is a three-part field. When converting a date field to XML, each part of a date should be stored with a separate property or tag.

For example, let's say that 29 May 1982 is a graduation date for a student. The graduationdate field in the student table is converted to a java.sql.Date object. 

  <dds:field name="graduationdate" year="1982" month="5" day="29"/>
or 
  <dds:field name="graduationdate">
    <year>1982</year>
    <month>5</month>
    <day>29</day>
  </dds:field>

From time field to XML

A time is a four-part or five-part field. When converting a time field to XML, each part of a time should be stored with a separate property or tag. 
  <dds:field name="eventtime">
    <hour>7</hour>
    <minute>5</minute>
    <second>29</second>
    <millisecond>29</millisecond>
    <xm>am</xm>
  </dds:field>

From datetime field to XML

A datetime is an eight-part field. It combines the date and time fields.

Result

The XML document contains enough information to create a copy of the database. Another JDBC data source connects to another JDBC-compatible database.
  • For each table, the meta-data tag provides enough information to compose an SQL create command. This command creates an empty user-defined table.
  • For each table, the data tag provides enough information to populate a table with data. Each row tag is used to compose an SQL insert command. This command adds a row to a user-defined table.
