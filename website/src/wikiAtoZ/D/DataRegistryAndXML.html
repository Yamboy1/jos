Introduction

This article provides questions about the possible relationship between the DataRegistry and XML.

Is it possible to write a data registry plug-in to store a registry?

Proposed x-registry:xml: scheme

An XML-compatible data registry plug-in should be mapped to the x-registry:xml: scheme. To load a registry from a local file, a uniform resource identifier might look something like this:
  • x-registry:xml:file:example.xml
To load a registry from a file from an HTTP service, a uniform resource identifier might look something like this:
  • x-registry:xml:file:example.xml

Example: Writing a data registry to XML

The following method adds data to a data registry. 
  public Registry example() {
    URI uri = new URI( "x-registry:new:" );
    Registry r1 = (Registry) uri.getObject();
    if ( r1 == null ) {
      return;
    }

    try {
      r1.setString( "n1", "v1" );
      r1.setString( "n2", "v2" );
      r1.setString( "n3", "v3" );
    }
    catch( ModifierException e ) {
      return;
    }

    Registry r = (Registry) uri.getObject();
    if ( r == null ) {
      return;
    }

    try {
      r.setString( "r.name", "r.value" );
      r.setRegistry( "r.registry", r1 );
    }
    catch( ModifierException e ) {
      return;
    }

    return r;
  }
The above registry might produce the following XML file: 
<?XML version=1.0>
<registry>
<item>
<name>r.name</name>
<value>r.value</value>
</item>
<registry>
<item>
<name>n1</name>
<value>v1</value>
</item>
<item>
<name>n2</name>
<value>v2</value>
</item>
<item>
<name>n3</name>
<value>v3</value>
</item>
</registry>
</registry>
