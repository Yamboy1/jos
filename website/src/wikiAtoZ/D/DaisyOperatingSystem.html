Introduction

An operating system is like a flower, like a daisy. You spend a lot of time working on the environment and cultivating the ground. You plant a seed and, if things go as you have planned, an operating system grows.

An operating system is not built in a day. As you would not expect a daisy to grow from seed to flower in a day, an operating system takes time. The environment must be reinvented. Devices must be cultivated.

Here is how a modern operating system grows. The first step, believe it or not, is to build applications upon a foreign operating system. Build applications which push a foreign operating system to its limits. The second step is to create a new operating system that has none of the limits of the foreign operating system. Optimize your operating system to run the applications that have already been built for it.

While it might sound like the cart is before the horse, it isn't really. A platform-independent application can run on its choice of platforms. It runs best on the best platform. Or rather, the best platform is the one where your application runs best.

Goal

Imagine you work with an operation system where the desktop is a plug-in so that you can simultaneously run multiple desktops at the same time, switching between desktops as you might currently switch between computers...

Is it necessary to provide security on a thread-by-thread basis? Within a single kernel process running on a foreign operating system, you should be able to boot your Java-based operating system. It should present its own desktop.

Should you be able to run all of your favorite applications without modification? No, you shouldn't. Likely, you'll want to run applications which have been written for your Java-based operating system from the ground up. An application should be optimized for an operating system, just as an operating system is optimized for an application.

Not only multiple primordial class loaders, but support for user accounts, passwords and profiles should be built into your Java-based operating system. Tools for maintaining your account list should be written in Java. But, drivers for your keyboard and serial port and network card and video card and audio card can be part of a foreign operating system or implemented as machine code. A Java-based operating system should not be implemented completely in bytecode; but, as much as possible should be implemented as bytecode. There are limitations to what one can do with bytecode especially when it comes to implementing a virtual machine itself.
