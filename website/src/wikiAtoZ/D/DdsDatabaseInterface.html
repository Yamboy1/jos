Puzzle

Introduction

This section describes the need for a well-known database interface.

Features

A database-aware program should be as simple as possible. It should be able to invoke an SQL command or query easily and simply.

Non-features

A database-aware program should not include the complexity of data source, connection and statement. This complexity can be moved to a library (framework or API), and should be.

Bad example

Here is a simplified example of the code required to invoke an SQL query without a database object. 
  public void badexample() {
    InitialContext ic = ...get initial context...
    DataSource ds = (DataSource) ic.get( "jdbc/josdemo" );
    Connection c = ds.getConnection();
    Statement s = c.createStatement();
    String sqlStr = "select * from t1";
    ResultSet rs = s.executeQuery( v );
    :
    rs.close();
    s.close();
    c.close();
  }

Good example

Here is a simplified example of an equivalent method that uses a database object. 
  public void example() {
    URI uri = new URI( "x-database:josdemo" );
    Database db = (Database) uri.getObject();
    String sqlStr = "select * from t1";
    ResultSet rs = db.performSQLex( v );
    :
    db.close();
  }

Proposal

Introduction

This section describes proposed features of a well-known database interface.

Database

A "database" is a highest-level object. It has-a data source, connect and statement. It encapsulates the complexity of JDBC and JNDI.

Perform SQL method

  public abstract int performSQL( String v )
      throws SQLException;

This method enables your program to invoke an SQL command. It invokes the java.sql.Statement.executeUpdate() method. It returns an integer, the number of rows affected. SQL commands include

  • alter
  • create
  • drop
  • insert
  • update
  • {call }

Perform SQL extended method

  public abstract java.sql.ResultSet performSQLex( String v )
      throws SQLException;

This method enables your program to invoke an SQL query. It invokes the java.sql.Statement.executeQuery() method. It returns a result set.

Get connection method

  public abstract java.sql.Connection getConnection()
      throws SQLException;

This method returns a connection (java.sql.Connection) for this database object.

Get statement method

  public abstract java.sql.Statement getStatement()
      throws SQLException;

This method returns a statement (java.sql.Statement) for this database object.

Get prepared statement method

  public abstract java.sql.PreparedStatement getPreparedStatement()
      throws SQLException;
This method returns a prepared statement (java.sql.PreparedStatement) for this database object.

Closing a result set

  public abstract void closeResultSet();
A result set might be open. This method closes an open result set.

A basic implementation of this method might look something like this: 

  public void closeResultSet() {
    if ( myResultSet != null ) {
      try {
        myResultSet.close();
      }
      catch( SQLException e ) {
      }
      myResultSet = null;
    }
  }

Closing a statement

  public abstract void closeStatement();
A statement might be open. This method closes an open statement.

A basic implementation of this method might look something like this: 

  public void closeStatement() {
    if ( myStatement != null ) {
      try {
        myStatement.close();
      }
      catch( SQLException e ) {
      }
      myStatement = null;
    }
  }

Close

  public abstract void close();
This method closes the database object. It closes all resources for this database object.
  1. A result set might be open. It invokes the closeResultSet() method.
  2. A statement might be open. It invokes the closeStatement() method.
  3. A connection might be open. It invokes the connection's close() method.
A database object is re-usable. Your application can invoke the close() method and then the performSQL() method, for example. A database object is intended for simplicity and efficiency. Your program should only close a database object just before it goes out of scope.

A basic implementation of this method might look something like this: 

  public void close() {
    closeResultSet();
    closeStatement();

    if ( myConnection != null ) {
      try {
        myConnection.close();
      }
      catch( SQLException e ) {
      }
      myConnection = null;
    }
  }

Data source cache

For optimum efficiency, this library should reduce the number of calls to JNDI. It should provide a cache of data source objects.

Packages

A well-known database interface should be separate from any implementation of this interface. The following package is proposed for the database interface itself: 
  org.jos.database1a
The following package is proposed for the first implementation of the database interface: 
  gchii.library.basicdatabase1a
The following package is proposed for the Smart API 4a plug-in. This plug-in re-defines the jdbc:database: scheme. This scheme should return an implementation of org.jos.database1a.Database (or an exception). 
  gchii.library.smartdatabase1x
