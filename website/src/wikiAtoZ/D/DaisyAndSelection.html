Introduction

Here are some preliminary ideas on abstract machine selection.

Configuration

Imagine that a wide variety of abstract machines are available to your Java application. A specific abstract machine must eventually be selected. When? When the Java application is invoked. If you invoke your Java application on a command line, you must be able to select an abstract machine.

Within a configuration file, a number of abstract machine can be made available for selection. Let's imagine we have installed both an experimental abstract machine and a production abstract machine. Every time we invoke a Java application, we must select one machine or the other. The production abstract machine is probably a reasonable default. So when no abstract machine is explicitly selected, the production configuration is used. 

; default.am.conf
runtimelibrary=javai

Here is a crude example of machine selection on a command line: 

$ java machine=test org.jos.demo.Main
The java tool must look for a corresponding "test" abstract machine configuration. 
; test.am.conf
runtimelibrary=javax
